# 薄壁统一映射调整记录_20260402

## 背景

`CADExchange` 之前的薄壁语义更接近 `SW/Creo` 的单厚度模型：

- `thickness`
- `isOneSided`
- `isOutward`
- `isCovered`

这无法完整表达 `UG/NX` 的 offset 语义，因为 `UG` 薄壁需要同时保留：

- 内侧 offset
- 外侧 offset
- 两侧都允许独立设置正负

因此统一层改为只保留一套定义：

- `StartOffset`
- `EndOffset`
- `Covered`

其中：

- `StartOffset` = 相对草图轮廓的内侧偏置，向内为负
- `EndOffset` = 相对草图轮廓的外侧偏置，向外为正
- `Covered` = 是否封口

XML 中不再写 `Thickness / OneSided / Outward`。

## 跨系统映射规则

### SW

`SW` 默认单侧薄壁方向向外，因此映射为：

- 单侧向外：`StartOffset=0, EndOffset=t`
- 单侧向内：`StartOffset=-t, EndOffset=0`
- 中面总厚 `t`：`StartOffset=-t/2, EndOffset=t/2`
- 双向不同厚：`StartOffset=-t_in, EndOffset=t_out`

### Creo

`Creo` 默认单侧薄壁方向向内，因此映射为：

- 单侧向内：`StartOffset=-t, EndOffset=0`
- 单侧向外：`StartOffset=0, EndOffset=t`
- 双侧等厚总厚 `t`：`StartOffset=-t/2, EndOffset=t/2`

注意：

- `Creo extrude/revolve` 的 native 模型本质上仍是 `single thickness + material side`
- 因此 `StartOffset` 与 `EndOffset` 同时非零且不满足对称双侧时，写回 `Creo` 只能做 best-effort

### UG / NX

`UG` 直接按 offset 语义落地：

- `StartOffset` 对应内侧 offset，向内通常为负
- `EndOffset` 对应外侧 offset，向外通常为正
- 保留符号，不额外压缩成单厚度

## 兼容接口推导

统一层的 XML 只保留 `StartOffset / EndOffset / Covered`。

旧的兼容接口现在按 signed offset 推导：

- `GetThinWallThickness()`
  - 单侧：返回单侧厚度
  - 双侧对称：返回总厚度，即 `abs(StartOffset) + abs(EndOffset)`
- `IsThinWallOneSided()`
  - 仅当一侧为零时返回 `true`
- `IsThinWallOutward()`
  - 仅对单侧薄壁有意义；`StartOffset≈0 且 EndOffset>0` 时返回 `true`

## 代码约束

1. Builder/Accessor/Serialization/Validation 统一围绕 `StartOffset/EndOffset/Covered`
2. 允许 `SetThinWall(...)` 作为兼容写法存在，但它只能映射到新字段，不能形成第二套 XML 定义
3. 读取侧和写入侧都应优先使用 `GetThinWallStartOffset()/GetThinWallEndOffset()`
4. 旧的 `GetThinWallThickness()/IsThinWallOneSided()/IsThinWallOutward()` 只能作为兼容推导接口，不能再作为统一模型定义
