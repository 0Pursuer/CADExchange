# 三系统倒圆角跨系统参数调研

## 1. 目的

本文用于整理 SolidWorks、Creo、UG/NX 三个系统中“倒圆角/圆角/round/fillet/blend”特征的：

- 支持的生成方式
- 可提取参数
- 创建所需参数
- 三系统之间的重要差异

目标是为后续在 `CADExchange\core\UnifiedFeatures.h` 中设计统一倒圆角存储结构提供依据。

## 3. 三系统支持的倒圆角生成方式

### 3.1 SolidWorks

当前证据确认 SW 实体倒圆角至少支持：

- `Simple`
  - 普通等半径边圆角
- `VariableRadius`
  - 变半径边圆角
- `Face`
  - 面圆角
- `FullRound`
  - 全圆角

本地 `swconst.tlh` 里的原生枚举也对应这四类：

- `swFeatureFilletType_Simple`
- `swFeatureFilletType_VariableRadius`
- `swFeatureFilletType_Face`
- `swFeatureFilletType_FullRound`

补充能力：

- simple/variable 圆角都支持非对称
- 支持不同 cross-section profile
  - `swFeatureFilletCircular`
  - `swFeatureFilletConicRho`
  - `swFeatureFilletConicRadius`
  - `swFeatureFilletConicRhoZeroChamfer`
- 支持 partial-edge fillet 数据接口

### 3.2 Creo

Creo 原生 round 比另外两家更重，既区分参考拓扑，也区分圆角 shape。

当前证据确认 Creo round 支持的 shape 至少包括：

- `PRO_ROUND_TYPE_CONSTANT`
  - 恒定半径
- `PRO_ROUND_TYPE_VARIABLE`
  - 变半径
- `PRO_ROUND_TYPE_FULL`
  - 全圆角
- `PRO_ROUND_TYPE_THROUGH_CURVE`
  - 过曲线圆角
- `PRO_ROUND_TYPE_CHORDAL`
  - 弦高圆角

当前证据确认 Creo round 支持的参考类型至少包括：

- `PRO_ROUND_REF_EDGE`
  - 边链圆角
- `PRO_ROUND_REF_SURF_SURF`
  - 面面圆角
- `PRO_ROUND_REF_EDGE_SURF`
  - 边面圆角
- `PRO_ROUND_REF_EDGE_EDGE`
  - 边边全圆角

补充能力：

- cross-section 还能区分 rolling ball / normal to spine
- conic/circular/C2-continuous 是正式参数，不是附属选项
- transition 体系比 SW/UG 都重

### 3.3 UG/NX

UG/NX 这次本地强证据落在 `Features::EdgeBlendBuilder`，不是 `ChamferBuilder` 那套。

当前证据确认 UG 实体倒圆角至少支持：

- 恒定半径 edge blend
  - `AddChainset(scCollector, radius)`
- 变半径 edge blend
  - `AddVariableRadiusDataNew(...)`
  - `AddVariablePointData(...)`
- conic edge blend
  - `AddChainset(..., sectionType, conicMethod, rhoType, radius, center, rho)`

补充能力：

- `Section`
  - `SectionCircular`
  - `SectionConic`
- `Conic`
  - `BoundaryPlusCenter`
  - `BoundaryPlusRho`
  - `CenterPlusRho`
- `FaceContinuity`
  - `Tangent`
  - `Curvature`
- 支持 setback、stop-short、overlap、trimming、blend order 等角点/过渡控制

当前没有拿到与 SW `FullRound` 或 Creo `PRO_ROUND_TYPE_FULL + surf/surf/surf` 同等强度的 UG 本地证据，因此这部分不能先假设为三系统公共能力。

## 4. 三系统可提取参数

### 4.1 SolidWorks

当前已确认可直接从 `ISimpleFilletFeatureData2` / `IVariableFilletFeatureData2` 提取：

- `filletType`

- `PropagateToTangentFaces`
- `KeepFeatures`

  - `swSmoothTransition`
  - `swStraightTransition`

  - `Default`

### 4.2 Creo

当前可确认 Creo round 能提取的核心字段包括：

- `shape`
  - constant / variable / full / through_curve / chordal
- `reference type`
  - edge / surf-surf / edge-surf / edge-edge
- conic 参数
  - `conic type`
  - `conic value`
  - `conic dep option`
- spine / cross-section 语义
  - `ball_spine`
  - spine curve collection
- `auto continue`
- 每个 radius item 的：

## 5. 三系统创建倒圆角所需参数

### 5.1 SolidWorks

最小创建参数集可归纳为：

- 通用：
  - `filletType`
  - 引用元素集合
- `Simple`
  - edge 集合
  - `radius1`
  - 若非对称还需要 `radius2`
- `VariableRadius`
  - edge 集合
  - 至少一个控制点
    - `position`
    - `radius1`
    - 若非对称还需要 `radius2`
- `Face`
  - set1 faces
  - set2 faces
  - `radius1`
  - 若非对称还需要 `radius2`
- `FullRound`
  - side1 faces
  - center faces
  - side2 faces

常见可选行为参数：

- `propagateTangent`
- `keepFeatures`
- `curvatureContinuous`
- `crossSectionProfile`
- `partialEdgeParameters`

### 5.2 Creo

最小创建参数集可归纳为：

- 通用：
  - `feature type = PRO_FEAT_ROUND`
  - 至少一个 `set`
  - `attachType`
- 每个 set：
  - `shape`
  - `referenceType`
  - 对应参考几何
  - `conicType`
  - 某些情况下还需要 `conicValue`
  - 某些情况下还需要 `conicDepOption`
  - `ball/spine`
  - `autoContinue`
- 非 full 类型通常还需要 `radii`
  - constant:
    - `leg1.value`
  - variable:
    - 每个点至少需要
      - point on edge
      - `leg1.value`
    - conic independent 时可能还需要 `leg2.value`
  - through_curve:
    - 还需要 through-curve 集合或 curve reference
  - full round:
    - 需要与 reference type 匹配的多面/多边引用，而不是普通半径列

可选但很重要的附加参数：

- `transition`
- `capping`
- `patch`
- `stop-at-ref`

### 5.3 UG/NX

最小创建参数集可归纳为：

- 通用：
  - edge chain collector
- 恒定半径：
  - `radius`
- conic chainset：
  - `sectionType = Conic`
  - `conicMethod`
  - `rhoType`
  - `radius`
  - `center`
  - `rho`
- 变半径：
  - target edge
  - 每个 variable point 至少需要
    - `parameter`
    - `radius`
    - `smartPoint`
    - `isExternal`
    - 可选 `isArclength`

常见可选控制：

- `tolerance`
- `allInstances`
- `removeSelfIntersection`
- `patchComplexGeometryAreas`
- `limitFailingAreas`
- `rollOverSmoothEdge`
- `rollOntoEdge`
- `moveSharpEdge`
- `trimmingOption`
- `overlapOption`
- `blendOrder`
- `setbackOption`
- `blendFaceContinuity`
- `setback data`
- `stop-short data`

## 6. 三系统之间的重要差异

### 6.1 参考拓扑差异比倒角更大

- SW
  - 主要围绕 edge、face set、full-round 三面组
- Creo
  - 明确区分 `edge / surf-surf / edge-surf / edge-edge`
- UG
  - 当前主 API 是 edge blend chainset 驱动

这意味着统一模型不能只存“references 数组”，还要考虑这些引用是怎么分组解释的。

### 6.2 “全圆角”并不是三系统同构能力

- SW
  - `FullRound` 是正式 fillet type
- Creo
  - `PRO_ROUND_TYPE_FULL` 也是正式 shape，且与 reference type 联动
- UG
  - 当前没有拿到同等强度的本地证据

因此 unified 层如果放 `FULL_ROUND` mode，需要允许某些系统无原生直映射。

### 6.3 变量半径表达方式不一样

- SW
  - `position + radius1 + radius2`
- Creo
  - 每个 radius item 是 `point collection + leg1 + leg2`
- UG
  - `parameter + radius + smartPoint + isExternal + isArclength`

这说明“变量圆角”不能只靠一个 `std::vector<double> radii` 统一。

### 6.4 圆角截面/剖面语义差异明显

- SW
  - circular / conic rho / conic radius / conic-rho-zero-chamfer
- Creo
  - circular / conic / C2 continuous，还能配 independent conic
- UG
  - circular / conic，加 `center/rho`

这部分比倒角的 `distance/angle` 差异大得多，统一时不能简单忽略。

### 6.5 Creo 和 UG 都有更重的 corner/transition 控制

- Creo
  - `transition + cap + sphere + patch + stop-at-ref`
- UG
  - `setback + stop-short + overlap + blend order`
- SW
  - 相对更偏 feature-data 式参数和局部 partial edge

这说明如果第一版 unified 只保留“圆角本体”，写回层以后仍然需要系统扩展位。

## 7. 对统一存储结构的直接建议

倒圆角不适合像当前倒角 v1 那样只做：

- `mode`
- 单组标量 `params`
- `references`

原因是变量圆角和全圆角会丢语义：

- 变量圆角至少需要“半径点列”
- 全圆角需要“多组引用集”
- conic/chordal/through-curve 不是单个 `radius` 能表达的

### 7.1 第一版建议保留的 unified 核心维度

建议第一版至少保留：

- `mode`
- `crossSection`
- `references`
- `radiusItems`

其中：

- `mode`
  - 描述圆角主类型
- `crossSection`
  - 描述圆截面 / conic / curvature-continuous 等
- `references`
  - 直接复用现有 `CRefEntityBase` 体系，但要允许分组
- `radiusItems`
  - 承接 constant / variable / asymmetric / full-round 之外的半径点列

### 7.2 建议的 mode 枚举

建议第一版先收敛为：

- `CONSTANT_RADIUS`
- `VARIABLE_RADIUS`
- `FULL_ROUND`
- `THROUGH_CURVE`
- `CHORDAL`

说明：

- SW `Simple` 和 UG 普通常半径 edge blend 可映射到 `CONSTANT_RADIUS`
- SW `VariableRadius`、Creo `PRO_ROUND_TYPE_VARIABLE`、UG variable radius 可映射到 `VARIABLE_RADIUS`
- SW/Creo 的 full round 统一映射到 `FULL_ROUND`
- Creo `PRO_ROUND_TYPE_THROUGH_CURVE` 单列
- Creo `PRO_ROUND_TYPE_CHORDAL` 单列

如果后续 UG 或 SW 找到 through-curve/chordal 对应能力，再往这些 mode 上补即可。

### 7.3 建议的 cross-section 枚举

建议不要把截面语义塞进 `mode`，而是单独保留：

- `CIRCULAR`
- `CONIC`
- `CURVATURE_CONTINUOUS`
- `UNKNOWN`

必要时可再配：

- `conicValue`
- `conicMode`
  - 比如 rho / radius / center-rho / boundary-center

### 7.4 建议的半径项结构

变量圆角建议至少有一层 item，而不是只留单个 `radius`：

- `parameter`
- `radius1`
- `radius2`
- `isAsymmetric`

如果后面要兼容 UG / Creo 的更重语义，可以再加：

- `pointRef`
- `edgeRef`
- `isExternal`
- `isArclength`
- `auxValue1`
- `auxValue2`

但这几个可以先不进第一版核心。

### 7.5 第一版 unified 结构草案

建议先考虑类似下面这种形状：

```cpp
enum class FilletMode {
  UNKNOWN = 0,
  CONSTANT_RADIUS,
  VARIABLE_RADIUS,
  FULL_ROUND,
  THROUGH_CURVE,
  CHORDAL
};

enum class FilletCrossSection {
  UNKNOWN = 0,
  CIRCULAR,
  CONIC,
  CURVATURE_CONTINUOUS
};

struct CFilletRadiusItem {
  std::optional<double> parameter;
  std::optional<double> radius1;
  std::optional<double> radius2;
};

struct CFilletParams {
  FilletCrossSection crossSection{FilletCrossSection::UNKNOWN};
  std::optional<double> radius;
  std::optional<double> conicValue;
  std::vector<CFilletRadiusItem> radiusItems;
};

struct CFillet : public CFeatureBase {
  FilletMode mode{FilletMode::UNKNOWN};
  CFilletParams params;
  std::vector<std::shared_ptr<CRefEntityBase>> references;
};
```

### 7.6 这里为什么比倒角多一层 `radiusItems`

倒角那次用户确认的 v1 可以极简成：

- `mode`
- `params(distance1/2/3/angle)`
- `references`

但倒圆角不行，主要是因为：

- SW 变量圆角有控制点列
- Creo 变量圆角和 chordal/through-curve 都依赖 set 内部复合参数
- UG 变量圆角 API 明确以 point data 为基本操作单元

所以如果 unified 里没有 `radiusItems`，后面 bridge 读出来只能退化成“拿第一半径”，这会直接丢主语义。

## 8. 当前阶段更务实的实现建议

如果后面准备像倒角一样分阶段落 `CADExchange`，建议顺序如下：

1. 先做 `CFillet` v1，只覆盖
   - `CONSTANT_RADIUS`
   - `VARIABLE_RADIUS`
   - `FULL_ROUND`
2. `params` 里先保留
   - `radius`
   - `crossSection`
   - `conicValue`
   - `radiusItems`
3. `references` 先直接复用现有引用体系
4. 暂不把 Creo transition / UG setback / SW partial-edge 进 unified 主结构
5. 这些差异先通过系统桥接层保留，等主模型稳定后再看是否需要扩展

这样做的原因是：

- 先覆盖三系统最常见的共同语义
- 不会像倒角那样过度精简到丢掉变量圆角
- 也不会第一版就把 Creo/UG 的复杂 transition 行为全部引进来，导致 unified 结构过重
