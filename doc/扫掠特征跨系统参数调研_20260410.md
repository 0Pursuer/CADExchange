# 扫掠特征跨系统参数调研_20260410

## 1. 目标

本次调研目标是为后续制定 Sweep 的 unified feature 定义做参数面梳理，重点回答：

1. `sw / creo / ug` 三个系统在“扫掠特征提取”和“扫掠特征重建”时，原生 API 到底需要哪些参数。
2. 哪些参数是三系统共有的，可以直接做 unified 字段。
3. 哪些参数只在单个系统中有明确原生语义，应作为条件字段或系统扩展字段处理。

本次调研对象默认聚焦：

- 实体扫掠 / 切除扫掠
- 单路径、单截面 sweep
- 不展开 sheet metal swept flange、rib-like 特化命令
- 不把 loft / boundary / tube routing 等相邻特征混进来

## 3. 三系统参数面总览

先给结论版。

| 参数簇 | SolidWorks | Creo | UG/NX |
| --- | --- | --- | --- |
| 特征类型（加料/减料） | 有 | 有 | 有 |
| 截面 profile | 有 | 有 | 有 |
| 路径 path/guide | 有 | 有 | 有 |
| 多导轨 guide curves | 有 | Toolkit 语义更偏 trajectory/frame 体系，非 SW 式 guide list | 该 builder 直接暴露 1 个 guide section |
| 圆形剖面快捷参数 | 有 | 未见等价快捷开关 | 未见等价快捷开关 |
| 扭转 twist | 有，显式 | 未见与 SW 对等的独立 twist 参数 | 在 `SweepAlongGuideBuilder` 里未见显式 twist 参数 |
| 路径对齐 alignment | 有，显式 | 有，但更偏 frame/orient 语义 | 未见 SW 式显式 path-alignment 参数 |
| 起止切线 tangency | 有，显式 | 未见同层参数 | 未见同层参数 |
| 薄壁 thin | 有，显式 | 有，显式 | 从该 demo/builder 未见 sweep 薄壁直接参数 |
| 起止偏置/左右偏置 | 没有 path start/end offset 这一层 | 主要是 frame / section，不是 NX 这种 offset | 有，`FirstOffset/SecondOffset` |
| 公差 tolerance | 不是该特征的主要输入面 | 不是该特征 element tree 主参数 | 有，显式 `DistanceTolerance/ChainingTolerance` |
| 特征作用域 / merge / multibody | 有 | 不突出 | Boolean + FeatureOptions |

因此：

- `SW` 最像“参数化 sweep UI 全量展开版”
- `Creo` 最像“轨迹 + frame 控制版”
- `UG SweepAlongGuideBuilder` 最像“section/guide/offset/boolean/tolerance 的瘦 builder”

## 4. SolidWorks 扫掠参数调查

## 4.1 提取侧可稳定读取的参数

根据
[`SwLibs/demo/SwLibs/src/SweepInfo.cpp`](E:/MyProject/SwLibs/demo/SwLibs/src/SweepInfo.cpp)
和
[`ISweepFeatureData_members.md`](E:/MyProject/SwLibs/apiDocs/SolidWorks.Interop.sldworks~SolidWorks.Interop.sldworks.ISweepFeatureData_members.md)，
SW 扫掠至少可稳定提取以下参数：

### A. 基础选择对象

- `Profile`
  - 草图剖面或 tool body
- `Path`
  - 扫掠路径
- `GuideCurves`
  - 多导轨数组
- `GetProfileType`
  - profile 对象类型
- `GetPathType`
  - path 对象类型
- `GetGuideCurvesCount`
  - 导轨数量
- `GetGuideCurvesType`
  - 每条导轨的对象类型

### B. 截面表达方式

- `CircularProfile`
  - 是否启用圆形剖面快捷模式
- `CircularProfileDiameter`
  - 圆形剖面直径

这说明 SW 的统一定义里，profile 不能只假定为“草图引用”，还要能表达“圆形截面快捷参数”。

### C. 路径对齐 / 方向 / 扭转

- `Direction`
  - 方向类型
- `PathAlignmentType`
  - 路径对齐类型
- `GetPathAlignmentDirectionVector`
  - 特定 path-alignment 方向向量
- `TwistControlType`
  - 扭转控制类型
- `GetTwistAngle`
  - 扭转角
- `D1ReverseTwistDir`
  - 方向 1 扭转反向
- `D2ReverseTwistDir`
  - 双向 sweep 第二方向扭转反向

这里是 SW 与另外两家差异最大的参数簇之一。

### D. 起点/终点切线控制

- `StartTangencyType`
- `EndTangencyType`
- 旧接口还可见 `StartDirectionVector` / `EndDirectionVector`
  - 但文档已标为 obsolete

### E. 几何质量 / 面合并策略

- `MaintainTangency`
- `AdvancedSmoothing`
- `AlignWithEndFaces`
- `MergeSmoothFaces`

### F. 薄壁

- `ThinFeature`
- `ThinWallType`
- `GetWallThickness`
- `SetWallThickness`

SW 薄壁 sweep 参数面比较完整，而且是 profile/path 之外的一级语义。

### G. 布尔 / 作用域 / 多实体

- `Merge`
- `FeatureScope`
- `FeatureScopeBodies`
- `AutoSelect`
- `AssemblyFeatureScope`
- `AutoSelectComponents`
- `PropagateFeatureToParts`

这说明 SW 侧“是否 merge 到目标体、影响哪些 body/component”是特征参数面的一部分。

## 5.4 Creo 原生约束里最关键的几点

根据
[`ProSweep.h`](E:/Application/Creo/install_path/CREO7/Creo%207.0.0.0/Common%20Files/protoolkit/includes/ProSweep.h)
和
[`prosweep_h.html`](E:/Application/Creo/install_path/CREO7/Creo%207.0.0.0/Common%20Files/protoolkit/protkdoc/api/prosweep_h.html)：

### A. `PRO_E_SWEEP_TYPE`

- `ProSweepType`
  - `PRO_SWEEP_TYPE_UNDEF`
  - `PRO_SWEEP_TYPE_MULTI_TRAJ = 128`

### B. `PRO_E_SWP_SEC_TYPE`

- `ProSweepSecType`
  - `PRO_SWEEP_VARIABLE_SECTION = 0`
  - `PRO_SWEEP_CONST_SECTION = (1<<23)`
- 但 Toolkit 文档明确写了：
  - 只有 `PRO_SWEEP_CONST_SECTION` 被 Creo Parametric TOOLKIT 支持

这意味着：

- Creo UI 语义里存在 variable section
- 但 Toolkit 创建侧不能按同样自由度直接走
- unified 定义如果想兼容未来 variable section，得把它设计成可选扩展，但不能要求当前 Creo bridge 完整支持

### C. `PRO_E_FRM_NORMAL`

可选值：

- `PRO_FRAME_NORM_ORIGIN`
- `PRO_FRAME_PIVOT_DIR`
- `PRO_FRAME_CONST_Z_DIR`

含义：

- 沿 origin trajectory 自带法向
- 由投影方向控制法向
- 由固定 Z 方向控制法向

### D. `PRO_E_FRM_ORIENT`

可选值：

- `PRO_FRAME_MIN`
  - Automatic
- `PRO_FRAME_NORM_SURF`
  - 法向依赖表面
- `PRO_FRAME_CONSTANT`
  - 仅与 `PRO_FRAME_PIVOT_DIR` 配合
- `PRO_FRAME_X_TRAJ`
  - 文档里存在，但 Toolkit 标注不支持

### E. `PRO_E_FRM_USER_X` / `PRO_E_FRM_START_X`

- 当 `PRO_E_FRM_ORIENT == PRO_FRAME_MIN` 时，允许控制起始 X 方向
- `PRO_E_FRM_USER_X`：
  - `PRO_FRAME_DEFAULT_START_X`
  - `PRO_FRAME_USER_START_X`
- 若为用户指定，则必须给 `PRO_E_FRM_START_X`

这在三系统里是很独特的一簇参数，建议 unified 定义显式保留，不要丢到模糊的“orientation options”字符串里。

## 5.5 Creo 结论

Creo sweep 最重要的不是 SW 那种 twist/tangency，而是：

- 轨迹 collection
- frame normal 模式
- frame orient 模式
- 是否用户指定起始 X 方向
- section type
- 薄壁材料侧和厚度

因此统一定义不能只按 “profile + path + guides + twist” 来想，否则会把 Creo 最关键的 frame 语义丢掉。

## 7. Unified 定义建议：参数分层

## 7.1 建议的第一层：跨系统必选核心字段

这层应尽量保证 `sw / creo / ug` 都能落地。

- `operation`
  - `add / cut / surface`
- `profile`
  - 截面定义
  - 支持草图引用
  - 支持简单快捷截面扩展，如 SW 圆形截面
- `path`
  - 主路径/引导路径
- `path_kind`
  - `single_path / section+guide`
  - 因为 UG 现成 builder 就是 `SweepAlongGuide`
- `is_thin`
- `material_side`
  - `side1 / side2 / both`
- `thickness`
  - 仅 `is_thin=true` 时有效
- `boolean_targets`
  - 多实体/布尔目标信息

## 7.2 建议的第二层：强建议保留的条件字段

这层不是三家都同构，但至少两家有明确语义，或者其中一家强依赖。

- `guide_curves[]`
  - SW 强依赖
- `offset_start`
  - UG `FirstOffset`
- `offset_end`
  - UG `SecondOffset`
- `distance_tolerance`
  - UG
- `chaining_tolerance`
  - UG
- `frame.normal_mode`
  - Creo
  - 值建议：
    - `origin`
    - `pivot_direction`
    - `constant_direction`
    - `surface_normal`
- `frame.orientation_mode`
  - Creo
  - 值建议：
    - `automatic`
    - `constant`

  - Creo
  - `default / user_defined`
- `frame.start_x_direction`
  - Creo
- `section_type`
  - `constant / variable`
  - 虽然当前 Creo Toolkit 只支持 `constant`，但 unified 定义建议保留这个语义位

## 8. 统一定义时的关键注意点

## 8.1 不要把 Sweep 简化成 “profile + path + twist”

这样会直接丢失：

- Creo 的 frame 语义
- UG 的 offset/tolerance 语义
- SW 的 tangency/alignment/thin 语义

## 8.2 profile 不能只支持 sketch_ref

因为 SW 明确支持：

- `CircularProfile`
- `CircularProfileDiameter`

建议 unified `profile` 做成 tagged union：

- `sketch`
- `circular`
- 未来可扩展 `tool_body`

## 8.3 path 不能只建成一条 curve ref

至少要支持：

- 主路径/guide
- guide curves 数组
- trajectory collection / section-based path

## 8.4 frame 必须单独建模

Creo sweep 的关键自由度是 frame，不是单纯 path 自然法向。
建议不要把这部分塞进模糊字符串或 CAD 原始枚举直透，而是做成结构化字段。

## 8.5 `section_type` 应保留，即便当前某桥只支持 constant

理由：

- Creo 原生枚举里有 `VARIABLE_SECTION`
- unified 语义层不能因为当前 Toolkit 限制而把未来能力永久删掉

## 9. 建议的 unified sweep 草案

下面是更接近后续建模代码的数据结构方向，不是最终版，只是本次调研后建议优先讨论的骨架。

```cpp
struct UnifiedSweepProfile {
  enum class Kind {
    SKETCH,
    CIRCULAR
  };

  Kind kind = Kind::SKETCH;
  std::string sketch_id;
  double circular_diameter = 0.0;
};

struct UnifiedSweepFrame {
  enum class NormalMode {
    UNSPECIFIED,
    ORIGIN,
    PIVOT_DIRECTION,
    CONSTANT_DIRECTION,
    SURFACE_NORMAL
  };

  enum class OrientationMode {
    UNSPECIFIED,
    AUTOMATIC,
    CONSTANT,
    SURFACE_NORMAL,
    X_TRAJECTORY
  };

  enum class StartXMode {
    UNSPECIFIED,
    DEFAULT,
    USER_DEFINED
  };

  NormalMode normal_mode = NormalMode::UNSPECIFIED;
  OrientationMode orientation_mode = OrientationMode::UNSPECIFIED;
  StartXMode start_x_mode = StartXMode::UNSPECIFIED;

  std::optional<CVector3D> normal_direction;
  std::optional<CVector3D> start_x_direction;
};

struct UnifiedSweep {
  FeatureOperation operation;

  UnifiedSweepProfile profile;
  std::string path_id;
  std::vector<std::string> guide_curve_ids;

  bool is_thin = false;
  MaterialSide material_side = MaterialSide::SIDE1;
  double thickness = 0.0;

  std::optional<double> offset_start;
  std::optional<double> offset_end;
  std::optional<double> distance_tolerance;
  std::optional<double> chaining_tolerance;

  enum class SectionType {
    CONSTANT,
    VARIABLE
  };
  SectionType section_type = SectionType::CONSTANT;

  UnifiedSweepFrame frame;

  SweepSystemExtensions system_ext;
};
```

## 10. 最终结论

### 10.1 三系统真正共有的 sweep 核心

- 操作类型
- 截面
- 路径/导线
- 薄壁开关
- 材料侧
- 厚度

### 10.2 统一定义里必须重点保留的差异语义

- SW：guide/twist/alignment/tangency
- Creo：frame normal / frame orient / start-x / section type
- UG：offset / tolerance / section-vs-guide builder 结构

### 10.3 对后续 unified feature 定义的直接建议

- 定义一个 `UnifiedSweep`，但不要追求“所有字段都必填”
- 用“三层参数模型”：
  - 核心公共字段
  - 条件公共字段
  - 系统扩展字段
- 首版实现优先覆盖：
  - constant-section solid add/cut
  - sketch profile
  - single path 或 section+guide
  - thin wall
  - Creo frame 基础模式
  - SW 基础 twist/alignment
  - UG offset/tolerance

如果后续要进入“正式制定 unified 定义”阶段，建议下一步直接做两件事：

1. 先冻结 `UnifiedSweep` 最小闭包字段集
2. 再逐系统做 `native -> unified -> native` 的映射表

这样不会在设计阶段把 SW 的丰富参数面硬套给 Creo/UG，也不会因为当前桥接实现不完整而把未来语义删掉。
