# 三系统拔模跨系统参数调研

## 1. 目的

本文用于整理 SolidWorks、Creo、UG/NX 三个系统中“拔模 / draft / taper”特征的：

- 支持的生成方式
- 可提取参数
- 创建所需参数
- 三系统之间的重要差异

目标是为后续在 `CADExchange\core\UnifiedFeatures.h` 中设计统一拔模存储结构提供依据。

## 3. 三系统支持的拔模生成方式

### 3.1 SolidWorks

当前证据确认 SW 独立 Draft feature 至少支持：

- `NeutralPlaneDraft`
  - 以中性面/neutral plane 为基准进行拔模
- `PartingLineDraft`
  - 以分型线/parting lines 为基准进行拔模
- `StepDraft`
  - 台阶拔模

本地 `swconst.tlh` 中有明确枚举：

- `swNeutralPlaneDraft`
- `swPartingLineDraft`
- `swStepDraft`

补充能力：

- Draft feature 直接支持被拔模面集合
- 支持 direction pull 参考
- 支持 face propagation
- step draft 还带 step type
- 还有 `AllowReducedAngle` 这类创建/求解相关选项

### 3.2 Creo

Creo 的普通 Draft 明显比 SW 重，而且是“一个 feature 家族 + 很多正交选项”。

当前证据确认 Creo 普通 Draft 支持的关键生成维度至少包括：

- `constant / variable`
  - 常角拔模
  - 变角拔模
- `split mode`
  - `split none`
  - `split neutral`
  - `split surface`
  - `split sketch`
- `hinge / neutral object type`
  - `no neutral`
  - `plane`
  - `curve`
  - `quilt`
- `dependence`
  - `independent`
  - `dependent`
  - `none`

也就是说，Creo 的“拔模方式”不是只分中性面拔模/分型线拔模，而是由以下因素共同决定：

- 拔模面集合
- 方向 reference + flip
- side1 / side2 的 neutral object type
- split 策略
- 常角还是变角
- 双侧是否独立

除了普通 `PRO_FEAT_DRAFT` 外，Creo 还单独存在：

- `PRO_FEAT_DRAFT_LINE`
- `PRO_FEAT_VPDD`
  - Variable Pull Direction Draft

其中 `VPDD` 不是普通 Draft 的一个小参数分支，而是单独 feature family，本地 `ProVPDD.h` 和 `CreateVPDD.cpp` 都证明它有独立的 element tree 和创建流程。

### 3.3 UG/NX

UG/NX 当前本地强证据显示至少存在两套拔模 builder：

- `Features::DraftBuilder`
  - 常规 draft / taper
- `Features::DraftBodyBuilder`
  - 面向 drafted body 的另一套能力

其中 `Features::DraftBuilder` 当前可确认支持的方式维度包括：

- method
  - `MethodIsocline`
  - `MethodTrueDraft`
- draft type
  - `TypeFace`
  - `TypeEdge`
  - `TypeTangent`
  - `TypePartingEdge`
- references method
  - `StationaryFace`
  - `PartingFace`
  - `StationaryAndPartingFace`
- angle model
  - face/edge set + 常角
  - variable angle
  - 双侧对 parting reference 的上下角

`Features::DraftBodyBuilder` 还能确认 UG 有另一类更偏“整体体拔模 / body draft”的能力，至少带：

- `TypesEdges`
- `TypesFaces`
- parting object
- stationary edges above/below parting
- faces to draft
- draft angle
- matching / repair / fillet radius / move-to-drafted-face 等体级选项

所以 UG 在“拔模”这件事上其实也不是单一模型。

## 4. 三系统可提取参数

### 4.1 SolidWorks

当前从 `IDraftFeatureData2` 可以直接确认的字段包括：

- `Type`
  - neutral plane / parting line / step
- `Angle`
- `ReverseDirection`
- `FacesToDraft`
- `PartingLines`
- `NeutralPlane`
- `DirectionPull`
- `FacePropagation`
- `DraftedEntities`
- `StepType`
- `AllowReducedAngle`

其中几个字段的语义要特别注意：

- `FacesToDraft`
  - 被拔模的目标面集合
- `PartingLines`
  - 仅 parting line draft 相关
- `NeutralPlane`
  - 仅 neutral plane draft 相关
- `DirectionPull`
  - pull / draft direction 参考
- `FacePropagation`
  - 本地 `swconst.tlh` 还能确认其枚举至少有：
    - `swFacePropNone`
    - `swFacePropTangent`
    - `swFacePropAllLoops`
    - `swFacePropInnerLoops`
    - `swFacePropOuterLoops`
- `StepType`
  - step draft 模式，本地枚举至少有：
    - `swDraftTaperedStep`
    - `swDraftPerpendicular`

从统一语义角度看，SW 至少已经证明：

- 单角度是核心参数
- “参考方式”不止一种
- 方向翻转是显式参数
- propagation 是正式参数，不是推导值

### 4.2 Creo

当前可确认 Creo 普通 Draft 可提取的核心字段包括：

- `DraftSurface`
  - 被拔模面集合
- `DraftDirection`
  - direction reference
  - `Dir_Flip`
- `Constant_or_Variable`
- `Split`
  - none / neutral / surface / sketch
- `Include_Tangent`
- `Tweak_or_Intersec`
- `Extend`
- `Exclude_Loop`

以及 side 级别字段：

- `Draft_Side1`
- `Draft_Side2`

每个 side 内部又包含：

- `Draft_Hinge_Type`
  - no neutral / plane / curve / quilt
- `Draft_Hinge`
- `Dependence`
  - independent / dependent / none
- `Draft_Angle_Constant`
- `Angles_Pnt`
  - 变角点列
  - 每项至少有：
    - `Point_Collection`
    - `Draft_Angle`
- `Hinge_quilt`

若 `Split == PRO_DRAFT_UI_SPLIT_SCTCH`，还会额外涉及：

- `Split_Sketch`
  - section plane
  - plane vectors / center
  - sketch section 数据

所以 Creo 普通 Draft 至少支持提取以下三类角度模型：

- 单侧常角
- 双侧常角
- 单侧/双侧变角点列

另外，Creo 还存在独立的 `VPDD` 特征。当前可确认 `VPDD` 可提取的字段包括：

- pull direction references + flip
- set 数组
- 每个 set 的 hinge 链
- split surface options + refs
- const / var
- point-and-angles 数组
- attachment option
- extent option
- extent length / extent reference

这说明 Creo 的“拔模”总能力边界比普通 Draft 要更大。

### 4.3 UG/NX

当前从 `Features::DraftBuilder` 可直接确认的字段包括：

- `Direction`
- `FaceSetAngleExpressionList`
  - face set + angle
- `EdgeSetAngleExpressionList`
  - edge set + angle
- `VariableAngleData`
- `DraftAllInstances`
- `DistanceTolerance`
- `AngleTolerance`
- `DraftIsoclineOrTruedraft`
  - isocline / true draft
- `TypeOfDraft`
  - face / edge / tangent / parting edge
- `StationaryReference`
- `StationaryPartingReference`
- `PartingReference`
- `DraftReferencesMethod`
- `DraftBothSides`
- `SymmetricAngle`
- `TwoDimensionFaceSetsData`
  - 上下两侧 face set + angle

从 `Features::DraftBodyBuilder` 还能确认体级拔模至少还有：

- `Type`
  - edges / faces
- `PartingObject`
- `UseDraftedBodyAsPartingObject`
- `DrawDirection`
- `StationaryEdgesAboveParting`
- `StationaryEdgesBelowParting`
- `FacesToDraft`
- `DraftAngle`
- `DraftMethod`
- `MatchScope`
- `MatchType`
- `UnmatchedEdges`
- `UnmatchedFaces`
- `RepairPartingEdge`
- `FilletRadius`
- `ExtremeFacePointOverridesStationary`
- `EdgesToMoveToDraftedFace`
- `FacesToMoveToDraftedFace`
- `Tolerance`

因此 UG 的拔模参数不能简单理解为“方向 + 角度 + 面集合”。

## 5. 三系统创建拔模时需要哪些参数

### 5.1 SolidWorks

按当前 `IDraftFeatureData2` 证据，SW 创建独立 Draft feature 时，最小必要参数大致取决于 `Type`：

#### 5.1.1 Neutral Plane Draft

至少需要：

- `Type = swNeutralPlaneDraft`
- `Angle`
- `FacesToDraft`
- `NeutralPlane`
- `DirectionPull`

常见附加参数：

- `ReverseDirection`
- `FacePropagation`
- `AllowReducedAngle`

#### 5.1.2 Parting Line Draft

至少需要：

- `Type = swPartingLineDraft`
- `Angle`
- `FacesToDraft`
- `PartingLines`
- `DirectionPull`

常见附加参数：

- `ReverseDirection`
- `FacePropagation`
- `AllowReducedAngle`

#### 5.1.3 Step Draft

至少需要：

- `Type = swStepDraft`
- `Angle`
- `FacesToDraft`
- `DirectionPull`
- `StepType`

常见附加参数：

- `ReverseDirection`
- `FacePropagation`
- `AllowReducedAngle`

注意：

- 仅从当前 `tlh` 强证据还不能反推出 SW UI 中每种草图/参考交互细节，因此这里列的是特征数据层最小集合，而不是 UI 操作步骤。

### 5.2 Creo

Creo 创建普通 Draft 时，最小必需参数明显比 SW 重。按 `CreateDraft.cpp` 和 `USR_Draft`，至少需要：

- `DraftSurface`
  - 被拔模面集合
- `DraftDirection`
  - direction reference
  - flip
- `Constant_or_Variable`
- `Split`
- `Include_Tangent`

以及 side 级别参数：

- `Draft_Side1.Draft_Hinge_Type`
- `Draft_Side1.Dependence`

若 side1 为独立常角：

- `Draft_Side1.Draft_Angle_Constant`

若 side1 为独立变角：

- `Draft_Side1.Angles_Pnt[]`
  - point + angle

若 `Split != NONE` 且存在双侧：

- `Draft_Side2.Dependence`
- 视情况再补：
  - `Draft_Side2.Draft_Angle_Constant`
  - `Draft_Side2.Angles_Pnt[]`

若 hinge type 是 plane / no neutral：

- `Draft_Side1.Draft_Hinge`

若 `Split == SPLIT_SCTCH`：

- `Split_Sketch.section_plane`
- 对应 section/sketch 数据

可选但常见的重要参数：

- `Tweak_or_Intersec`
- `Extend`
- `Exclude_Loop`

这意味着 Creo 普通 Draft 的创建参数至少有四层：

1. 全局面集合与方向
2. split 策略
3. side1 / side2 的 hinge + dependence
4. 常角或变角数据

若创建 `VPDD`，所需参数又是另一套模型，至少涉及：

- pull direction refs + flip
- sets[]
- hinge chains
- split surfaces option + refs
- constant / variable
- point-and-angles
- attach option
- extent option

### 5.3 UG/NX

按 `Features::DraftBuilder`，UG 常规 Draft 创建最小参数也依赖具体方式：

#### 5.3.1 通用必需

至少需要：

- `Direction`
- `DraftIsoclineOrTruedraft`
- `TypeOfDraft`

#### 5.3.2 若按 face/edge 常角方式

至少还需要其一：

- `FaceSetAngleExpressionList`
- `EdgeSetAngleExpressionList`

#### 5.3.3 若按 variable angle

至少还需要：

- `VariableAngleData`

#### 5.3.4 若涉及 stationary / parting 参考

至少还需要：

- `DraftReferencesMethod`
- `StationaryReference`
- `PartingReference` 或 `StationaryPartingReference`

#### 5.3.5 若是双侧 parting draft

至少还需要：

- `DraftBothSides`
- `SymmetricAngle`
- `TwoDimensionFaceSetsData`

常见附加参数：

- `DraftAllInstances`
- `DistanceTolerance`
- `AngleTolerance`

若走 `DraftBodyBuilder`，则是另一套更偏 body 级的创建参数：

- `Type`
- `PartingObject`
- `DrawDirection`
- `FacesToDraft` 或 stationary edges above/below
- `DraftAngle`
- `DraftMethod`

以及 match / repair / fillet radius / move-to-drafted-face / tolerance 等附加项。

## 6. 三系统重要差异

### 6.1 SW 最像“单 feature + 几种 reference mode”

SW 的独立 Draft feature 当前看起来最接近：

- 一个 feature
- 一个主角度
- 由 `Type` 决定参考方式

它的结构重点在：

- neutral plane / parting line / step 三种模式
- 方向翻转
- propagation

所以 SW 比较像“单层参数模型”。

### 6.2 Creo 的普通 Draft 已经是“双侧 + split + hinge + constant/variable”的组合模型

Creo 的普通 Draft 最大的特点不是模式多，而是参数层级深：

- 全局方向
- split 策略
- side1 / side2
- hinge type
- dependence
- constant / variable
- point-wise variable angles

如果 unified 结构只存：

- `angle`
- `references`

那在 Creo 这里会丢掉大量核心语义，尤其是：

- split 是怎么做的
- 双侧是不是独立
- 变角点列是挂在哪一侧
- neutral object 到底是 plane / curve / quilt

### 6.3 Creo 还单独有 VPDD，这不是普通 Draft 的一个小 mode

`VPDD` 的本地官方头文件和 demo 都说明：

- 它有独立 feature type
- 有自己的 element tree
- pull direction 可以是多 reference
- set 内可以有多 hinge
- 角度是“点 -> 多角度数组”
- 还有 attach / extent 体系

所以从 unified 角度：

- 不建议第一版把 `VPDD` 强塞进普通 Draft 的最小结构里
- 更合理的做法是先把它记为 Creo 特有扩展能力

### 6.4 UG 把“method/type/reference-method/body-draft”分得更细

UG 的重要差异是建模方法维度暴露得很直接：

- `isocline / true draft`
- `face / edge / tangent / parting edge`
- `stationary / parting / both`
- `both sides / symmetric`

另外还有单独的 `DraftBodyBuilder`。

所以 UG 的 unified 压缩重点不是“类型很多”，而是要避免把：

- 建模方法
- 参考组织方式
- 角度模型

混成一个枚举。

### 6.5 三系统的共同核心并不是“mode + angle”这么简单

三系统能真正求交的核心语义至少包括：

- 被拔模对象
- 方向
- 角度模型
- 参考方式

但从这次证据看，`参考方式` 本身已经至少分成：

- neutral plane
- parting line / parting reference
- stationary + parting
- edge / face set
- step

因此 unified 第一版如果只留一个 `angle` 和一组 `references`，大概率不够。

## 7. 对统一存储结构的建议

### 7.1 不建议第一版只做 `angle + references`

原因：

- SW 至少已经有 `Type`
- Creo 至少需要表达 split、双侧、常角/变角
- UG 至少需要表达 method、type、reference method

如果只留：

- `angle`
- `references`

那么：

- SW 的 step / neutral / parting 会混掉
- Creo 的 side1 / side2 / variable angles 会丢失
- UG 的 isocline / true draft 也会丢失

### 7.2 建议的 unified 最小结构

如果目标是类似当前 `UnifiedFeatures.h` 的“第一版能落地、但不过度复杂”的结构，我更建议拔模至少保留这几层：

#### 7.2.1 mode

用于表达拔模主模式，例如：

- `UNKNOWN`
- `NEUTRAL_PLANE`
- `PARTING`
- `STEP`
- `STATIONARY_AND_PARTING`
- `VARIABLE_ANGLE`

说明：

- 这里的 `mode` 不必完全等同某一家的原生枚举。
- 它的目标是区分 unified 写回时必须走不同逻辑的主路径。

#### 7.2.2 method

用于表达几何求解方法，建议至少保留：

- `NONE`
- `ISOCLINE`
- `TRUE_DRAFT`

说明：

- 这主要是为 UG 留的正式字段。
- SW 普通情况下可以映射到 `NONE`。
- Creo 普通 Draft 也可以先映射到 `NONE`。

#### 7.2.3 angle model

仅存一个角度不够，建议单独抽象角度组织方式：

- `CONSTANT_SINGLE`
- `CONSTANT_TWO_SIDES`
- `VARIABLE_POINTS`

配套参数建议至少有：

- `angle`
- `angleSide1`
- `angleSide2`
- `angleItems[]`

其中 `angleItems[]` 每项至少包含：

- `pointRef`
- `angle`
- `side`

这样才能承接：

- Creo variable draft
- UG variable angle draft

### 7.2.4 references

拔模的统一 references 不建议只是一坨匿名数组，最好至少按角色分组：

- `draftFaces`
- `neutralReference`
- `partingReferences`
- `directionReference`
- `stationaryReference`
- `splitReferences`
- `excludeLoops`

即使第一版内部实现仍复用统一 `CRefEntityBase`，语义层也最好分槽存放，否则后续写回很难稳定。

### 7.2.5 behavior / options

建议至少预留这些正式字段：

- `reverseDirection`
- `tangentPropagation`
- `bothSides`
- `symmetricAngle`

以及枚举型选项：

- `splitMode`
- `stepType`

### 7.2.6 system extensions

建议保留扩展位，但不要一开始就把所有系统特有参数都升级为统一主字段。

短期更适合先放扩展层的包括：

- Creo `VPDD`
- Creo `dependence`
- Creo `hinge type`
- Creo `tweak/intersect`
- Creo `extend`
- SW `allowReducedAngle`
- UG `draftAllInstances`
- UG `distanceTolerance / angleTolerance`
- UG `DraftBodyBuilder` 的 match / repair / fillet radius 系列

## 8. 一个更务实的第一版方向

如果后续你希望像倒角那次一样，先做一个比较克制的 unified 结构，我建议拔模第一版至少比倒角多保留这些字段：

- `mode`
- `method`
- `angleModel`
- `angle`
- `angleSide1`
- `angleSide2`
- `angleItems`
- `reverseDirection`
- `draftFaces`
- `directionReference`
- `neutralReference`
- `partingReferences`

可选增强：

- `stationaryReference`
- `splitMode`
- `bothSides`
- `symmetricAngle`
- `excludeLoops`

而下面这些可以先不进核心结构，先放系统扩展：

- Creo `VPDD`
- Creo `hinge type`
- Creo `dependence`
- UG `DraftBodyBuilder` 的 repair / match
- SW `allowReducedAngle`

## 9. 当前对 unified 设计的压缩结论

如果只看“普通拔模”的三系统公共交集，我认为最小统一语义至少要覆盖：

1. 拔模主模式
2. 方向参考
3. 被拔模对象
4. 常角还是变角
5. 是否需要 neutral / parting / stationary 这类参考

如果只看“完全最小”的第一版，我不建议比下面更轻：

- `mode`
- `method`
- `reverseDirection`
- `draftFaces`
- `directionReference`
- `neutralReference`
- `partingReferences`
- `angle`
- `angleItems`

其中：

- SW 大概率主要落在 `NEUTRAL_PLANE / PARTING / STEP`
- Creo 普通 Draft 会大量使用 `angleItems`、双侧和 split 相关扩展
- UG 会强依赖 `method` 和 stationary/parting 参考组织方式

## 10. 对后续 `UnifiedFeatures.h` 的建议

和倒角不同，拔模更像“参数层次化特征”，不太适合只做一个平铺参数包。

因此我建议后续若正式落 `CDraft`，优先考虑：

- 主结构表示普通 Draft 公共语义
- 先不把 `VPDD` 并进第一版 `CDraft`
- 如果后续确认 Creo `VPDD` 必须统一，再单独评估是否：
  - 作为 `CDraft` 的扩展模式
  - 或者单独建 `CVariablePullDirectionDraft`

当前更推荐的策略是：

- `普通拔模` 先统一
- `VPDD / DraftBody` 先记录为系统特有能力

这样不会把第一版 unified 结构做得过重，也不至于在普通 Draft 上丢掉三系统公共语义。
