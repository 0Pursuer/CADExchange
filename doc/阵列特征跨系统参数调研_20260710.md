# 阵列特征跨系统参数调研与统一设计 (SolidWorks, Creo, UG/NX)

本报告对主流 CAD 系统（SolidWorks, Creo, UG/NX）中的阵列特征（线性阵列、圆周阵列、镜像）进行深度调研，分析各系统的 API 字段、提取（Read）与重建（Write）的关键参数，并为 `CADExchange` 设计一套统一的阵列语义模型。

---

## 1. 阵列特征概述与跨系统设计对比

在 CAD 中，阵列（Pattern）是一种通过规律性变换（平移、旋转、镜像等）复制源对象（特征、面或实体）的工程操作。虽然三系统的概念一致，但在实现机制和数据模型上存在一些本质性差异：

### 1.1 核心设计差异

| 差异维度 | SolidWorks | Creo (Pro/ENGINEER) | UG/NX |
| :--- | :--- | :--- | :--- |
| **特征树结构** | 阵列是**独立特征**，包含对源特征/面/体的引用。 | 阵列是对**源特征的操作**。创建后，源特征会被包装在一个“阵列组”中，源特征作为第一实例。 | 阵列是**独立特征**，通过 `PatternFeature` 记录并引用源特征/面/体。 |
| **API 建模方式** | 通过专门的 FeatureData 接口（如 `ILinearPatternFeatureData`）。 | 通过 `ProElement` 树（`PRO_FEAT_PATTERN`）及元素标签进行读写。 | 通过 `PatternFeatureBuilder` 和布局类（`LayoutDefinition`）统一表达。 |
| **跳过实例 (Skip)** | 使用网格坐标系下的行列索引元组（如 `(row, col)`）定位并跳过。 | 使用阵列实例序列号或索引列表（`PRO_E_PATTERN_EXCL_LIST`）来排除。 | 在 `PatternFeatureBuilder` 中通过实例点或位置序号指定跳过。 |
| **几何阵列选项** | 支持 `GeometryPattern = True`（只复制几何体，不重新解算特征，可极大提升大型阵列效率）。 | 支持类似只复制几何的“只复制/只阵列”选项（`PRO_PATTERN_DIR_REG_VAL` 等）。 | 支持 Pattern Method 为 `Simple` 或 `Geometry`。 |

## 2. 线性阵列 (Linear Pattern)

线性阵列沿一个或两个线性方向复制源对象。

### 2.1 各系统 API 提取参数 (Read)

#### SolidWorks (`ILinearPatternFeatureData` / `ILinearPatternFeatureData2`)
- **方向 1 & 2 参考实体**：`GetDirection1` / `GetDirection2`（可返回线性边、轴线、基准面法向或尺寸线）。
- **方向 1 & 2 反向标记**：`D1ReverseDirection` / `D2ReverseDirection` (bool)。
- **方向 1 & 2 间距**：`D1Spacing` / `D2Spacing` (double，单位为**米**)。
- **方向 1 & 2 数量**：`D1Number` / `D2Number` (long)。
- **终止条件**：`D1EndCondition` (0 = 间距与数量，1 = 延伸到参考点/面)。
- **延伸到参考实体**：`D1Reference` (如果为 Up to Reference)。
- **延伸偏移量**：`D1Offset` (double) 和是否反向偏移 `D1ReverseOffset`。
- **仅阵列源**：`D2PatternSeedOnly` (bool，如果为 true，则方向 2 仅复制源特征，而不复制方向 1 生成的拷贝)。
- **跳过实例**：`SkippedItemArray` (包含跳过实例行列位置的坐标数组)。

#### Creo (Direction Pattern / `PRO_PATTERN_DIR_DIR`)
- **方向 1 & 2 参考**：`PRO_E_DIR_DIR_REF1` / `PRO_E_DIR_DIR_REF2` (引用实体，如边、轴、平面 normal)。
- **方向 1 & 2 数量**：`PRO_E_DIR_DIR_NUM1` / `PRO_E_DIR_DIR_NUM2` (int)。
- **方向 1 & 2 间距增量**：`PRO_E_DIR_DIR_INCR1` / `PRO_E_DIR_DIR_INCR2` (double，模型单位，可为负值表示反向)。
- **跳过实例列表**：`PRO_E_PATTERN_EXCL_LIST` (被抑制的实例索引列表)。

#### UG/NX (`PatternFeatureBuilder` + `LayoutDefinitionLinear`)
- **方向 1 & 2 矢量**：`Direction1` / `Direction2` (`Direction` 类型的指针/表达式)。
- **间距类型**：`SpacingType` (如 `CountAndPitch` 数量与间距，`CountAndSpan` 数量与总跨度)。
- **间距距离**：`D1Pitch` / `D2Pitch` (表达式，单位为模型单位)。
- **实例数量**：`D1Count` / `D2Count` (表达式)。
- **总跨度**：`D1Span` / `D2Span` (表达式)。
- **仅阵列源**：`Direction2` 下的 `PatternAlongDirection1` (bool)。
- **跳过实例**：`PatternService::GetSkippedInstances` 获取跳过的实例。

### 2.2 各系统创建所需参数 (Write)

- **SolidWorks**：使用 `IFeatureManager::InsertLinearPattern2` 或 `InsertLinearPattern3`，输入主副方向参考、间距、数量、仅阵列源标记、跳过列表、是否几何阵列等。
- **Creo**：构造特征元素树，特征类型为 `PRO_FEAT_PATTERN`，Pattern 类型为 `PRO_PATTERN_DIR_DIR`，填充 `PRO_E_DIR_DIR_REF1` 等元素，最后调用 `ProFeatureCreate`。
- **UG/NX**：创建 `PatternFeatureBuilder`，设置布局为 `Linear`，配置 `Direction1` / `Direction2`、间距及数量表达式，把源特征列表加入 `FeatureList`，最后 `Commit()`。

## 3. 圆周阵列 (Circular Pattern)

圆周阵列沿一旋转轴或圆柱表面旋转复制源对象。

### 3.1 各系统 API 提取参数 (Read)

#### SolidWorks (`ICircularPatternFeatureData`)
- **旋转轴参考**：`GetAxis` (轴线、线性边、圆柱面)。
- **旋转方向反转**：`ReverseDirection` (bool)。
- **间距角度**：`Spacing` (double，单位为**弧度**)。
- **实例数量**：`TotalInstances` (long)。
- **等间距分布**：`EqualSpacing` (bool)。
- **等间距总角度**：`TotalAngle` (double，单位为**弧度**)。
- **方向 2 (径向阵列)**：`Direction2` (bool)，包含 `D2Spacing` (径向间距)，`D2TotalInstances` (径向数量)。
- **仅阵列源**：`D2PatternSeedOnly` (bool)。
- **跳过实例**：`SkippedItemArray` (跳过的行列索引元组)。

#### Creo (Axis Pattern / `PRO_PATTERN_DIR_AXIS`)
- **中心轴参考**：`PRO_E_AXIS_AXIS_REF` (轴、圆柱面)。
- **角度方向实例数量**：`PRO_E_AXIS_NUM1` (int)。
- **间距增量角度**：`PRO_E_AXIS_INCR1` (double，单位为**度**)。
- **等间距选项**：`PRO_E_AXIS_VAL_OPT1` (设置是指定增量角度，还是根据总跨度自动均分)。
- **径向数量 & 增量**：`PRO_E_AXIS_NUM2` / `PRO_E_AXIS_INCR2` (用于径向同心圆环阵列)。
- **跳过列表**：`PRO_E_PATTERN_EXCL_LIST`。

#### UG/NX (`PatternFeatureBuilder` + `LayoutDefinitionCircular`)
- **旋转轴**：`RotationAxis` (点 + 方向或 Axis 实体)。
- **角度间距类型**：`SpacingType` (数量与间距、数量与角度跨度、均分等)。
- **间距角度**：`AngularPitch` (表达式，单位为**度**)。
- **角度实例数**：`AngularCount` (表达式)。
- **旋转总跨度**：`AngularSpan` (表达式)。
- **径向阵列数量 & 增量**：`RadialCount` / `RadialPitch` (表达式)。
- **跳过实例**：排除项的索引信息。

### 3.2 各系统创建所需参数 (Write)

- **SolidWorks**：使用 `IFeatureManager::InsertCircularPattern5`，设置轴参考、总实例、间距角度（或等间距总角）、等间距标置、源对象、排除列表等。
- **Creo**：组装 `PRO_FEAT_PATTERN` 元素树，Pattern 类型设为 `PRO_PATTERN_DIR_AXIS`，填入轴参考 `PRO_E_AXIS_AXIS_REF`、实例数与角度增量等。
- **UG/NX**：设置布局为 `Circular`，配置旋转轴、角度增量/数量表达式、径向参数（可选）并提交。

## 4. 镜像特征 (Mirror Pattern)

镜像特征通过将源对象关于某个对称平面反射生成拷贝。

### 4.1 各系统 API 提取参数 (Read)

#### SolidWorks (`IMirrorPatternFeatureData` / `IMirrorSolidFeatureData`)
- **镜像平面参考**：`GetMirrorPlane` (基准面、实体平面)。
- **镜像类型**：`MirrorBodies` (bool，为 true 时表示对整个实体镜像；为 false 时对特征或面镜像)。
- **源特征列表**：`PatternFeatureArray`。
- **源面列表**：`PatternFaceArray`。
- **源实体列表**：`PatternBodyArray`。

#### Creo (`PRO_FEAT_MIRROR`)
- **镜像面参考**：`PRO_E_MIRROR_PLANE` (基准面、实体平面)。
- **源特征列表**：`PRO_E_MIRROR_FEAT_LIST` (引用的特征句柄列表)。
- **源实体列表**：`PRO_E_MIRROR_BODY_LIST` (对实体镜像时的实体列表)。

#### UG/NX (`MirrorFeatureBuilder` / `MirrorBodyBuilder`)
- **镜像平面**：`Plane` (基准面、实体平面)。
- **源对象列表**：`FeatureList` (特征镜像) 或 `MirrorBody` (实体镜像)。

## 5. CADExchange 统一阵列模型方案设计

为了在 `CADExchange` 顶层统一表达这些阵列特征，我们可以在 `CADExchange/core/UnifiedFeatures.h` 中为线性阵列、圆周阵列和镜像定义一套清晰的数据结构。

### 5.1 核心枚举与公共结构

```cpp
namespace CADExchange {

/**
 * @brief 阵列作用域：特征、拓扑面、还是整个实体
 */
enum class PatternScope {
  FEATURES, ///< 阵列某些指定特征（如拉伸、切除）
  FACES,    ///< 阵列指定的拓扑表面
  BODIES    ///< 阵列实体/多实体
};

/**
 * @brief 阵列间距/分布模式
 */
enum class PatternSpacingType {
  PITCH_AND_COUNT, // 给定 数量(Count) + 间距(Pitch)
  SPAN_AND_COUNT,  // 给定 数量(Count) + 总跨度(Span)
};

/**
 * @brief 网格坐标，用于精确标记需要跳过（抑制）的阵列实例
 */
struct CPatternIndex {
  int dir1Index = 0; // 方向 1 索引（从0开始）
  int dir2Index = 0; // 方向 2 索引（如果不启用方向2，则为0）

  bool operator==(const CPatternIndex& other) const {
    return dir1Index == other.dir1Index && dir2Index == other.dir2Index;
  }
};

} // namespace CADExchange
```

### 5.2 线性阵列模型 (`CLinearPattern`)

```cpp
namespace CADExchange {

/**
 * @brief 线性阵列的方向控制参数
 */
struct CLinearPatternDir {
  std::shared_ptr<CRefEntityBase> directionRef; ///< 方向参考（边、轴、面法向等）
  bool reverse = false;                         ///< 是否反转方向
  
  PatternSpacingType spacingType = PatternSpacingType::PITCH_AND_COUNT;
  double spacing = 0.0;                         ///< 线性间距值（按模型单位，SW提取时需从米转为模型单位）
  int count = 1;                                ///< 实例数量
  
  // 支持 Up to Reference 的高阶属性（可选扩展）
  bool upToReference = false;
  std::shared_ptr<CRefEntityBase> referenceEntity;
  double offset = 0.0;
};

/**
 * @brief 统一线性阵列特征
 */
struct CLinearPattern : public CFeatureBase {
  CLinearPatternDir dir1;
  std::optional<CLinearPatternDir> dir2;
  
  bool patternSeedOnly = false; ///< 是否在方向2中仅对“源特征”进行阵列（而不阵列方向1产生的拷贝）
  
  // 阵列的源对象与范围
  PatternScope scope = PatternScope::FEATURES;
  std::vector<std::shared_ptr<CRefEntityBase>> seedObjects; ///< 特征、面、或实体的引用列表
  
  std::vector<CPatternIndex> skippedInstances; ///< 跳过的阵列实例索引列表
  bool geometryPattern = false;                ///< 是否启用几何阵列（加速复制）

  CLinearPattern() { featureType = FeatureType::Unknown; /* 之后增加 FeatureType::LinearPattern */ }
};

} // namespace CADExchange
```

};

## 6. 参数归一化与单位转换映射规则

为了确保在 C++ 桥接层读写数据时的绝对一致，在具体实现时必须遵循以下参数转换原则：

1. **长度单位规整**：
   - **SolidWorks API** 始终提取输出 **米 (m)**。
   - **Creo / UG** 提取的是**模型单位**（通常是毫米 mm 或英寸 inch）。
   - **映射逻辑**：在桥接到 `CADExchange` 时，使用 `UnitConverter` 或相应因子进行尺度缩放，统一归一化为**模型当前的主单位**（通常为 mm），写入时也作相同逆变换。
2. **角度单位规整**：
   - **SolidWorks API** 读写圆周阵列时使用**弧度 (rad)**。
   - **Creo / UG** 使用**度 (deg)**（其中 UG 中为数值或表达式字符串）。
   - **映射逻辑**：`CADExchange` 模型内圆周阵列的角度统一强制采用**弧度制 (radians)**。在 Creo 和 UG 的 Read 侧转换为弧度，Write 侧转换回度。
3. **坐标索引统一**：
   - 各种三维软件的排除（Skip）索引机制有异，但本质上都是由行列定位的格网。
   - 统一使用 `(dir1Index, dir2Index)` 二维索引表达。如果是单向阵列，`dir2Index` 默认为 0。在桥接层进行相应的本地映射转换。
