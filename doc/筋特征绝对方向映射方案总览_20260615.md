# 筋特征绝对方向映射方案总览_20260615

## 1. 文档目的

本文系统整理当前仓库中筋特征（Rib）的统一建模方案、SolidWorks/Creo 两端的方向映射算法、关键数学公式、代码落点以及关键提交脉络。

本文重点回答以下问题：

1. 当前筋特征方案到底在统一什么语义
2. 为什么不能继续使用 `SideA/SideB` 一类相对侧别作为最终统一语义
3. `SW -> CADExchange -> Creo` 与 `Creo -> CADExchange -> SW` 的读写映射具体如何计算
4. 关键点积、叉积、投影公式是什么
5. 当前设计文档口径与实际代码口径分别是什么

## 2. 方案总览

当前筋特征方案的核心，不是“把 SW 和 Creo 的原生字段一一抄过来”，而是：

- 先在 `CADExchange` 中把筋特征表达为**绝对三维方向语义**
- 再在 SW / Creo 两端分别把本地 API 的布尔值、枚举值、侧别值投影成统一方向
- 写回目标 CAD 时，再把统一方向反投影成目标 CAD 能理解的原生参数

也就是说，统一链路是：

- `原生 CAD 参数 -> 草图局部正交基底 -> 绝对三维向量`
- `绝对三维向量 -> 目标 CAD 草图局部正交基底 -> 目标 CAD 原生参数`

这套方案的根因是：

- SolidWorks 和 Creo 对筋特征的“单侧厚度”和“材料填充方向”并不是同一套命名语义
- 同一个 `Side1/Side2`、`FlipSide`、`ReverseThicknessDir`，在不同 CAD 中依赖不同局部坐标解释
- 一旦跨 CAD 重建，草图法向、局部 X/Y/Z 以及平面翻转规则变化，单靠“相对侧别”无法稳定恢复实体方向

因此，当前最稳定的统一语义是：

- 厚度值 `thickness`
- 厚度偏置方向向量 `thicknessOption.direction`
- 材料填充方向向量 `materialOption.direction`

## 4. 统一几何基底定义

无论 SW 还是 Creo，当前方案都先建立筋草图的局部正交基底。

对任意筋草图，定义：

- 草图法向：\(\vec{N}\)
- 草图切向：\(\vec{T}\)
- 面内参考向量：\(\vec{V}_{ref}\)

其中：

\[
\vec{V}_{ref} = \vec{N} \times \vec{T}
\]

这三个向量共同构成后续全部方向判定的基础。

## 4.1 法向 \(\vec{N}\)

\(\vec{N}\) 定义为草图局部坐标系的 `zDir`，归一化后使用：

\[
\vec{N} = \operatorname{normalize}(zDir)
\]

## 4.2 切向 \(\vec{T}\)

\(\vec{T}\) 的选取规则为：

1. 优先选筋草图中第一条非构造直线段的方向
2. 若没有直线段，则退化采用草图局部 `xDir`

若线段在草图二维坐标中的端点为 \((x_1,y_1)\)、\((x_2,y_2)\)，草图局部 X/Y 轴分别为 \(\vec{X}\)、\(\vec{Y}\)，则：

\[
dx = x_2 - x_1,\quad dy = y_2 - y_1
\]

\[
\vec{T}_{raw} = dx\vec{X} + dy\vec{Y}
\]

\[
\vec{T} = \operatorname{normalize}(\vec{T}_{raw})
\]

## 4.3 面内参考向量 \(\vec{V}_{ref}\)

\[
\vec{V}_{ref} = \operatorname{normalize}(\vec{N} \times \vec{T})
\]

这一步的几何意义是：

- \(\vec{T}\) 定义筋草图主线方向
- \(\vec{N}\) 定义草图法向
- \(\vec{V}_{ref}\) 则是草图平面内、垂直于主线方向的“侧向参考方向”

## 5. 为什么必须改成绝对方向

早期相对侧别方案的问题在于：

- `OneSideA / OneSideB` 只能表达“某侧”
- 但并没有定义这个“某侧”在三维空间中到底对应哪一个向量

在单一 CAD 内部，`A/B` 之所以看起来够用，是因为：

- CAD 自己隐含保存了草图局部法向
- CAD 自己隐含保存了 section 坐标系
- CAD 自己知道 `reverse` 或 `flip` 是相对哪条局部轴

一旦跨 CAD：

- 草图局部平面可能旋转
- datum plane 可能因错误逻辑发生法向翻转
- 目标 CAD 内部重建出的 section 坐标系未必与源草图名义坐标完全一致

于是“相对侧别”在跨系统时就会丢失语义锚点。

绝对方向方案的本质是：

- 不直接记“左/右”
- 直接记“朝哪个三维方向”

## 6.1 SW 读侧：原生参数 -> 绝对方向

SW 提取时关键原生参数是：

- `Thickness`
- `IsTwoSided`
- `ReverseThicknessDir`
- `FlipSide`
- `ExtrusionDirection`

当前代码用 `ExtrusionDirection == swRibNormalToSketch` 判定：

- `isNormToSketch = true`
- 否则视为平行于草图面

### 6.1.1 厚度偏置方向

若筋是非对称厚度：

#### 情况 A：`isNormToSketch = true`

此时厚度单侧偏置落在草图平面内，对应 \(\pm \vec{V}_{ref}\)

\[
\vec{v}_{thick} =
\begin{cases}
-\vec{V}_{ref}, & reverseThicknessDir = true \\
\vec{V}_{ref}, & reverseThicknessDir = false
\end{cases}
\]

#### 情况 B：`isNormToSketch = false`

此时厚度单侧偏置落在法向上，对应 \(\pm \vec{N}\)

\[
\vec{v}_{thick} =
\begin{cases}
-\vec{N}, & reverseThicknessDir = true \\
\vec{N}, & reverseThicknessDir = false
\end{cases}
\]

### 6.1.2 材料填充方向

#### 情况 A：`isNormToSketch = true`

材料填充方向沿法向：

\[
\vec{v}_{mat} =
\begin{cases}
\vec{N}, & reverseMaterialDir = true \\
-\vec{N}, & reverseMaterialDir = false
\end{cases}
\]

#### 情况 B：`isNormToSketch = false`

材料填充方向沿面内参考方向：

\[
\vec{v}_{mat} =
\begin{cases}
-\vec{V}_{ref}, & reverseMaterialDir = true \\
\vec{V}_{ref}, & reverseMaterialDir = false
\end{cases}
\]

## 6.2 SW 写侧：绝对方向 -> 原生参数

写回 SW 时，输入是：

- `thicknessOption.direction`
- `materialOption.direction`

需要反推出：

- `isNormToSketch`
- `reverseMaterialDir`
- `reverseThicknessDir`

### 6.2.1 判定 `isNormToSketch`

先把材料方向归一化：

\[
\hat{v}_{mat} = \operatorname{normalize}(\vec{v}_{mat})
\]

计算与法向的点积：

\[
d_{matN} = \hat{v}_{mat} \cdot \vec{N}
\]

当前实现使用阈值：

\[
|d_{matN}| > 0.5 \Rightarrow isNormToSketch = true
\]

否则：

\[
|d_{matN}| \le 0.5 \Rightarrow isNormToSketch = false
\]

几何意义是：

- 若材料方向更接近法向，则该筋按 `NormalToSketch` 理解
- 若材料方向更接近平面内方向，则按 `ParallelToSketch` 理解

### 6.2.2 判定 `reverseMaterialDir`

#### 当 `isNormToSketch = true`

\[
reverseMaterialDir =
\begin{cases}
true, & \vec{v}_{mat} \cdot \vec{N} > 0 \\
false, & \text{otherwise}
\end{cases}
\]

#### 当 `isNormToSketch = false`

\[
reverseMaterialDir =
\begin{cases}
true, & \vec{v}_{mat} \cdot \vec{V}_{ref} < 0 \\
false, & \text{otherwise}
\end{cases}
\]

### 6.2.3 判定 `reverseThicknessDir`

先令：

\[
\hat{v}_{thick} = \operatorname{normalize}(\vec{v}_{thick})
\]

#### 当 `isNormToSketch = true`

\[
reverseThicknessDir =
\begin{cases}
true, & \hat{v}_{thick} \cdot \vec{V}_{ref} < 0 \\
false, & \text{otherwise}
\end{cases}
\]

#### 当 `isNormToSketch = false`

\[
reverseThicknessDir =
\begin{cases}
true, & \hat{v}_{thick} \cdot \vec{N} < 0 \\
false, & \text{otherwise}
\end{cases}
\]

参考文档：

- [Creo筋特征PRO_RIB参数绝对方向映射记录_20260603.md](D:/CADHHU/CADProj/CreoLibs/doc/Creo筋特征PRO_RIB参数绝对方向映射记录_20260603.md)

Creo 侧的关键原生参数是：

- `PRO_E_RIB_SIDE_OPTS`
- `PRO_E_STD_MATRLSIDE`

## 7.1 Creo 读侧：原生参数 -> 绝对方向

### 7.1.1 厚度偏置方向

当前确认的最关键结论是：

- `PRO_RIB_SIDE_ONE` 对应 \(-\vec{N}\)
- `PRO_RIB_SIDE_TWO` 对应 \(+\vec{N}\)

因此：

\[
\vec{v}_{thick} =
\begin{cases}
-\vec{N}, & PRO\_RIB\_SIDE\_ONE \\
\vec{N}, & PRO\_RIB\_SIDE\_TWO
\end{cases}
\]

对称厚度则直接记：

- `symmetric = true`
- 不强制要求 `direction`

### 7.1.2 材料填充方向

当前代码采用：

\[
\vec{v}_{mat} =
\begin{cases}
\vec{V}_{ref}, & PRO\_RIB\_SEC\_SIDE\_ONE \\
-\vec{V}_{ref}, & PRO\_RIB\_SEC\_SIDE\_TWO
\end{cases}
\]

这与 SW 侧保持了统一：材料方向最终都映射成“某个绝对三维向量”。

## 7.2 Creo 写侧：绝对方向 -> 原生参数

### 7.2.1 判定 `PRO_E_RIB_SIDE_OPTS`

若是非对称厚度，令：

\[
\hat{v}_{thick} = \operatorname{normalize}(\vec{v}_{thick})
\]

计算点积：

\[
d_{thick} = \hat{v}_{thick} \cdot \vec{N}
\]

当前规则：

\[
d_{thick} < 0 \Rightarrow PRO\_RIB\_SIDE\_ONE
\]

\[
d_{thick} \ge 0 \Rightarrow PRO\_RIB\_SIDE\_TWO
\]

若对称：

\[
PRO\_E\_RIB\_SIDE\_OPTS = PRO\_RIB\_SYMMETRIC
\]

### 7.2.2 判定 `PRO_E_STD_MATRLSIDE`

先归一化：

\[
\hat{v}_{mat} = \operatorname{normalize}(\vec{v}_{mat})
\]

计算与参考方向的点积：

\[
d_{mat} = \hat{v}_{mat} \cdot \vec{V}_{ref}
\]

当前规则：

\[
d_{mat} < 0 \Rightarrow PRO\_RIB\_SEC\_SIDE\_TWO
\]

\[
d_{mat} \ge 0 \Rightarrow PRO\_RIB\_SEC\_SIDE\_ONE
\]

## 8. Creo 写侧为什么必须读取真实 section 坐标系

这是当前方案里非常关键的一点。

如果只使用统一模型里源草图的 `sketchCSys`，跨 CAD 后会有两个风险：

1. 目标 CAD 实际创建成功后的 sketch 坐标系可能与源模型名义方向存在微小偏差
2. Datum plane、旋转、局部平面约束变化可能导致目标 CAD 内部 section 的真正法向与源法向不完全一致

因此当前 Creo 写侧引入 `GetSectionCSys`，优先读取目标 Creo 中**实际创建成功后的 sketch 局部坐标系**。

当前处理流程是：

1. 从 Creo section 读出真实 `localZ`
2. 将其作为新的 \(\vec{N}\)
3. 统一模型中的切向 \(\vec{T}\) 投影到垂直于 \(\vec{N}\) 的平面上
4. 再重新构造 \(\vec{V}_{ref}\)

投影公式如下：

若 \(\vec{T}\) 是统一模型中的世界切向，则：

\[
\vec{T}' = \vec{T} - (\vec{T} \cdot \vec{N})\vec{N}
\]

\[
\vec{T}_{proj} = \operatorname{normalize}(\vec{T}')
\]

然后：

\[
\vec{V}_{ref} = \operatorname{normalize}(\vec{N} \times \vec{T}_{proj})
\]

这一步的作用是确保：

- 新的 \(\vec{T}_{proj}\) 与 \(\vec{N}\) 严格正交
- 即使 section 重建后发生轻微偏转，也能保持方向解释稳定

## 9. Datum plane 翻转问题为什么会直接破坏 Rib

相关代码在：

- [CreoLibs/read_write/src/internal/feature_datum_plane.cpp](D:/CADHHU/CADProj/CreoLibs/read_write/src/internal/feature_datum_plane.cpp)

历史错误做法是：

- 当 datum plane 的 offset 约束 `reversed = true` 时
- 直接设置 `PRO_E_DTMPLN_FLIP_DIR`

这会导致：

\[
\vec{N} \rightarrow -\vec{N}
\]

也就是整个平面法向翻转。

对筋特征而言，这不是“单纯偏置方向变了”，而是：

- 整个草图的局部 Z 轴变了
- 依附于该 plane 的 sketch 和 rib 坐标系被整体翻转
- 所有基于 \(\vec{N}\) 的 side 判断一起失效

当前修复策略是：

- 不翻 plane 法向
- 只翻 offset 值本身的符号

也就是：

\[
offset' =
\begin{cases}
-offset, & reversed = true \\
offset, & reversed = false
\end{cases}
\]

这保留了 datum plane 局部坐标系方向，仅改变偏置距离方向。

## 13. 当前方案的稳定结论

当前可以明确认为已经稳定的结论有：

1. Rib 的统一核心不应再依赖相对侧别，而应依赖绝对方向向量
2. 统一几何基底必须显式构造 \(\vec{N}\)、\(\vec{T}\)、\(\vec{V}_{ref}\)
3. SW 侧读写都必须通过点积判定把布尔翻转量映射到绝对方向
4. Creo 侧必须接受一个反直觉事实：
   - `PRO_RIB_SIDE_ONE = -N`
   - `PRO_RIB_SIDE_TWO = +N`
5. Datum plane 不能因为 offset 反向就整体翻法向
6. Creo 写侧必须优先使用目标 CAD 实际创建成功的 section 坐标系，而不是只信源模型名义方向

- SW 扩展块

- 桥接算法已比较稳定
- 统一结构本身还可能继续演进

### 14.2 SW 特有高级语义尚未完全提升为统一公共语义

例如：

- `Linear / Natural`
- `ParallelToSketch / NormalToSketch`
- `Draft`
- `ReferenceEdgeIndex`
- `RefSketchIndex`

这些在当前方案中更像 SW 扩展信息，而不是公共主干。

---

## 15. 结论

当前仓库中的筋特征方案，实质上已经从“相对侧别映射”演进为“绝对三维方向映射”。

它的统一思想可以总结为：

1. 用草图局部坐标系构造稳定的几何基底
2. 将 SW/Creo 的原生厚度侧与材料侧解释为绝对三维方向
3. 将这些绝对方向保存在 `CADExchange`
4. 写回目标 CAD 时，再通过点积和投影恢复目标侧别枚举和翻转布尔值

这套方案真正解决的不是“字段名翻译”，而是：

- 跨 CAD 局部坐标系变化后，如何仍然保持 Rib 的建模意图不变
- 在真实三维空间中，如何稳定定义“厚度在哪一侧、材料往哪边长”

因此，当前 Rib 方案的本质是一套**基于局部正交基底的方向语义归一化方案**。
