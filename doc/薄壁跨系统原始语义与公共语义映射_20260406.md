# 薄壁跨系统原始语义与公共语义映射_20260406

## 1. 目标

这份文档记录三套 CAD 原生薄壁参数的真实语义，以及它们应该如何映射到
`CADExchange` 的统一公共语义。

本次统一后的公共语义只有一套：

- `StartOffset`
- `EndOffset`
- `Covered`

其中：

- `StartOffset`：相对草图轮廓的内侧偏置，向内为负
- `EndOffset`：相对草图轮廓的外侧偏置，向外为正
- `Covered`：是否封口

这不是“每侧厚度幅值”语义，而是“signed offset”语义。

---

## 2. 公共语义定义

统一层按草图轮廓局部法向来理解薄壁材料分布：

- 单侧向内厚度 `10`
  - `StartOffset=-10`
  - `EndOffset=0`
- 单侧向外厚度 `10`
  - `StartOffset=0`
  - `EndOffset=10`
- 对称总厚 `20`
  - `StartOffset=-10`
  - `EndOffset=10`
- 双侧不等厚
  - 例如内侧 `6`、外侧 `9`
  - `StartOffset=-6`
  - `EndOffset=9`

兼容接口只允许从这套 signed offset 语义推导，不允许再形成第二套 XML 定义。

---

## 3. SW 原始语义

`SolidWorks` 原生更接近“厚度 + 方向枚举”，不是 signed offset。

原始 API 关键信息：

- `swThinWallOneDirection`
  - 单侧向外
- `swThinWallOppDirection`
  - 单侧向内
- `swThinWallMidPlane`
  - 中面对称
- `swThinWallTwoDirection`
  - 双向独立厚度

读取时原始值来源：

- `GetWallThickness(TRUE)`
  - 正向/外侧厚度
- `GetWallThickness(FALSE)`
  - 反向/内侧厚度

映射到公共语义：

- 向外单侧 `t`
  - `StartOffset=0`
  - `EndOffset=t`
- 向内单侧 `t`
  - `StartOffset=-t`
  - `EndOffset=0`
- 中面对称总厚 `t`
  - `StartOffset=-t/2`
  - `EndOffset=t/2`
- 双向不等厚
  - `StartOffset=-t_in`
  - `EndOffset=t_out`

写回 SW 时再反推成：

- `0 / +t` -> `OneDirection`
- `-t / 0` -> `OppDirection`
- `-t/2 / +t/2` -> `MidPlane`
- `-t_in / +t_out` -> `TwoDirection`

---

## 4. Creo 原始语义

`Creo` 原生更接近“单厚度 + material side”，也不是 signed offset。

原始 API 关键信息：

- `PRO_E_THICKNESS`
  - 薄壁厚度
- `PRO_E_STD_MATRLSIDE`
  - 材料侧
- 单侧默认规则：
  - `SIDE_ONE` 更接近向内
  - `SIDE_TWO` 更接近向外
- 双侧：
  - `BOTH_SIDES`

这次验证中确认的一个关键点是：

- `Creo` 对称薄壁的 `PRO_E_THICKNESS`
  - 对应的是总厚语义
  - 例如总厚 `20 in`
  - 应映射成 `StartOffset=-10, EndOffset=10`
  - 不能再落成 `20 / 20`

映射到公共语义：

- `SIDE_ONE + thickness=t`
  - `StartOffset=-t`
  - `EndOffset=0`
- `SIDE_TWO + thickness=t`
  - `StartOffset=0`
  - `EndOffset=t`
- `BOTH_SIDES + thickness=t`
  - `StartOffset=-t/2`
  - `EndOffset=t/2`

注意：

- `Creo` native 并不天然支持“内外两侧独立不等厚”
- 因此对于 `StartOffset` 与 `EndOffset` 同时非零且不对称的情况
  - 写回只能 `best-effort`
  - 不能宣称完全无损

---

## 5. UG / NX 原始语义

`UG/NX` 原生最接近公共语义。

原始 API 关键信息：

- `FeatureOffset::StartOffset`
- `FeatureOffset::EndOffset`
- `TypeSingleOffset`
- `TypeSymmetricOffset`
- `TypeNonsymmetricOffset`

UG 原始值本身就是偏置量，不只是厚度。

已验证样例：

- `E:\MyProject\tmp\CADparts\Ug\revolve_test_1\step1_ug_read.tinyxml.xml`
- 末尾薄壁旋转：
  - `<ThinWall Covered="false" StartOffset="-1" EndOffset="5"/>`

这说明 UG 读出来的原始值已经接近：

- `StartOffset`
  - 内侧偏置，向内为负
- `EndOffset`
  - 外侧偏置，向外为正

因此：

- UG -> CADExchange
  - 应尽量原样保留
- CADExchange -> UG
  - 应尽量直接落回 offset builder

---

## 6. 当前统一规则

跨系统统一时，不允许把：

- SW 的“方向枚举”
- Creo 的“material side”
- UG 的“offset 值”

混成“厚度幅值”模型。

正确做法是：

1. 先把各系统原始语义转换成 signed offset 公共语义
2. XML 里只保留 `StartOffset / EndOffset / Covered`
3. 写回目标 CAD 时，再从公共语义反推目标 CAD 的原生表达

---

## 7. 本次落地结论

本次代码修正后，已确认：

- `SW -> CADExchange`
  - 单侧内外、中面、双向厚度都按 signed offset 提取
- `Creo -> CADExchange`
  - 单侧与双侧等厚已按 signed offset 提取
  - 对称总厚 `20` 现已提成 `-10 / +10`
- `UG -> CADExchange`
  - 保留原始 offset 语义

当前仍需注意的限制：

- `Creo` 对非对称双侧薄壁只能 best-effort
- `UG` 对“旋转到点”转“基准面 + 旋转到面”时，回读应忠实反映目标 UG 特征树，而不是强行折叠回源设计意图
