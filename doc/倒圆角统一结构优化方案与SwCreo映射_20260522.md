# 倒圆角统一结构优化方案与 Sw/Creo 映射_20260522

## 1. 文档目标

本文用于整理 `CADExchange` 中倒圆角（fillet / round）的下一版统一结构优化方案，目标是：

- 在当前 `CFillet` 基础上，进一步简化变半径倒圆角的结构表达
- 减少不必要的嵌套层级
- 保留 SolidWorks 与 Creo 已验证过的公共语义
- 明确 SW / Creo 如何映射到这版新统一结构
- 给出推荐 XML 展示形式，便于后续评估序列化风格

本文是结构设计文档，不直接代表当前代码已实现。

---

## 2. 本轮优化的核心结论

针对“变半径倒圆角”这一块，建议从当前：

- `CFillet`
  - `params`
    - `radiusItems`
      - `position`
      - `radius1`
      - `radius2`
      - `refEdge`

进一步整理为更扁平、语义更直接的形式：

- 顶层仍保留 `CFillet`
- 参数层仍保留 `CFilletParams`
- 但变半径控制点改为单层扁平结构 `CFilletRadiusPoint`
- 不再增加 `profile`、`anchor` 之类的新嵌套容器

设计原则如下：

1. `position` 是 SW 与 Creo 的共享语义，不应被当作 Creo 私有补丁字段
2. `radius1 / radius2` 是变半径控制点的主值语义，应直接挂在控制点上
3. `edgeRef` 虽然更偏向写回稳定性辅助信息，但它仍然是“控制点附带的定位信息”，可以直接与控制点平铺放在一起
4. 统一结构应优先表达跨系统公共语义，而不是直接镜像某一个 CAD API 的原生树结构

## 3. 推荐统一结构

### 3.1 枚举

```cpp
enum class FilletMode {
  UNKNOWN = 0,
  CONSTANT_RADIUS,
  VARIABLE_RADIUS,
  FACE_FILLET,
  FULL_ROUND,
  CHORDAL
};

enum class FilletCrossSection {
  UNKNOWN = 0,
  CIRCULAR,
  CONIC,
  CURVATURE_CONTINUOUS
};

enum class FilletReferenceMode {
  UNKNOWN = 0,
  EDGE_CHAIN,
  FACE_FACE,
  FULL_ROUND_THREE_FACES
};

enum class FilletConicValueMode {
  NONE = 0,
  RHO,
  RADIUS,
  GENERIC_VALUE
};
```

### 3.2 变半径控制点

```cpp
struct CFilletRadiusPoint {
  double position = 0.0;                  // 统一归一化到 [0, 1]
  double radius1 = 0.0;                   // 必填
  std::optional<double> radius2;          // 非对称时可选
  std::shared_ptr<CRefEdge> edgeRef;      // 可选，稳定定位用
};
```

这一定义有几个关键点：

- `position` 直接位于控制点上
- `radius1` 直接位于控制点上
- `radius2` 只在 D1-D2 / 非对称场景中出现
- `edgeRef` 不是单独的新层，而是控制点的可选增强信息

### 3.3 参数结构

```cpp
struct CFilletParams {
  std::optional<double> defaultRadius;
  std::optional<double> defaultRadius2;
  std::vector<CFilletRadiusPoint> radiusPoints;

  FilletCrossSection crossSection{FilletCrossSection::UNKNOWN};
  FilletConicValueMode conicValueMode{FilletConicValueMode::NONE};
  std::optional<double> conicValue;

  bool isAsymmetric = false;
  bool tangentPropagation = false;
};
```

### 3.4 倒圆角特征主体

```cpp
struct CFillet : public CFeatureBase {
  FilletMode mode{FilletMode::UNKNOWN};
  FilletReferenceMode referenceMode{FilletReferenceMode::UNKNOWN};
  CFilletParams params;

  std::vector<std::shared_ptr<CRefEntityBase>> references;
  std::vector<std::shared_ptr<CRefFace>> side1Faces;
  std::vector<std::shared_ptr<CRefFace>> side2Faces;
  std::vector<std::shared_ptr<CRefFace>> centerFaces;

  std::optional<CPoint3D> firstEndFaceMarker;
  std::optional<std::string> swOverflowType;
  bool swKeepFeatures = false;
  std::optional<int> creoAttachType;
  std::optional<int> creoConicDepOption;
};
```

## 4. 与当前结构相比，优化点在哪里

### 4.1 `radiusItems` -> `radiusPoints`

建议将当前名称：

- `CFilletRadiusItem`
- `radiusItems`

调整为：

- `CFilletRadiusPoint`
- `radiusPoints`

原因：

- `item` 过于泛化，只能表达“列表项”
- `point` 更接近真实语义，即“沿倒圆角链路的控制点”

### 4.2 `referenceMode` 上提到 `CFillet` 顶层

当前 `referenceMode` 放在 `params` 内部，但从语义上看：

- `referenceMode` 决定的是“该倒圆角是边链、面面、还是全圆角三面”
- 这属于特征定义方式
- 它比 `crossSection` 更接近 feature 层，不属于纯形状参数

因此更合适的布局是：

- `CFillet.mode`
- `CFillet.referenceMode`
- `CFillet.params`

这样能把“特征如何组织”和“参数如何取值”区分开。

### 4.3 不再引入新的嵌套层

本轮明确不建议再引入：

- `VariableRadiusProfile`
- `RadiusAnchor`

原因很简单：

- 会增加结构层数
- 调用侧访问路径更长
- 但实际没有带来足够强的新抽象收益

当前更合适的做法就是一个扁平的 `CFilletRadiusPoint`。

## 5. 各字段语义约束

### 5.1 `CONSTANT_RADIUS`

- `mode = CONSTANT_RADIUS`
- `params.defaultRadius` 必填
- `params.radiusPoints` 应为空

### 5.2 `VARIABLE_RADIUS`

- `mode = VARIABLE_RADIUS`
- `params.radiusPoints` 非空
- 每个 `CFilletRadiusPoint.radius1` 必填
- `position` 统一约束在 `[0, 1]`
- `radius2` 仅在 `params.isAsymmetric = true` 时出现
- `defaultRadius / defaultRadius2` 允许保留，用作默认值或退化兜底值

### 5.3 `FULL_ROUND`

- `mode = FULL_ROUND`
- 主要依赖 `referenceMode = FULL_ROUND_THREE_FACES`
- `side1Faces / centerFaces / side2Faces` 为主
- 一般不使用 `defaultRadius`
- 一般不使用 `radiusPoints`

### 5.4 `FACE_FILLET`

- `mode = FACE_FILLET`
- 主要依赖 `referenceMode = FACE_FACE`
- `side1Faces / side2Faces` 为主

### 5.5 `CHORDAL`

- `mode = CHORDAL`
- 当前在结构上允许进入统一模型
- 参数侧通常仍走 `defaultRadius`
- SW 侧当前无稳定公共对位

## 6. 推荐 XML 展示形式

本轮推荐 XML 风格遵循当前仓库现有 `ReferenceEntity` 的表现方式，即：

- 引用尽量展示为单节点属性
- 避免把一个简单 edge reference 展开成过多子节点

### 6.1 顶层 `References`

建议展示为：

```xml
<References>
  <ReferenceEntity ParentFeatureID="FB-3"
                   TopologyIndex="32"
                   StartPoint="(0,4148.58,0)"
                   EndPoint="(5477.64,4148.58,0)"
                   MidPoint="(2738.82,4148.58,0)"
                   CurveType="Line"
                   Type="Edge"/>
</References>
```

这个形式的优点是：

- 与当前仓库 XML 风格一致
- 可读性高
- 引用是“一个对象”，不必再拆成多层

### 6.2 推荐的变半径倒圆角 XML

```xml
<Feature Type="Fillet"
         ID="FB-4"
         Name="Fillet_4"
         Suppressed="false"
         Mode="VariableRadius"
         ReferenceMode="EdgeChain">
  <Parameters DefaultRadius="0.008"
              DefaultRadius2="0.006"
              IsAsymmetric="true"
              TangentPropagation="true"
              CrossSection="Circular"/>

  <References>
    <ReferenceEntity ParentFeatureID="FB-3"
                     TopologyIndex="32"
                     StartPoint="(0,4148.58,0)"
                     EndPoint="(5477.64,4148.58,0)"
                     MidPoint="(2738.82,4148.58,0)"
                     CurveType="Line"
                     Type="Edge"/>
  </References>

  <RadiusPoints>
    <RadiusPoint Position="0.0" Radius1="0.008" Radius2="0.006"/>
    <RadiusPoint Position="0.5" Radius1="0.012" Radius2="0.009"/>
    <RadiusPoint Position="1.0" Radius1="0.010" Radius2="0.007"/>
  </RadiusPoints>
</Feature>
```

这是推荐的“默认最简版”。

### 6.3 带控制点专属 edge reference 的扩展写法

如果某些写回场景必须把控制点与具体 edge 强绑定，则允许：

```xml
<RadiusPoints>
  <RadiusPoint Position="0.0" Radius1="0.008" Radius2="0.006">
    <ReferenceEntity ParentFeatureID="FB-3"
                     TopologyIndex="32"
                     StartPoint="(0,4148.58,0)"
                     EndPoint="(5477.64,4148.58,0)"
                     MidPoint="(2738.82,4148.58,0)"
                     CurveType="Line"
                     Type="Edge"/>
  </RadiusPoint>
</RadiusPoints>
```

但这应视为扩展写法，而不是默认写法。

推荐原则是：

1. `References` 用于表达该 fillet 作用到哪些边/面
2. `RadiusPoints` 用于表达变半径分布
3. 只有在需要控制点精确绑定具体 edge 时，才在 `RadiusPoint` 下追加 `ReferenceEntity`

## 7. SolidWorks 到新统一结构的映射

## 7.1 总体映射

| SolidWorks 原生语义 | 统一结构 |
|---|---|
| `swFeatureFilletType_Simple` | `mode = CONSTANT_RADIUS` |
| `swFeatureFilletType_VariableRadius` | `mode = VARIABLE_RADIUS` |
| `swFeatureFilletType_Face` | `mode = FACE_FILLET` |
| `swFeatureFilletType_FullRound` | `mode = FULL_ROUND` |

### 7.2 参考方式

| SolidWorks 原生语义 | 统一结构 |
|---|---|
| edge chain fillet | `referenceMode = EDGE_CHAIN` |
| face fillet | `referenceMode = FACE_FACE` |
| full round 三组面 | `referenceMode = FULL_ROUND_THREE_FACES` |

对应容器：

- `EDGE_CHAIN` -> `references`
- `FACE_FACE` -> `side1Faces + side2Faces`
- `FULL_ROUND_THREE_FACES` -> `side1Faces + centerFaces + side2Faces`

### 7.3 截面和 conic

| SolidWorks 原生语义 | 统一结构 |
|---|---|
| `swFeatureFilletCircular` | `params.crossSection = CIRCULAR` |
| conic rho | `params.crossSection = CONIC` + `conicValueMode = RHO` |
| conic radius | `params.crossSection = CONIC` + `conicValueMode = RADIUS` |
| curvature continuous | `params.crossSection = CURVATURE_CONTINUOUS` |

### 7.4 对称 / 非对称

| SolidWorks 原生语义 | 统一结构 |
|---|---|
| `AsymmetricFillet = false` | `params.isAsymmetric = false` |
| `AsymmetricFillet = true` | `params.isAsymmetric = true` |

若为非对称：

- 主值进入 `defaultRadius` / `radiusPoints[*].radius1`
- 第二值进入 `defaultRadius2` / `radiusPoints[*].radius2`

### 7.5 变半径控制点

SW 变化圆角读侧可提取：

- control point `position`
- control point `radius1`
- 非对称情况下的 `radius2`
- control point 对应的边引用

因此建议映射为：

| SolidWorks 原生语义 | 统一结构 |
|---|---|
| control point parameter | `radiusPoints[i].position` |
| control point radius | `radiusPoints[i].radius1` |
| control point distance / second leg | `radiusPoints[i].radius2` |
| control point edge | `radiusPoints[i].edgeRef` |

需要特别说明：

1. SW 中 `position` 是真实公共语义，不应被视为 Creo 专属字段
2. 对于当前 SW 写侧，如果底层 API 尚不能完整回放 interior control point，则仍然建议统一结构先保留这些点
3. 统一结构的职责是表达原始语义，不应因为当前写侧有能力边界就把结构先做窄

### 7.6 默认半径

| SolidWorks 原生语义 | 统一结构 |
|---|---|
| simple fillet radius | `params.defaultRadius` |
| asymmetric second value | `params.defaultRadius2` |
| variable fillet baseline/default radius | `params.defaultRadius` / `defaultRadius2` |

## 8. Creo 到新统一结构的映射

## 8.1 总体映射

| Creo 原生语义 | 统一结构 |
|---|---|
| `PRO_ROUND_TYPE_CONSTANT` | `mode = CONSTANT_RADIUS` |
| `PRO_ROUND_TYPE_VARIABLE` | `mode = VARIABLE_RADIUS` |
| `PRO_ROUND_TYPE_FULL` | `mode = FULL_ROUND` |
| `PRO_ROUND_TYPE_CHORDAL` | `mode = CHORDAL` |

说明：

- `FACE_FILLET` 在 Creo 侧通常不是单独一层 shape type，而是 shape + reference type 组合判定的结果

## 8.2 参考方式

| Creo 原生语义 | 统一结构 |
|---|---|
| edge-based round | `referenceMode = EDGE_CHAIN` |
| surf-surf round | `referenceMode = FACE_FACE` |
| full round 三面 | `referenceMode = FULL_ROUND_THREE_FACES` |

容器映射与 SW 一致：

- `EDGE_CHAIN` -> `references`
- `FACE_FACE` -> `side1Faces + side2Faces`
- `FULL_ROUND_THREE_FACES` -> `side1Faces + centerFaces + side2Faces`

## 8.3 截面与 conic

| Creo 原生语义 | 统一结构 |
|---|---|
| 普通滚珠圆角 | `params.crossSection = CIRCULAR` |
| conic enabled | `params.crossSection = CONIC` |
| conic value | `params.conicValue` |
| conic dependent option | `params.conicValueMode` 或 Creo 扩展字段 |

如果当前 `Creo` 原生语义无法稳定区分 `RHO / RADIUS / GENERIC_VALUE`，则可以：

1. 优先落到 `params.conicValue`
2. 详细差异保留到 `creoConicDepOption`

## 8.4 非对称

Creo 更常通过双 leg 表达非对称，因此建议：

| Creo 原生语义 | 统一结构 |
|---|---|
| 仅 leg1 | `params.isAsymmetric = false` |
| leg1 + leg2 | `params.isAsymmetric = true` |

并映射为：

- leg1 -> `defaultRadius` / `radiusPoints[*].radius1`
- leg2 -> `defaultRadius2` / `radiusPoints[*].radius2`

## 8.5 变半径控制点

Creo 变半径 round 读取时，如果能拿到：

- point ref
- 对应 edge
- 沿 edge 的参数位置
- leg1 / leg2

则建议统一映射为：

| Creo 原生语义 | 统一结构 |
|---|---|
| point parameter / point on edge | `radiusPoints[i].position` |
| leg1 | `radiusPoints[i].radius1` |
| leg2 | `radiusPoints[i].radius2` |
| hosting edge | `radiusPoints[i].edgeRef` |

这里的关键理解是：

- `position` 不是 Creo 私货
- `edgeRef` 才更偏向 Creo 写侧稳定回放辅助信息

## 8.6 Chordal

对于：

- `PRO_ROUND_TYPE_CHORDAL`

建议映射为：

- `mode = CHORDAL`
- `referenceMode` 按实际 edge/face 组织方式填写
- 主值仍优先进入 `defaultRadius`

说明：

- 统一结构允许承接 chordal
- 但写回 SW 侧当前仍无稳定公共对位
- 因此它属于“结构支持先于 SW 闭环支持”

## 9. 新结构下的 SW <-> CADExchange <-> Creo 理解方式

这版结构的核心思想可以总结成三层：

### 9.1 第一层：特征类型层

由：

- `mode`
- `referenceMode`

决定这是哪类倒圆角：

- 常半径
- 变半径
- 面圆角
- 全圆角
- chordal

### 9.2 第二层：形状参数层

由 `params` 表达：

- 默认半径
- 变半径控制点列
- 截面类型
- conic 值
- 是否非对称
- 是否切线传播

### 9.3 第三层：参考对象层

由：

- `references`
- `side1Faces`
- `side2Faces`
- `centerFaces`

表达该倒圆角到底作用到哪些边/面。

这使得：

- SW 的 control point `position`
- Creo 的 point-on-edge `position`

都能落到同一个 `CFilletRadiusPoint.position` 上。

同时：

- SW / Creo 对具体 edge 的附着信息

都能作为 `CFilletRadiusPoint.edgeRef` 被保留下来。

---

## 10. 与当前 `CFillet v1` 的关系

这版方案不是推翻现有 `CFillet v1`，而是在 v1 的基础上做结构收敛：

1. `CFilletRadiusItem` 更名为 `CFilletRadiusPoint`
2. `radiusItems` 更名为 `radiusPoints`
3. `referenceMode` 从 `params` 提升到 `CFillet` 顶层
4. 保持 `CFilletParams` 只承载形状参数
5. 不新增 `profile` / `anchor` 等嵌套层

因此，这是一版“更扁平、更清晰”的收敛方案，而不是重起炉灶。

---

## 11. 结论

对于 SW 与 Creo 的倒圆角统一结构，尤其是变半径倒圆角，本轮更合理的优化方向不是：

- 继续往下拆更多辅助结构
- 或为了当前某侧写回方便，把控制点结构过度拍扁成单半径数组

而是：

1. 保留控制点这一层
2. 把控制点压平成单个 `CFilletRadiusPoint`
3. 把 `position` 明确定义为 SW / Creo 的共享语义
4. 把 `edgeRef` 保留为控制点的可选增强定位信息
5. 把 `referenceMode` 从纯参数层提升回特征定义层

这版结构既足够简洁，也仍然能承接当前已验证过的：

- SW 常半径 / 变半径 / face fillet / full round
- Creo constant / variable / full / chordal

以及后续 XML 表达和写回能力扩展。
