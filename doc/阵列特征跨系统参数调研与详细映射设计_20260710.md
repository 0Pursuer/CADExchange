# 阵列特征跨系统参数调研与详细映射设计 (SolidWorks, Creo, UG/NX)

本报告在第一阶段调研的基础上，深入分析 SolidWorks、Creo 和 UG/NX 各个系统底层关于阵列特征（线性、圆周、镜像）的底层 API 接口、数据元素、控制参数以及构建逻辑；设计并定义统一的“最小公共特征结构”；并给出每个系统在读取与写入时的核心参数映射与数据转换规则。

---

## 1. 阵列特征系统级深入分析

### 1.1 SolidWorks COM 接口与数据模型

在 SolidWorks 中，阵列并不是一种带有不同子类型的单一特征，而是多个完全独立的特征类型（各自由不同的 FeatureData 接口控制）。从已存在的特征中提取或新创建阵列特征，必须通过特定的 FeatureData 接口。

#### 1.1.1 线性阵列 (Linear Pattern)
- **提取接口**：`ILinearPatternFeatureData` 与新版 `ILinearPatternFeatureData2`（推荐）。
- **主要参数项**：
  - **方向一**：
    - `GetDirection1` / `SetDirection1`：返回一个代表方向的 `IEntity` 接口指针（可以代表线性边、轴线、基准面法线或尺寸）。
    - `D1ReverseDirection` (bool)：方向是否反向。
    - `D1Spacing` (double)：间距，**API 规定单位始终为米(m)**。
    - `D1Number` (long)：实例数量（包括源实体）。
  - **方向二**（可选，当 `Direction2 = true` 时）：
    - `GetDirection2` / `SetDirection2`：方向二实体指针。
    - `D2ReverseDirection` (bool)。
    - `D2Spacing` (double)：间距，单位为米。
    - `D2Number` (long)。
    - `D2PatternSeedOnly` (bool)：如果为 true，则在方向二中只复制源，而不复制方向一产生的阵列实例。
  - **源对象类型与数组**：
    - `PatternFeatureArray` (Variant/IDispatch Array)：被阵列的特征数组。
    - `PatternFaceArray` (Variant/IDispatch Array)：被阵列的表面（常用于面阵列）。
    - `PatternBodyArray` (Variant/IDispatch Array)：被阵列的实体数组（用于多实体环境下的实体拷贝）。
  - **排除的实例**：
    - `SkippedItemArray` (Variant Array)：行列索引元组形式，格式为 `[r1, c1, r2, c2, ...]` 的一维数组，代表跳过的格网位置（1-indexed 或 0-indexed，依赖 API 版本）。
  - **解算选项**：
    - `GeometryPattern` (bool)：如果为 true，则仅在几何上复制源对象的面或体，而不对每个拷贝实例重新执行建模特征计算（大大加快重建速度）。

#### 1.1.2 圆周阵列 (Circular Pattern)
- **提取接口**：`ICircularPatternFeatureData`。
- **主要参数项**：
  - **旋转轴**：`GetAxis` / `SetAxis`（指向旋转轴、圆柱面、圆环面或线性边的 `IEntity` 指针）。
  - **旋转反向**：`ReverseDirection` (bool)。
  - **实例数量**：`TotalInstances` (long)。
  - **间距类型**：
    - `EqualSpacing` (bool)：是否等间距分布。
    - `Spacing` (double)：当 `EqualSpacing = false` 时生效，为相邻两实例的角度，**单位为弧度(rad)**。
    - `TotalAngle` (double)：当 `EqualSpacing = true` 时生效，为阵列分布的总角度，**单位为弧度(rad)**。
  - **方向二 (Concentric Rings)**：
    - `Direction2` (bool)：是否启用径向/同心环阵列。
    - `D2Spacing` (double)：径向环与环之间的线性距离（单位：米）。
    - `D2TotalInstances` (long)：同心环的数量。
    - `D2PatternSeedOnly` (bool)：是否只阵列源。
  - **源对象数组**：`PatternFeatureArray`、`PatternFaceArray`、`PatternBodyArray`。
  - **排除的实例**：`SkippedItemArray` (行列坐标)。
  - **解算选项**：`GeometryPattern` (bool)。

#### 1.1.3 镜像特征 (Mirror Pattern)
- **提取接口**：`IMirrorPatternFeatureData` (特征镜像数据) / `IMirrorSolidFeatureData` (实体镜像数据)。
- **主要参数项**：
  - **镜像面**：`GetMirrorPlane` / `SetMirrorPlane` (基准面或平表面的 `IEntity` 指针)。
  - **镜像源对象**：
    - 特征镜像：`PatternFeatureArray`。
    - 面镜像：`PatternFaceArray`。
    - 实体镜像：`PatternBodyArray` (在 `IMirrorSolidFeatureData` 中为 `BodyArray`)。
  - **解算选项**：`GeometryPattern` (bool)。

### 1.2 Creo (Pro/ENGINEER) Element Tree 与数据模型

在 Creo 中，阵列是一种对已有特征进行变换复制的“操作”。其底层并非一个包含源列表的独立节点，而是通过构造包含变换描述的特征树（`PRO_FEAT_PATTERN`）来重构该特征。创建阵列前，目标特征（Seed Feature）必须已存在。

#### 1.2.1 阵列的模式标记 (`PRO_E_GENPAT_TYPE`)
在 Creo Toolkit 中，通过填充元素 `PRO_E_GENPAT_TYPE` 控制阵列的大类：
*   `PRO_GENPAT_DIR_DRIVEN`：方向阵列（Direction Pattern），类似线性阵列。
*   `PRO_GENPAT_AXIS_DRIVEN`：轴阵列（Axis Pattern），类似圆周阵列。
*   `PRO_GENPAT_DIM_DRIVEN`：尺寸阵列（Dimension Pattern），Creo 特有的通过修改源特征尺寸参数驱动的阵列。
*   `PRO_GENPAT_REF_DRIVEN`：引用阵列（Reference Pattern），自动跟随父特征的阵列分布。
*   `PRO_GENPAT_TABLE_DRIVEN`：表格阵列（Table Pattern）。
*   `PRO_GENPAT_FILL_DRIVEN`：填充阵列（Fill Pattern）。
*   `PRO_GENPAT_CRV_DRIVEN`：曲线驱动阵列（Curve Pattern）。

#### 1.2.2 方向阵列 (Direction Pattern - `PRO_GENPAT_DIR_DRIVEN`)
基于平移增量，采用如下元素控制：
- **方向一参考**：`PRO_E_DIR_PAT_DIR1_OPT` 设为 `PRO_GENPAT_TRANSLATIONAL` (平移)，并且 `PRO_E_GENPAT_DIR1` 下包含 `PRO_E_DIRECTION_REFERENCE` (方向参考 `ProSelection` 指向边、轴或面) 和 `PRO_E_DIR_PAT_DIR1_FLIP` (是否翻转)。
- **方向一数量**：`PRO_E_GENPAT_FIRST_DIR_NUM_INST` (int)。
- **方向一间距**：`PRO_E_GENPAT_DIR1_INC` (double，**模型实际长度单位，可为负**)。
- **方向二参数**（可选）：
  - 方向二参考：`PRO_E_DIR_PAT_DIR2_OPT`、`PRO_E_GENPAT_DIR2` 下的 `PRO_E_DIRECTION_REFERENCE`、`PRO_E_DIR_PAT_DIR2_FLIP`。
  - 方向二数量：`PRO_E_GENPAT_SECOND_DIR_NUM_INST` (int)。
  - 方向二间距：`PRO_E_GENPAT_DIR2_INC` (double)。
- **抑制/排除列表**：
  - `PRO_E_PATTERN_EXCL_LIST` (Array of ints)：被屏蔽的实例索引序列号。

#### 1.2.3 轴阵列 (Axis Pattern - `PRO_GENPAT_AXIS_DRIVEN`)
基于旋转增量，采用如下元素控制：
- **轴线参考**：`PRO_E_AXIS_AXIS_REF` (`ProSelection` 对应基准轴或圆柱面)。
- **实例数量**：`PRO_E_AXIS_NUM1` (int)。
- **角度增量**：`PRO_E_AXIS_INCR1` (double，**度数，degree**)。
- **角度均分控制**：`PRO_E_AXIS_VAL_OPT1` (可设置为由增量决定，或者指定总跨度角度由系统等分)。
- **同心圆环扩展（径向阵列，类似方向二）**：
  - `PRO_E_AXIS_NUM2` (int，圆环数量)。
  - `PRO_E_AXIS_INCR2` (double，径向间距增量)。
- **排除列表**：`PRO_E_PATTERN_EXCL_LIST`。

#### 1.2.4 镜像特征 (Mirror - `PRO_FEAT_MIRROR`)
在 Creo 中，镜像不是阵列的一种类型，而是一个独立的特征类型：
- **镜像平面**：`PRO_E_MIRROR_PLANE` (`ProSelection` 指向面或基准面)。
- **源特征列表**：`PRO_E_MIRROR_FEAT_LIST` (多特征时记录源特征列表)。
- **源实体**：`PRO_E_MIRROR_BODY_LIST` (实体镜像)。

### 1.3 UG/NX C++ (NXOpen) 数据模型

UG/NX 将所有阵列类型统一集成在 `NXOpen::Features::PatternFeature` 特征下，并通过 `NXOpen::Features::PatternFeatureBuilder` 进行操作。其具体的阵列形态由布局对象（`LayoutDefinition`）派生类来指定。

#### 1.3.1 线性布局 (Linear Layout - `LayoutDefinitionLinear`)
- **定义类**：`NXOpen::Features::LayoutDefinitionLinear`。
- **主要参数项**：
  - **方向一**：
    - `Direction1` (`NXOpen::Direction`)：包含矢量表达，可来自边、轴线或 Csys 方向。
    - `D1SpacingType` (`NXOpen::Features::SpacingType`)：
      - `CountAndPitch`：实例数与间距。
      - `CountAndSpan`：实例数与总跨度。
      - `PitchAndSpan`：间距与总跨度。
    - `D1Count` (`NXOpen::Expression`)：数量表达式。
    - `D1Pitch` (`NXOpen::Expression`)：间距表达式（单位：毫米/英寸）。
    - `D1Span` (`NXOpen::Expression`)：跨度表达式。
  - **方向二**（通过 `Direction2Enabled` 激活）：
    - `Direction2` (`NXOpen::Direction`)。
    - `D2SpacingType`。
    - `D2Count`、`D2Pitch`、`D2Span` 表达式。
    - `PatternAlongDirection1` (bool)：如果为 false，则类似 SW 中的 D2PatternSeedOnly。

#### 1.3.2 圆周布局 (Circular Layout - `LayoutDefinitionCircular`)
- **定义类**：`NXOpen::Features::LayoutDefinitionCircular`。
- **主要参数项**：
  - **旋转轴**：`RotationAxis` (`NXOpen::Axis`：由点和矢量构成，或者引用圆柱面)。
  - **角向分布参数**：
    - `AngularSpacing` (`SpacingType`)：间距类型（如 `CountAndPitch` 等）。
    - `AngularCount` (`NXOpen::Expression`)。
    - `AngularPitch` (`NXOpen::Expression`，**度数，degree**)。
    - `AngularSpan` (`NXOpen::Expression`)。
  - **径向分布参数（方向二）**：
    - `RadialSpacing` (`SpacingType`)。
    - `RadialCount` (`NXOpen::Expression`)。
    - `RadialPitch` (`NXOpen::Expression`，线性环距)。
    - `RadialSpan` (`NXOpen::Expression`)。

#### 1.3.3 镜像布局与独立镜像特征
UG/NX 支持在 `PatternFeatureBuilder` 中直接配置 `LayoutDefinitionMirror`，也支持独立的镜像特征（由 `NXOpen::Features::MirrorFeature` 或 `MirrorBody` 表达）：
- **镜像平面**：`Plane` (`NXOpen::Plane`)。
- **源对象**：`FeatureList` (特征列表) 或 `MirrorBody` 包含的体。

## 2. 最小公共特征统一结构设计 (CADExchange Level)

为了确保能在 `CADExchange` 顶层以无损和最小冗余方式映射上述三种系统，我们设计如下的“最小公共特征结构”（包含线性、圆周、镜像）。

```cpp
#pragma once
#include "UnifiedFeatures.h"
#include <memory>
#include <vector>
#include <optional>

namespace CADExchange {

/**
 * @brief 阵列作用域
 */
enum class PatternScope {
  FEATURES, ///< 针对特征列表阵列
  FACES,    ///< 针对指定的一组面阵列
  BODIES    ///< 针对多实体环境下的体阵列
};

/**
 * @brief 间距分布类型
 */
enum class PatternSpacingType {
  PITCH_AND_COUNT, ///< 间距和数量（最通用）
  SPAN_AND_COUNT   ///< 总长度/总角度和数量（用于等间距均分）
};

/**
 * @brief 阵列格网索引元组，用于标记跳过项
 */
struct CPatternIndex {
  int dir1Index = 0; ///< 方向一（或角向）索引，从 0 开始
  int dir2Index = 0; ///< 方向二（或径向）索引，若单向阵列则为 0

  bool operator==(const CPatternIndex& other) const {
    return dir1Index == other.dir1Index && dir2Index == other.dir2Index;
  }
};

/**
 * @brief 统一线性阵列方向参数
 */
struct CLinearPatternDir {
  std::shared_ptr<CRefEntityBase> directionRef; ///< 方向参考（边、轴、面法线等）
  bool reverse = false;                         ///< 是否反向

  PatternSpacingType spacingType = PatternSpacingType::PITCH_AND_COUNT;
  double spacing = 0.0;                         ///< 间距值（单位：模型主单位 mm，不是米）
  int count = 1;                                ///< 包含源特征在内的实例总数
};

/**
 * @brief 统一线性阵列特征定义
 */
struct CLinearPattern : public CFeatureBase {
  CLinearPatternDir dir1;
  std::optional<CLinearPatternDir> dir2;
  
  bool patternSeedOnly = false; ///< 方向二是否仅对“源特征”进行阵列

  PatternScope scope = PatternScope::FEATURES;
  std::vector<std::shared_ptr<CRefEntityBase>> seedObjects; ///< 源对象引用（特征、面或体）
  
  std::vector<CPatternIndex> skippedInstances; ///< 跳过的阵列实例索引集合
  bool geometryPattern = false;                ///< 几何阵列加速选项

  CLinearPattern() { featureType = FeatureType::Unknown; /* 应注册为 FeatureType::LinearPattern */ }
};

/**
 * @brief 统一圆周阵列方向参数
 */
struct CCircularPatternDir {
  std::shared_ptr<CRefEntityBase> axisRef; ///< 旋转轴参考（轴线、线性边、圆柱表面等）
  bool reverse = false;                    ///< 是否反向旋转

  PatternSpacingType spacingType = PatternSpacingType::PITCH_AND_COUNT;
  double angle = 0.0;                      ///< 旋转角（单位：弧度 rad）
  int count = 1;                           ///< 实例数
};

/**
 * @brief 统一圆周阵列特征定义
 */
struct CCircularPattern : public CFeatureBase {
  CCircularPatternDir dir1;
  
  // 圆周阵列的方向二一般表现为径向同心圆环阵列（基于线性平移距离）
  std::optional<CLinearPatternDir> dir2;
  bool patternSeedOnly = false;

  PatternScope scope = PatternScope::FEATURES;
  std::vector<std::shared_ptr<CRefEntityBase>> seedObjects;
  
  std::vector<CPatternIndex> skippedInstances;
  bool geometryPattern = false;

  CCircularPattern() { featureType = FeatureType::Unknown; /* 应注册为 FeatureType::CircularPattern */ }
};

/**
 * @brief 统一镜像特征定义
 */
struct CMirrorPattern : public CFeatureBase {
  std::shared_ptr<CRefEntityBase> mirrorPlaneRef; ///< 镜像对称面（基准面或平面面）

  PatternScope scope = PatternScope::FEATURES;
  std::vector<std::shared_ptr<CRefEntityBase>> seedObjects;
  bool geometryPattern = false;

  CMirrorPattern() { featureType = FeatureType::Unknown; /* 应注册为 FeatureType::MirrorPattern */ }
};

} // namespace CADExchange
```

## 3. 各系统参数映射与转换规则

### 3.1 线性阵列参数映射表

| 统一结构字段 (`CLinearPattern`) | SolidWorks API 映射 | Creo Elements 映射 | UG/NX (NXOpen) 映射 |
| :--- | :--- | :--- | :--- |
| **`dir1.directionRef`** | `GetDirection1` (IEntity) | `PRO_E_DIRECTION_REFERENCE` | `Direction1` (NXOpen::Direction) |
| **`dir1.reverse`** | `D1ReverseDirection` | `PRO_E_DIR_PAT_DIR1_FLIP` | 矢量的反向检查 |
| **`dir1.spacing`** | `D1Spacing` **(米 $\rightarrow$ 毫米转换)** | `PRO_E_GENPAT_DIR1_INC` | `D1Pitch` 表达式值 |
| **`dir1.count`** | `D1Number` | `PRO_E_GENPAT_FIRST_DIR_NUM_INST` | `D1Count` 表达式值 |
| **`dir2` (激活状态)** | `Direction2 == True` | `PRO_E_GENPAT_SECOND_DIR_NUM_INST > 1` | `Direction2Enabled == True` |
| **`dir2->spacing`** | `D2Spacing` **(米 $\rightarrow$ 毫米转换)** | `PRO_E_GENPAT_DIR2_INC` | `D2Pitch` 表达式值 |
| **`patternSeedOnly`** | `D2PatternSeedOnly` | 根据子元素设置映射 | `!PatternAlongDirection1` |
| **`seedObjects`** | 根据 Scope 解析三个 Array | 操作目标 Seed Feature | `FeatureList` / `GeometryList` |
| **`skippedInstances`** | 解析 `SkippedItemArray` 一维行列列表 | 解析 `PRO_E_PATTERN_EXCL_LIST` | 解析排除实例点索引 |
| **`geometryPattern`** | `GeometryPattern` | 映射特殊再生选项 | `PatternMethod = Geometry` |

### 3.2 圆周阵列参数映射表

| 统一结构字段 (`CCircularPattern`) | SolidWorks API 映射 | Creo Elements 映射 | UG/NX (NXOpen) 映射 |
| :--- | :--- | :--- | :--- |
| **`dir1.axisRef`** | `GetAxis` (IEntity) | `PRO_E_AXIS_AXIS_REF` | `RotationAxis` |
| **`dir1.reverse`** | `ReverseDirection` | 旋转方向判定 | 旋转轴矢量方向判定 |
| **`dir1.spacingType`** | `EqualSpacing ? SPAN : PITCH` | 依据 `PRO_E_AXIS_VAL_OPT1` | 依据 `AngularSpacing.SpacingType` |
| **`dir1.angle`** | `Spacing` / `TotalAngle` **(弧度直接读取)** | `PRO_E_AXIS_INCR1` **(度数 $\rightarrow$ 弧度转换)** | `AngularPitch` 表达式值 **(度数 $\rightarrow$ 弧度)** |
| **`dir1.count`** | `TotalInstances` | `PRO_E_AXIS_NUM1` | `AngularCount` |
| **`dir2` (径向激活)** | `Direction2 == True` | `PRO_E_AXIS_NUM2 > 1` | `RadialCount` 表达式值 $> 1$ |
| **`dir2->spacing`** | `D2Spacing` **(米 $\rightarrow$ 毫米)** | `PRO_E_AXIS_INCR2` | `RadialPitch` 表达式值 |
| **`dir2->count`** | `D2TotalInstances` | `PRO_E_AXIS_NUM2` | `RadialCount` 表达式值 |
| **`skippedInstances`** | 解析 `SkippedItemArray` | 解析 `PRO_E_PATTERN_EXCL_LIST` | 提取排除索引 |
| **`geometryPattern`** | `GeometryPattern` | 再生方法映射 | `PatternMethod = Geometry` |

### 3.3 镜像特征参数映射表

| 统一结构字段 (`CMirrorPattern`) | SolidWorks API 映射 | Creo Elements 映射 | UG/NX (NXOpen) 映射 |
| :--- | :--- | :--- | :--- |
| **`mirrorPlaneRef`** | `GetMirrorPlane` (IEntity) | `PRO_E_MIRROR_PLANE` | `Plane` (NXOpen::Plane) |
| **`scope`** | 依据 `MirrorBodies` 控制 | 区分 Feature 还是 Body 列表 | 区分特征镜像还是实体镜像类 |

| **`geometryPattern`** | `GeometryPattern` | 常规重建模式 | 默认几何特征生成 |

### 3.4 核心参数转换与边界处理逻辑

#### 3.4.1 长度单位规整转换 (Distance Units)
*   **SolidWorks $\rightarrow$ CADExchange**：
    $$Spacing_{mm} = Spacing_{SW\_Meter} \times 1000.0$$
*   **CADExchange $\rightarrow$ SolidWorks**：
    $$Spacing_{SW\_Meter} = Spacing_{mm} / 1000.0$$
*   **Creo / UG $\leftrightarrow$ CADExchange**：
    保持数值一致（它们本身代表模型主单位，例如毫米）。若 Creo 使用了英寸（Inch）模板，则应在读侧根据全局单位制先换算至 mm，写入时再换算回 Creo 所需的主单位。

#### 3.4.2 角度单位规整转换 (Angle Units)
*   **SolidWorks $\leftrightarrow$ CADExchange**：
    直接映射，两端均采用**弧度 (rad)** 表达。
*   **Creo $\rightarrow$ CADExchange**：
    $$Angle_{rad} = Angle_{Creo\_Degree} \times \frac{\pi}{180.0}$$
*   **CADExchange $\rightarrow$ Creo**：
    $$Angle_{Creo\_Degree} = Angle_{rad} \times \frac{180.0}{\pi}$$
*   **UG/NX $\leftrightarrow$ CADExchange**：
    UG底层的表达式（Expression）解析出来的数值为度（degree），需与 Creo 同样进行度与弧度的双向换算。

#### 3.4.3 排除/跳过实例格网索引转换 (Skipped Instance Grid)
*   **SolidWorks $\rightarrow$ CADExchange**：
    SW 的 `SkippedItemArray` 是一个扁平数组。比如跳过位置 `(r1, c1)` 和 `(r2, c2)`，其数组为 `[r1, c1, r2, c2]`。
    *   **处理规则**：在读侧解析该数组，并转化为 `std::vector<CPatternIndex>`。如果为 1-indexed，根据需要将其标准化为以 0 开始的索引，如 `dir1Index = r - 1`，`dir2Index = c - 1`。
*   **Creo $\leftrightarrow$ CADExchange**：
    Creo 的排除列表为 `PRO_E_PATTERN_EXCL_LIST`，存储的是一维扁平序列号。
    *   **转换逻辑**：由于线性阵列在逻辑上是双向格网，序列号 $Idx_{Flat}$ 与行列索引的关系通常为：
        $$Idx_{Flat} = dir2Index \times dir1Count + dir1Index$$
        在读写转换时，利用此算式进行一维索引与二维索引的双向互转。
*   **UG/NX $\leftrightarrow$ CADExchange**：
    UG 通过特定实例点的局部标识或排除项序列定义排除。读取后，根据其实例生成顺序，同样将其折算为二维网格索引 `(dir1Index, dir2Index)`。

#### 3.4.4 Creo 写回时的 Seed Feature 特性处理
在 SolidWorks 和 UG 中，创建阵列时可以以“独立特征”创建，并引用任意的源特征。但 Creo 要求阵列是附加在某个现存特征上的操作。
*   **写入规则**：
    1.  当 `CADExchange` 重建 `CLinearPattern` 时，写侧逻辑必须先通过特征 ID 定位到 `seedObjects[0]` 对应的 Creo 实体特征（例如已重建完毕的拉伸或孔特征，此时为 `ProFeature` 句柄）。
    2.  调用 `ProFeaturePatternCreate` 作用于该特征句柄，并在特征树中填充阵列类型和方向等参数。这可以保证 Creo 模型的特征树拓扑完美贴合其原生行为。
