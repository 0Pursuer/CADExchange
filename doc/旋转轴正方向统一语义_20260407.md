# 旋转轴正方向统一语义_20260407

## 目标

把 `XML.Axis.Direction` 统一定义为：

- 该旋转特征的**正旋转方向**
- 即：按该向量应用右手定则时，对应 CAD 原生“默认正角度/默认旋转侧”的方向

同时保留：

- `ReferenceEntity` 只表达原始引用元素
- 其 `StartPoint/EndPoint/Direction` 等几何数据不因统一语义而重排

## 与旧语义的区别

旧实现里，不同 CAD 对 `Axis.Direction` 的来源不一致：

- `Creo`：多半来自轴引用几何方向
- `SW`：多半来自轴几何方向，再配合 `ReverseDirection`
- `UG`：更接近 builder 当前采用的轴方向

这会导致：

- `ReferenceEntity` 和 `Axis.Direction` 有时同向，有时反向
- 用户看到的 `Axis.Direction` 无法直接解释“特征正向旋转”

## 新规则

统一层只保留一条规则：

- `Axis.Direction` = 特征正旋转方向

各 CAD 系统的原始几何方向与草图法向、方向枚举的关系，由桥接层负责转换。

## 各系统转换原则

### Creo

- 原始轴引用几何仍来自 `PRO_E_REVOLVE_AXIS`
- 正旋转方向沿旋转轴线本身
- 当前已验证规则：
  - 先取原始轴几何方向
  - `SIDE_ONE` 保持不变
  - `SIDE_TWO` 取反
- 证据样件：`revolve_test_1.prt.3`

### SolidWorks

- 原始轴引用几何仍来自 axis edge / sketch line / ref axis
- 正旋转方向预计由：
  - axis 几何方向
  - `ReverseDirection`
 共同决定
- 需进一步用样件验证

### UG/NX

- 当前 `builder->Axis()->Direction()` 很可能已经是 feature 实际采用的正向
- 但仍需结合实际样件确认其是否稳定等价于“正旋转方向”

## 兼容性注意

- 旧 XML 中的 `Axis.Direction` 不能再直接解释为轴引用几何方向
- compare / 回归脚本在跨版本对比时，要注意语义切换带来的差异
