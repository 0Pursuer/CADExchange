# 倒圆角统一结构设计（SW 与 Creo）

## 1. 目的

本文定义 `CADExchange` 中倒圆角（fillet / round）的第一版统一结构方案，用于承接：

- SolidWorks 倒圆角
- Creo Round

本文重点解决：

- 统一结构应该长什么样
- 哪些字段进入公共主结构
- SW 如何映射到该结构
- Creo 如何映射到该结构
- 第一版能力边界应该画在哪里

本文目标是为后续在 `CADExchange\core\UnifiedFeatures.h`、builder / accessor、以及 TinyXML 序列化中的正式实现提供直接依据。

---

## 2. 设计原则

### 2.1 不使用极简 `edges + radius` 模型

倒圆角不能像部分简单特征那样，只用：

- 参考边集合
- 一个半径值

来统一表示。

原因是 SW 与 Creo 的倒圆角都至少还包含：

- 特征模式差异
- 参考类型差异
- 截面形状差异
- 变半径控制点
- full round 多组参考面

如果只保留 `edges + radius`，会直接丢失：

- SW `VariableRadius / Face / FullRound`
- Creo `reference type / full round / dual-leg`

### 2.2 v1 先做公共主干，不追求 GUI 级全量等价

第一版结构优先覆盖 SW 与 Creo 的稳定公共交集：

- `constant radius`
- `variable radius`
- `face-face`
- `full round`
- `circular / conic / curvature continuous`
- `asymmetric / dual-leg`

不建议 v1 直接纳入主结构的能力：

- SW `partial edge`
- SW `keep features`
- SW `overflow`
- Creo `through curve`
- Creo `chordal`
- Creo `attach`
- Creo `transition`
- Creo `spine`

这些能力应先保留在扩展字段中，避免一开始把主结构做得过重。

### 2.3 公共字段与扩展字段分层

统一结构应分成两层：

1. 公共主结构
2. 厂商扩展字段

公共主结构负责承接可稳定跨系统映射的语义。  
厂商扩展字段负责保留当前暂不进入公共建模的本地行为控制参数。

## 3. v1 要覆盖的能力范围

### 3.1 需要覆盖

- SW `Simple`
- SW `VariableRadius`
- SW `Face`
- SW `FullRound`
- Creo `PRO_ROUND_TYPE_CONSTANT`
- Creo `PRO_ROUND_TYPE_VARIABLE`
- Creo `PRO_ROUND_TYPE_FULL`
- Creo `PRO_ROUND_REF_EDGE`
- Creo `PRO_ROUND_REF_SURF_SURF`

- Creo `transition`
- Creo `ball spine / compound spine`
- SW `partial-edge fillet`
- SW `overflow type`
- SW 更细粒度 transition / overflow 族行为

## 4. 建议的数据结构

### 4.1 FeatureType 扩展

首先需要在 `FeatureType` 中新增：

```cpp
Fillet
```

### 4.2 枚举定义

```cpp
enum class FilletMode {
  UNKNOWN = 0,
  CONSTANT_RADIUS,
  VARIABLE_RADIUS,
  FACE_FILLET,
  FULL_ROUND,
  CHORDAL
};

enum class FilletCrossSection {
  UNKNOWN = 0,
  CIRCULAR,
  CONIC,
  CURVATURE_CONTINUOUS
};

enum class FilletReferenceMode {
  UNKNOWN = 0,
  EDGE_CHAIN,
  FACE_FACE,
  FULL_ROUND_THREE_FACES
};

enum class FilletConicValueMode {
  NONE = 0,
  RHO,
  RADIUS,
  GENERIC_VALUE
};
```

### 4.3 变半径控制点结构

```cpp
struct CFilletRadiusItem {
  std::optional<double> position;
  std::optional<double> radius1;
  std::optional<double> radius2;
  std::shared_ptr<CRefEdge> refEdge;
};
```

字段说明：

- `position`
  - 表示控制点沿参考边的参数位置
  - 统一建议使用 `0..1` 比例
- `radius1`
  - 主半径值
- `radius2`
  - 第二半径值，用于非对称或 dual-leg 语义
- `refEdge`
  - 控制点依附的参考边

### 4.4 参数结构

```cpp
struct CFilletParams {
  FilletCrossSection crossSection{FilletCrossSection::UNKNOWN};
  FilletReferenceMode referenceMode{FilletReferenceMode::UNKNOWN};

  std::optional<double> defaultRadius;
  std::optional<double> defaultRadius2;
  bool isAsymmetric = false;
  bool tangentPropagation = false;
  bool curvatureContinuous = false;

  std::optional<double> conicValue;
  FilletConicValueMode conicValueMode{FilletConicValueMode::NONE};

  std::vector<CFilletRadiusItem> radiusItems;
};
```

字段说明：

- `crossSection`
  - 统一承接 circular / conic / curvature continuous
- `referenceMode`
  - 定义该倒圆角主要使用哪种参考组织方式
- `defaultRadius`
  - 恒定倒圆角的主半径
- `defaultRadius2`
  - 双值或非对称时的第二半径
- `isAsymmetric`
  - 标记是否存在双值半径语义
- `tangentPropagation`
  - 公共层只保留一个“近似切线延展”语义
- `curvatureContinuous`
  - 补充记录连续性标记
- `conicValue`
  - conic 数值
- `conicValueMode`
  - 区分 `rho / radius / generic`
- `radiusItems`
  - 变半径控制点集合

### 4.5 特征结构

```cpp
struct CFillet : public CFeatureBase {
  FilletMode mode{FilletMode::UNKNOWN};
  CFilletParams params;

  // EDGE_CHAIN
  std::vector<std::shared_ptr<CRefEntityBase>> references;

  // FACE_FACE or FULL_ROUND_THREE_FACES
  std::vector<std::shared_ptr<CRefFace>> side1Faces;
  std::vector<std::shared_ptr<CRefFace>> side2Faces;

  // FULL_ROUND_THREE_FACES
  std::vector<std::shared_ptr<CRefFace>> centerFaces;

  // Common marker for D1 side
  std::optional<CPoint3D> firstEndFaceMarker;

  // SW extension
  std::optional<std::string> swOverflowType;
  bool swKeepFeatures = false;

  // Creo extension
  std::optional<int> creoAttachType;
  std::optional<int> creoConicDepOption;

  CFillet() { featureType = FeatureType::Fillet; }
};
```

字段补充说明：
- `firstEndFaceMarker`
  - 用于标识 `D1` 所在的第一端面
  - 保存值是该端面上的稳定几何点，而不是 CAD 原生面 ID
  - 当前统一规则使用“第一端面中点”作为 marker
  - XML 字段名为 `FirstEndFaceMarker`

### 4.6 第一端面标识字段补充

为了承接 SW / Creo 在 `D1-D2` 倒圆角上的公共语义，统一结构补充：

```cpp
std::optional<CPoint3D> firstEndFaceMarker;
```

该字段的进入条件如下：
- 仅对 `params.isAsymmetric = true` 的倒圆角开放
- 仅对 `params.crossSection = CONIC` 或 `params.crossSection = CURVATURE_CONTINUOUS` 开放
- `Chordal` 当前仅在 Creo 侧纳入；SW 侧仍无稳定公共对位
- 普通对称倒圆角不纳入
- 如果源 CAD 侧无法稳定确认 `D1` 对应端面，则不写入 XML

这条约束的目的，是只保留两系统都可解释、且写侧确实需要的“端面侧别”语义，避免把单一 CAD 的内部端面标识强塞进公共模型。

## 5. 为什么这样设计

### 5.1 `mode` 不能省

仅靠参考集合和半径值，无法区分：

- `constant radius`
- `variable radius`
- `face fillet`
- `full round`

所以必须显式保留 `mode`。

### 5.2 `referenceMode` 不能省

倒圆角的参考对象不只是“若干边”：

- edge chain
- 两组面
- full round 三组面

它们的参数解释方式完全不同，因此必须有独立的 `referenceMode`。

### 5.3 `crossSection` 不能省

SW 与 Creo 都支持不止一种截面语义：

- circular
- conic
- curvature continuous

如果不保留这个维度，conic 与 C2 连续特征会被错误归并成普通圆角。

### 5.4 `radiusItems` 不能省

SW variable fillet 与 Creo variable round 都依赖控制点集合。  
只保留 `defaultRadius` 无法表达变半径。

### 5.5 厂商扩展字段必须存在

如果把 SW / Creo 的所有行为参数都直接灌进公共主结构，会导致：

- 公共层语义污染
- builder / accessor 复杂度过高
- TinyXML schema 过早绑定本地实现细节

因此必须保留扩展区。

---

### 5.6 为什么 `firstEndFaceMarker` 放在公共结构里

`firstEndFaceMarker` 虽然不是所有倒圆角都需要，但在以下场景中属于跨系统公共语义：
- 非对称 `D1-D2 Conic`
- 非对称 `D1-D2 CurvatureContinuous`

这些场景里，`D1` 与 `D2` 不是无方向的两个数值，而是分别绑定到两侧端面。
如果统一结构不保存这个侧别信息：
- 读侧可以提取 `D1 / D2`
- 写侧却无法稳定判断哪一侧应回写为 `D1`

因此：
- `firstEndFaceMarker` 进入公共结构
- 但它是“按条件出现”的公共字段
- 不能稳定映射时，不写入 XML

### 7.7 `firstEndFaceMarker` 的 Creo 提取规则

当前 Creo 读侧对 `firstEndFaceMarker` 的统一提取规则如下：

1. 仅在以下条件满足时尝试提取：
   - `params.isAsymmetric = true`
   - `params.crossSection = CONIC` 或 `params.crossSection = CURVATURE_CONTINUOUS`
   - 能稳定取到 `D1 / D2`
2. 先找到 fillet 的主引用边。
3. 再找到该引用边的两个相邻端面，记为候选面 `face1 / face2`。
4. 在每个候选端面的 contour 上，只保留同时满足以下条件的 edge：
   - 该 edge 属于当前 fillet feature 自己生成的几何
   - 该 edge 与原引用边近似平行
5. 对每个候选端面，从这些生成 edge 中选出“到原引用边偏移距离最小”的那一条，记为：
   - `generated_edge_face1`
   - `generated_edge_face2`
6. 分别计算这两条生成 edge 的中点到原引用边中点的几何偏移距离：
   - `distance_face1`
   - `distance_face2`
7. 用 `D1` 做严格判面：
   - 比较 `|distance_face1 - D1|` 与 `|distance_face2 - D1|`
   - 哪一侧偏差更小，哪一侧就是“第一端面”
8. 确定第一端面后，再遍历该端面的 contour，取其模型空间轮廓包围盒中心，写入 `firstEndFaceMarker`。

可将这条规则概括为：

```text
first_end_face = argmin( |distance(face_i) - D1| )
```

这条规则的关键点是：
- 不再使用“优先 `face1`，失败再 `face2`”的顺序策略
- 不再依赖 `face1_dir / face2_dir` 一类启发式方向判断
- 而是显式比较候选端面与 `D1` 的几何对齐关系

当前验证结论：
- 已在 `round_test_2.prt` 上验证：
  - `FB-5`: `D1 = 20`, `D2 = 40`, `distance_face1 = 20`, `distance_face2 = 40`
  - `FB-6`: `D1 = 40`, `D2 = 20`, `distance_face1 = 40`, `distance_face2 = 20`
  - 因此两者都能通过 `D1` 对齐稳定选出第一端面，并得到 `FirstEndFaceMarker="(115.241,0,-57.8927)"`
- 已在 `round_test_5.prt` 上验证：
  - `FB-3`: `D1 = 6`, `D2 = 12`, `distance_face1 = 6`, `distance_face2 = 12`
  - `FB-4`: `D1 = 5`, `D2 = 10`, `distance_face1 = 5`, `distance_face2 = 10`
  - `FB-5`: `D1 = 20`, `D2 = 10`, `distance_face1 = 20`, `distance_face2 = 10`
  - `FB-6`: `D1 = 20`, `D2 = 10`, `distance_face1 = 20`, `distance_face2 = 10`

如果以下任一条件不满足，则该字段不写入：
- 无法稳定取到原引用边
- 无法稳定取到两侧候选端面
- 无法在候选端面上稳定识别属于当前 fillet 的生成 edge
- 无法计算候选端面对原引用边的稳定偏移距离
- 两侧距离无法与 `D1 / D2` 建立稳定对应关系

### 7.8 `firstEndFaceMarker` 的 SW 映射约束

SW 侧如果同样能稳定确认 `D1` 对应端面，也可写入：

```cpp
firstEndFaceMarker
```

但统一规则保持保守：
- 没有稳定公共侧别证据时，不写入
- 不允许为了填字段而构造仅 SW 本地可解释的私有标记点

### 7.9 `CHORDAL` 的当前承接边界

当前统一结构对 `CHORDAL` 的处理是：
- Creo 侧可读
- Creo 侧可写
- SW 侧仍无稳定公共对位

因此 v1 的结论不是“完全三方公共”，而是：
- `FilletMode::CHORDAL` 已进入统一结构
- 但当前属于“Creo 可往返、SW 暂不直映射”的受控能力

当前 `CHORDAL` v1 承接方式：
- `mode = CHORDAL`
- `referenceMode` 仍按现有主参考方式映射
- 读侧当前以 `defaultRadius` / `radiusItems[].radius1` 承接主数值
- 写侧当前按 Creo `PRO_ROUND_TYPE_CHORDAL` 回写

验证证据：
- `round_test_3.prt` 中 `feature 141`
- 原始 `fillet_shape_code = 262144`
- 本地 SDK `ProRound.h` 中 `PRO_ROUND_TYPE_CHORDAL = (1<<18)`
- 现已完成 `Creo -> CADExchange -> Creo` 重建

## 9. 数值映射规则

### 9.1 长度统一

所有半径、距离类字段在进入统一结构前，应统一换算到 `CADExchange` 内部标准单位。

包括：

- `defaultRadius`
- `defaultRadius2`
- `radiusItems[].radius1`
- `radiusItems[].radius2`
- `conicValue` 中属于长度语义的情况

### 9.2 控制点位置统一

建议统一把 `position` 定义为：

- 沿参考边 `0..1` 的比例参数

如果 SW 与 Creo 对同一拓扑边的方向不一致，则必须做方向归一化。

推荐规则：

```cpp
mappedPosition = sameEdgeDirection ? srcPosition : (1.0 - srcPosition);
```

否则会导致控制点位置镜像翻转。

### 9.3 conic 值不能裸存

`conicValue` 不能只保留一个 `double`。

必须连同：

- `conicValueMode`

一起保存。

原因是 SW 侧可能表示：

- `rho`
- `radius`

Creo 侧则可能还叠加：

- `conic dep option`

如果只保留数值本身，会丢失解释方式。

## 10. v1 边界与降级策略

### 10.1 可稳定承接

- SW `Simple`
- SW `VariableRadius`
- SW `Face`
- SW `FullRound`
- Creo `CONSTANT`
- Creo `VARIABLE`
- Creo `FULL`
- Creo `CHORDAL`
- Creo `REF_EDGE`
- Creo `REF_SURF_SURF`
- asymmetric / dual-leg
- conic
- curvature continuous
- `firstEndFaceMarker`（仅非对称 Conic / CurvatureContinuous，且 `D1` 端面可稳定确认时）

### 10.2 暂不稳定承接

- Creo `THROUGH_CURVE`
- Creo `EDGE_SURF`
- Creo `EDGE_EDGE`
- Creo `transition`
- Creo `spine`
- SW `partial-edge`

### 10.3 推荐处理方式

对于暂不承接的能力，建议：

1. 读侧保留必要扩展字段
2. 公共主结构不强行吸收
3. 写回不支持时明确标记 `unsupported`
4. 不要静默降级成普通 constant fillet

## 12. 最终结论

对 SW 与 Creo 来说，倒圆角统一结构的最小合理形态不是：

- `edges + radius`

而应至少包含：

- `mode`
- `referenceMode`
- `crossSection`
- `defaultRadius / defaultRadius2`
- `radiusItems`
- `references / side1Faces / side2Faces / centerFaces`
- `firstEndFaceMarker`（按条件出现）
- `vendor extensions`

这套 `CFillet v1` 结构的目标不是一次性覆盖所有 GUI 细节，而是在不污染 `CADExchange` 主模型的前提下，先稳定承接 SW 与 Creo 倒圆角最重要、最常见、最可验证的跨系统语义。
