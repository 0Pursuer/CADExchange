# 旋转特征跨系统参数调研

## 1. 目的

本文整理 SolidWorks、Creo、UG/NX 三套系统中旋转特征（Revolve）的官方 API 参数与建模方式，用于后续扩展 `CADExchange` 的统一数据结构、读写桥接和 round-trip 验证逻辑。

本文只做参数归纳和能力对照，不直接拍板统一结构。

## 2. 当前仓库现状

仓库里已经有最小版旋转特征定义，但覆盖范围明显偏窄：

- [UnifiedFeatures.h](E:/MyProject/CADExchange/core/UnifiedFeatures.h)
- [RevolveBuilder.h](E:/MyProject/CADExchange/service/builders/RevolveBuilder.h)
- [RevolveAccessor.h](E:/MyProject/CADExchange/service/accessors/RevolveAccessor.h)
- [TinyXMLSerializer.cpp](E:/MyProject/CADExchange/service/serialization/TinyXMLSerializer.cpp)

当前 `CRevolve` 仅包含：

- `profileSketchID`
- `axis.referenceLocalID / referenceEntity / origin / direction`
- `angleKind`
- `primaryAngle`
- `secondaryAngle`

这只能覆盖“草图 + 旋转轴 + 单/双向角度”的最小场景，无法完整表达：

- boss/cut/surface/sheet
- thin revolve
- 到参考终止
- multibody scope
- trim/cap
- helper point
- open profile 智能补闭

## 3. 横向结论

三套系统对旋转特征的共同点是：

- 都需要轮廓或 section
- 都需要旋转轴
- 都支持角度或某种范围定义
- 都会区分实体与非实体结果

但真正影响统一结构设计的差异在于：

- `输出类型` 并不只是 solid，还包括 surface/sheet
- `布尔语义` 并不只是 add/remove，NX 还有 `intersect/sew`
- `终止方式` 不只是数值角度，还包括 `to reference / until selected / through all / symmetric`
- `旋转轴来源` 不只是显式向量，还可能是内部轴、edge、datum axis、sketch centerline
- `薄特征` 在三套系统里的建模方式不同
- `作用范围` 在 SW/NX 中是明确的一等参数

因此，统一模型至少要把参数按“能力桶”拆开，而不是继续只保留一个简化角度模型。

## 4. SolidWorks

### 4.1 官方入口

SolidWorks 的 revolve 主要分成：

- 创建接口：`IFeatureManager::FeatureRevolve2`
- 读取/编辑接口：`IRevolveFeatureData2`

官方资料：

- `FeatureRevolve2`  
  https://help.solidworks.com/2011/english/api/sldworksapi/SolidWorks.Interop.sldworks~SolidWorks.Interop.sldworks.IFeatureManager~FeatureRevolve2.html?format=P&value=
- `IRevolveFeatureData2 Properties`  
  https://help.solidworks.com/2024/english/api/sldworksapi/solidworks.interop.sldworks~solidworks.interop.sldworks.irevolvefeaturedata2_properties.html
- `SetWallThickness`  
  https://help.solidworks.com/2024/english/api/sldworksapi/SOLIDWORKS.Interop.sldworks~SOLIDWORKS.Interop.sldworks.IRevolveFeatureData2~SetWallThickness.html
- `GetAxisType`  
  https://help.solidworks.com/2012/English/api/sldworksapi/SolidWorks.Interop.sldworks~SolidWorks.Interop.sldworks.IRevolveFeatureData2~GetAxisType.html?format=P&value=

### 4.2 主要参数维度

#### 4.2.1 输出类型

通过创建参数组合可区分：

- `IsSolid`
- `IsCut`

因此 SolidWorks revolve 至少覆盖：

- solid boss revolve
- revolve cut
- 非实体旋转场景

#### 4.2.2 轮廓

`IRevolveFeatureData2` 明确暴露：

- `Contours`

这说明 SW 不应被简化成“只绑定一个草图 ID”，而应允许：

- 一个草图中的一个或多个 contour
- contour 选择语义

#### 4.2.3 轴

`IRevolveFeatureData2` 明确暴露：

- `Axis`
- `GetAxisType()`

官方文档明确轴类型可来自：

- datum axis
- edge
- sketch segment

这意味着统一模型里，轴来源必须可区分“拓扑引用”和“草图引用”，不能只剩向量。

#### 4.2.4 终止/范围

`FeatureRevolve2` 的参数可区分：

- `SingleDir`
- `Dir1Type`
- `Dir2Type`
- `Dir1Angle`
- `Dir2Angle`
- `ReverseDir`
- `BothDirectionUpToSameEntity`

这说明 SW revolve 至少支持：

- 单向旋转
- 双向旋转
- 双向共用同一终止参考
- 数值角度
- up-to 类终止

#### 4.2.5 薄特征

SW 明确支持 thin revolve：

- `IsThin`
- `ThinType`
- `ThinThickness1`
- `ThinThickness2`
- `ThinWallType`

这不是一个简单布尔位，而是一整组厚度与壁类型参数。

#### 4.2.6 偏置

`FeatureRevolve2` 明确包含：

- `OffsetReverse1`
- `OffsetReverse2`
- `OffsetDistance1`
- `OffsetDistance2`

因此 unified revolve 若完全忽略 offset，会损失 SW 能力。

#### 4.2.7 作用域与多实体

SW revolve 明确支持：

- `Merge`
- `FeatureScope`
- `AutoSelect`
- `AssemblyFeatureScope`
- `AutoSelectComponents`
- `PropagateFeatureToParts`

这说明 SW 在 multibody / assembly scope 上有显式参数，而不是只靠隐式当前 body。

### 4.3 SolidWorks 参数分组总结

SolidWorks revolve 至少应拆成以下参数桶：

- profile / contours
- axis reference + axis source type
- output kind
- direction / extent
- thin wall
- offset
- merge / feature scope / assembly scope

## 5. Creo

### 5.1 官方入口

Creo revolve 的核心不是单个创建函数，而是 element tree。

官方与本机资料：

- 官方说明页  
  https://support.ptc.com/help/creo_toolkit/protoolkit_plus/usascii/creo_toolkit/user_guide/Element_Trees_Extrude_and_Revolve.html
- 本机头文件  
  `E:\Application\Creo\install_path\CREO7\Creo 7.0.0.0\Common Files\protoolkit\includes\ProRevolve.h`

### 5.2 Creo revolve 覆盖的特征形式

`ProRevolve.h` 明确列出同一棵 revolve tree 可表达：

- Solid Protrusion
- Thin Protrusion
- Solid Cut
- Thin Cut
- Surface
- Surface Trim
- Thin Surface Trim
- Solid Protrusion as First Feature
- Thin Protrusion as First Feature
- Sheet Metal unattached wall

Creo 是三套系统里语义覆盖最宽的一套之一。

### 5.3 共用核心元素

#### 5.3.1 结果类型

- `PRO_E_EXT_SURF_CUT_SOLID_TYPE`
  - `PRO_REV_FEAT_TYPE_SOLID`
  - `PRO_REV_FEAT_TYPE_SURFACE`

#### 5.3.2 材料增减

- `PRO_E_REMOVE_MATERIAL`
  - `PRO_REV_MATERIAL_ADD`
  - `PRO_REV_MATERIAL_REMOVE`

#### 5.3.3 轮廓

- `PRO_E_STD_SECTION`
- first feature 时改为 `PRO_E_SKETCHER`

这意味着 first feature revolve 不是普通 revolve 的完全同构版本。

#### 5.3.4 轴

- `PRO_E_REVOLVE_AXIS`
- `PRO_E_REVOLVE_AXIS_OPT`
  - `PRO_REV_AXIS_EXT_REF`
  - `PRO_REV_AXIS_INT_REF`

外部轴允许的引用类型包括：

- Axis
- Straight Edge
- Straight Curve
- Coordinate System Axis

因此 Creo 明确要求轴模型支持：

- internal axis
- external axis reference
- 轴引用对象类型

#### 5.3.5 方向

- `PRO_E_STD_DIRECTION`
  - `PRO_REV_CR_IN_SIDE_ONE`
  - `PRO_REV_CR_IN_SIDE_TWO`

#### 5.3.6 角度/终止

Creo revolve 的角度通过 compound element 建模：

- `PRO_E_REV_ANGLE`
  - `PRO_E_REV_ANGLE_FROM`
    - `TYPE`
    - `VAL`
    - `REF`
  - `PRO_E_REV_ANGLE_TO`
    - `TYPE`
    - `VAL`
    - `REF`

类型枚举：

- `PRO_REV_ANG_FROM_ANGLE`
- `PRO_REV_ANG_FROM_REF`
- `PRO_REV_ANG_FROM_NONE`
- `PRO_REV_ANG_TO_ANGLE`
- `PRO_REV_ANG_TO_REF`
- `PRO_REV_ANG_SYMMETRIC`

这说明 Creo 的终止方式至少包括：

- from-angle
- to-angle
- from-ref
- to-ref
- symmetric

可作为角度参考的类型包括：

- point
- edge start / edge end
- curve start / curve end
- plane

如果是 datum plane，文档还特别说明需要通过 selection point 消解多解。

### 5.4 薄特征

Creo 薄旋转的参数远比当前 CADExchange 结构丰富：

- `PRO_E_FEAT_FORM_IS_THIN`
- `PRO_E_THICKNESS`
- `PRO_E_STD_MATRLSIDE`
- `PRO_E_FEAT_THIN`
  - `PRO_E_FEAT_THIN_STRT_OPT`
  - `PRO_E_FEAT_THIN_STRT_REF`
  - `PRO_E_FEAT_THIN_END_OPT`
  - `PRO_E_FEAT_THIN_END_REF`

即使统一结构暂不支持 full-fidelity，也必须预留：

- 是否 thin
- thickness
- material side
- start/end thin solution 参考

### 5.5 Surface / Trim / Cap

Creo 明确支持：

- `PRO_E_SRF_END_ATTRIBUTES`
  - `OPEN`
  - `CAPPED`
- `PRO_E_TRIM_QUILT`
- `PRO_E_TRIM_QLT_SIDE`

这部分在当前 `CRevolve` 完全缺失。

### 5.6 Body 范围

Creo 用 body 元素显式表达多体作用范围：

- `PRO_E_BODY`
- `PRO_E_BODY_USE`
- `PRO_E_BODY_SELECT`

不同场景可出现：

- 新建 body
- 加到单个选中 body
- cut all bodies
- cut selected bodies

### 5.7 Sheet Metal 特有项

Creo revolve 还可落到 sheet metal unattached wall：

- `PRO_E_STD_SMT_THICKNESS`
- `PRO_E_STD_SMT_SWAP_DRV_SIDE`
- `PRO_E_SMT_WALL_SHARPS_TO_BENDS`
- `PRO_E_SMT_FILLETS`
- `PRO_E_SMT_DEV_LEN_CALCULATION`
- `PRO_E_SMT_MERGE_DATA`

如果统一结构第一版不支持 sheet metal，应在文档里明确列为 out-of-scope，而不是静默丢弃。

### 5.8 Creo 参数分组总结

Creo revolve 至少应拆成：

- feature kind: solid / surface / trim / sheet-metal
- add/remove
- section / first-feature section
- axis ref + axis opt
- side direction
- from/to extent
- thin definition
- trim/cap definition
- body scope
- helper point / selection location

### 6.2 主 builder 参数

`RevolveBuilder` 一级参数包括：

- `Section`
- `Axis`
- `Offset`
- `BooleanOperation`
- `Tolerance`
- `Limits`
- `SmartVolumeProfile`
- `FeatureOptions`
- `StartLimitHelperPoint`
- `EndLimitHelperPoint`

### 6.3 输出类型与布尔

#### 6.3.1 体类型

`FeatureOptions.BodyType`：

- `Solid`
- `Sheet`

#### 6.3.2 BooleanOperation

NX boolean 类型明确包括：

- `Create`
- `Unite`
- `Subtract`
- `Intersect`
- `Sew`

并可指定：

- `TargetBodies`

相比 SW/Creo，NX 的布尔类型明显更宽。

### 6.4 Limit / Extend 体系

NX revolve 的范围不是简单的 angle 对，而是两侧 `Limits`：

- `StartExtend`
- `EndExtend`
- `SymmetricOption`

每个 extend 的 `TrimType` 可取：

- `Value`
- `UntilNext`
- `UntilSelected`
- `UntilExtended`
- `ThroughAll`
- `Symmetric`
- `Percent`
- `ArcLength`
- `ThruPoint`

并配套：

- `Target`
- `Value`

这说明 NX revolve 的终止表达能力比当前 `angleKind + angle1 + angle2` 强很多。

### 6.5 Offset

`FeatureOffset` 明确支持：

- `NoOffset`
- `NonsymmetricOffset`
- `SymmetricOffset`
- `SingleOffset`

并有：

- `StartOffset`
- `EndOffset`

如果后续要支持 NX revolve，offset 不能继续缺席。

### 6.6 Open Profile Smart Volume

`SmartVolumeProfileBuilder` 提供：

- `OpenProfileSmartVolumeOption`
- `CloseProfileRule`
  - `FCI`
  - `LCI`
  - `CCI`

这是一类当前 CADExchange 里完全没有的语义：开口轮廓自动延展并寻找闭合。

### 6.7 Helper Point

`SetStartLimitHelperPoint` / `SetEndLimitHelperPoint` 用于：

- 当 until-selected 与旋转体发生多次相交时，指定靠近哪个交点解算

这与 Creo 里“plane through axis 时需 selection point 消解多解”的问题非常相似，说明 unified 层最好预留“解消歧位置”语义。

### 6.8 NX 参数分组总结

NX revolve 至少应拆成：

- section
- axis
- body type
- boolean type + target bodies
- start/end limits
- offset
- tolerance
- smart volume profile
- helper points

## 7. 三套系统的共性与差异矩阵

| 参数维度 | SolidWorks | Creo | UG/NX |
|---|---|---|---|
| 轮廓 | sketch contour | `STD_SECTION` / `SKETCHER` | `Section` |
| 轴来源 | datum axis / edge / sketch segment | internal axis / external axis / edge / curve / csys axis | `Axis` |
| 输出体类型 | solid 为主，也可非 solid | solid / surface | solid / sheet |
| 布尔 | boss / cut | add / remove | create / unite / subtract / intersect / sew |
| 单/双向 | 有 | 有 | 通过 limits |
| 对称 | 有 | 有 | `SymmetricOption` |
| 到参考 | 有 | 有 | 有 |
| through all / until next | 有部分 up-to 语义 | 主要是 angle/ref | 明确支持 |
| thin | 明确支持 | 明确支持 | 不以“thin revolve”命名，但可通过其他组合实现类似能力 |
| trim/cap | 不突出 | 明确支持 | 通过 sheet/sew/limit 组合 |
| multibody scope | 明确支持 | body use/select | target bodies |
| helper point | 不突出 | selection point 消歧 | 明确支持 |

## 8. 对统一结构设计的直接约束

这部分不是最终结构设计，只是对后续建模的约束总结。

### 8.1 必须独立建模的桶

后续 unified revolve 至少要能独立表达：

- `profile/section`
- `axis`
- `output body kind`
- `boolean operation`
- `extent / limits`
- `thin definition`
- `scope / target bodies`
- `surface / trim / cap options`
- `helper point / disambiguation point`

### 8.2 必须区分的引用来源

以下来源不能都压扁成坐标和向量：

- sketch centerline
- edge
- datum axis
- internal axis
- curve
- plane / point reference for extent

### 8.3 当前模型的缺口

当前 `CRevolve` 缺少：

- boolean kind
- output body kind
- limit kind
- reference-based extent
- thin parameters
- scope
- trim/cap
- helper point
- offset

因此下一步如果要设计统一数据结构，建议先定义“能力分层”，再决定哪些字段进入 MVP，哪些字段放扩展层。

## 9. 建议的后续工作顺序

1. 先定义 revolve 的参数分层，不直接写最终字段名
2. 再划分 `MVP` 与 `full fidelity`
3. 明确每个 CAD 在 MVP 中哪些参数可保真，哪些只能降级
4. 最后再改 `UnifiedFeatures.h`、builder/accessor、tinyxml schema 和 validator
