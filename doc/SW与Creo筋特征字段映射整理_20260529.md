# SW 与 Creo 筋特征字段映射整理

## 1. 目的

本文单独整理 SolidWorks 与 Creo 两套系统在筋特征（rib）上的：

- 生成方式
- 可提取信息
- 创建所需信息
- 字段级映射关系
- 可直接映射、条件映射、不可直接映射的边界

本文目标不是讨论 GUI 操作流程，而是为后续 `CADExchange` 统一数据结构设计、读写桥接和 round-trip 语义保真提供一份可直接引用的映射说明。

## 3. 总结结论

SW 与 Creo 的筋特征不能压缩成单一的：

- `sketch + thickness`

这种极简结构。

更合理的统一方式是：

1. 先定义一层 `UnifiedRib`
2. SW / Creo 分别映射到 `UnifiedRib`
3. 无法进入公共语义的字段保留在 `vendor extension`

建议统一主结构至少覆盖：

- `section`
- `thickness`
- `thicknessSideMode`
- `materialSide`
- `targetBody`

否则会丢失：

- SW 的 `linear / natural`
- SW 的 `parallel / normal to sketch`
- SW 的 `draft`

当前证据下，Creo rib 的原生参数面明显比 SW 更窄，核心就是：

- 截面
- 厚度
- 厚度分布侧
- 材料侧
- body

而 SW rib 是：

- 草图驱动
- 还叠加了厚度方向、材料方向、筋类型、草图法向模式、拔模、多实体 body

## 4. SolidWorks

## 4.1 生成方式

### 4.1.1 命令式创建

`IFeatureManager` 提供：

- `InsertRib()`
- `InsertRib2()`

其中 `InsertRib2()` 比 `InsertRib()` 多一个 `IsNormToSketch` 参数，因此是更完整的创建入口。

这说明 SW rib 至少存在一条明确的“选择集 + 命令式参数”创建路径。

### 4.1.2 定义对象读改

已有 `IFeature` 时，可走：

1. `IFeature::GetDefinition()`
2. 转为 `IRibFeatureData2`
3. 修改参数
4. `IFeature::ModifyDefinition()`

这说明 SW rib 同时支持“特征数据对象式”的提取与编辑。

### 4.1.3 原生模式枚举

从 `swconst.tlh` 可确认两组 rib 专用枚举：

- `swRibExtrusionDirection_e`
  - `swRibParallelToSketch`
  - `swRibNormalToSketch`
- `swRibType_e`
  - `swRibLinear`
  - `swRibNatural`

这两组枚举很关键，说明 SW rib 不只是“薄拉伸的别名”，而是有自己的原生类型和方向模式。

## 4.2 能提取到的信息

从 `IRibFeatureData2` 可确认当前至少能提取：

- `IsTwoSided`
- `ReverseThicknessDir`
- `Thickness`
- `RefSketchIndex`
- `FlipSide`
- `EnableDraft`
- `DraftOutward`
- `DraftAngle`
- `ExtrusionDirection`
- `Type`
- `Body`
- `DraftFromWall`

此外还存在：

- `AccessSelections()`
- `IAccessSelections()`
- `ReleaseSelectionAccess()`

这说明：

1. rib 的参数不只是一组纯标量
2. 其定义还依赖选择对象
3. 提取引用对象时需要进入 selection access 模式

## 4.3 创建时需要的信息

从 `InsertRib2()` 的签名可直接确认创建参数为：

- `Is2Sided`
- `ReverseThicknessDir`
- `Thickness`
- `ReferenceEdgeIndex`
- `ReverseMaterialDir`
- `IsDrafted`
- `DraftOutward`
- `DraftAngle`
- `IsNormToSketch`

除这些显式参数外，还隐含需要：

- 当前被选中的草图 / 轮廓 / 引用对象

也就是说，SW rib 创建本质上是：

- 选择集
- 再叠加一组 feature 参数

## 4.4 对 SW 语义层的直接判断

当前证据下，SW rib 的参数面至少可以拆成这些参数桶：

- 草图/引用桶
- 厚度桶
- 材料侧桶
- 筋类型桶
- 方向模式桶
- 拔模桶
- body 桶

## 5. Creo

## 5.1 生成方式

Creo rib 的特征类型为：

- `PRO_FEAT_RIB`

创建方式不是单独高层 builder，而是标准 element tree + `ProFeatureCreate()`。

`ProRib.h` 已经直接给出 rib 的 feature element tree 定义，因此 Creo rib 的创建本质是：

1. 组织完整 rib element tree
2. 调 `ProFeatureCreate()`

## 5.2 能提取到的信息

Creo 侧标准提取路径是：

1. `ProFeatureTypeGet()` 确认 `PRO_FEAT_RIB`
2. `ProFeatureElemtreeExtract()` 提取 element tree
3. `ProElemtreeElementGet()` 逐项读取
4. 如需补几何对象，则 `ProFeatureGeomitemVisit()`

按 `ProRib.h` 当前定义，rib 的核心可提取信息至少包括：

- `PRO_E_STD_SECTION`
- `PRO_E_STD_MATRLSIDE`
- `PRO_E_RIB_THICKNESS`
- `PRO_E_RIB_SIDE_OPTS`
- `PRO_E_BODY`

## 5.3 创建时需要的信息

从 `ProRib.h` 的 feature elements table 可确认，创建一个 rib 至少需要组织这些字段：

- `PRO_E_FEATURE_TYPE = PRO_FEAT_RIB`
- `PRO_E_FEATURE_FORM = PRO_EXTRUDE`
- `PRO_E_RIB_SECTION_COMP`
  - `PRO_E_STD_SECTION`
  - `PRO_E_STD_MATRLSIDE`
- `PRO_E_RIB_THICKNESS`
- `PRO_E_RIB_SIDE_OPTS`
- `PRO_E_BODY`
  - 视场景需要

对应枚举也已明确：

- `ProRibSideOption`
  - `PRO_RIB_SYMMETRIC`
  - `PRO_RIB_SIDE_ONE`
  - `PRO_RIB_SIDE_TWO`
- `ProRibSectionSideOption`
  - `PRO_RIB_SEC_SIDE_ONE`
  - `PRO_RIB_SEC_SIDE_TWO`

## 5.4 对 Creo 语义层的直接判断

当前证据下，Creo rib 的高层参数面比 SW 明显更小：

- 截面
- 材料侧
- 厚度
- 厚度分布侧
- body

在 `ProRib.h` 当前公开定义里，没有看到与 SW 下列字段对等的 rib 专用高层参数：

- `linear / natural`
- `parallel / normal to sketch`
- `draft angle / draft outward / draft from wall`

因此这些更像是 SW rib 的原生扩展能力，而不是当前 SW/Creo rib 的稳定公共交集。

## 6. 字段级映射

### 6.1 核心字段映射

| SW 字段/语义 | Unified 字段 | Creo 字段/语义 | 映射状态 | 说明 |
|---|---|---|---|---|
| 选中的草图 / 轮廓 | `section` | `PRO_E_STD_SECTION` | 条件映射 | SW 更依赖选择集，Creo 直接挂在 element tree |
| `Thickness` | `thickness` | `PRO_E_RIB_THICKNESS` | 可直接映射 | 最稳定公共能力 |
| `IsTwoSided = true` | `thicknessSideMode = Symmetric` | `PRO_RIB_SYMMETRIC` | 条件映射 | SW 是布尔语义，Creo 是枚举语义 |
| `IsTwoSided = false` + `ReverseThicknessDir = false/true` | `thicknessSideMode = OneSideA/OneSideB` | `PRO_RIB_SIDE_ONE / SIDE_TWO` | 条件映射 | 需要统一“哪边算 A/B” |
| `FlipSide` / `ReverseMaterialDir` | `materialSide` | `PRO_E_STD_MATRLSIDE` | 条件映射 | 语义接近，但 SW 命名更偏“翻转”，Creo 更偏“截面材料侧” |
| `Body` | `targetBody` | `PRO_E_BODY` | 可直接映射 | 多 body 目标语义基本对齐 |

### 6.2 SW 扩展字段

| SW 字段/语义 | Unified 建议 | Creo 对位 | 映射状态 | 说明 |
|---|---|---|---|---|
| `Type = swRibLinear` | `swExtension.ribType = Linear` | 无直接字段 | 不可直接映射 | 当前 Creo rib 公开参数面未见对应项 |
| `Type = swRibNatural` | `swExtension.ribType = Natural` | 无直接字段 | 不可直接映射 | 建议保留 vendor extension |
| `ExtrusionDirection = swRibParallelToSketch` | `swExtension.extrusionDirection = ParallelToSketch` | 无直接字段 | 不可直接映射 | 当前 Creo 侧未见 rib 专用顶层字段 |
| `ExtrusionDirection = swRibNormalToSketch` | `swExtension.extrusionDirection = NormalToSketch` | 无直接字段 | 不可直接映射 | 同上 |
| `EnableDraft` | `swExtension.draft.enabled` | 无直接字段 | 不可直接映射 | 需保留 SW 扩展 |
| `DraftAngle` | `swExtension.draft.angle` | 无直接字段 | 不可直接映射 | 需保留 SW 扩展 |
| `DraftOutward` | `swExtension.draft.outward` | 无直接字段 | 不可直接映射 | 需保留 SW 扩展 |
| `DraftFromWall` | `swExtension.draft.fromWall` | 无直接字段 | 不可直接映射 | 需保留 SW 扩展 |
| `ReferenceEdgeIndex` | `swExtension.referenceEdgeIndex` | 无直接顶层字段 | 不可直接映射 | 更像 SW 选择语义的一部分 |
| `RefSketchIndex` | `swExtension.refSketchIndex` | 无直接顶层字段 | 不可直接映射 | 更像 SW 草图/引用定位语义 |

## 7. 建议统一字段

```cpp
enum class RibThicknessSideMode {
  Symmetric,
  OneSideA,
  OneSideB,
  Unknown
};

enum class RibMaterialSide {
  SideA,
  SideB,
  Unknown
};

struct UnifiedRib {
  std::string sectionSketchID;

  double thickness = 0.0;
  RibThicknessSideMode thicknessSideMode = RibThicknessSideMode::Unknown;
  RibMaterialSide materialSide = RibMaterialSide::Unknown;

  std::optional<RefEntity> targetBody;

  Json::Value swExtension;
  Json::Value creoExtension;
};
```

## 8. 对 `CADExchange` 的直接建议

### 8.1 第一版 unified rib 只吃公共交集

第一版主结构建议只承接：

- `section`
- `thickness`
- `thicknessSideMode`
- `materialSide`
- `targetBody`

这样最稳。

### 8.2 SW 特有能力不要硬塞进公共字段

当前证据下，这些字段应优先放进 `swExtension`：

- `ribType`
- `extrusionDirection`
- `referenceEdgeIndex`
- `refSketchIndex`
- `draft.*`

否则统一层字段会被 SW 语义绑架，而 Creo 侧没有可靠原生对位。

### 8.3 Creo 侧更适合 element tree 驱动

后续如果实现 Creo rib：

- 读取：`ProFeatureTypeGet` + `ProFeatureElemtreeExtract`
- 创建：直接组 rib element tree 后 `ProFeatureCreate`

不要先假设存在像 SW `IRibFeatureData2` 一样的高层 rib 参数对象。

### 8.4 SW 侧更适合 feature-data 驱动

后续如果实现 SW rib：

- 读取：`IFeature::GetDefinition()` -> `IRibFeatureData2`
- 创建：优先 `InsertRib2()`

`InsertRib2()` 当前是本机类型库里最明确的 rib 创建入口。

## 10. 结论

当前证据下，SW 与 Creo rib 的真正稳定公共交集并不大。

最稳的公共语义是：

- 截面
- 厚度
- 厚度分布侧
- 材料侧
- body

而 SW 额外公开了：

- `linear / natural`
- `parallel / normal to sketch`
- `draft`

这些字段当前不应强行提升为 SW/Creo 的共同主字段。

因此，对 `CADExchange` 更合理的落地方式是：

1. 先定义一个小而稳的 `UnifiedRib`
2. SW 扩展字段单独保留
3. Creo 继续按 element tree 原生语义实现

这能最大限度降低后续 round-trip 时的错误降级与语义丢失。
