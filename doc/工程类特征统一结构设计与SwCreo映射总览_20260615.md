# 工程类特征统一结构设计与 Sw/Creo 映射总览

## 1. 文档目的

本文从 `CADExchange` 统一模型视角，系统整理以下五类工程类特征：

- 倒角（Chamfer）
- 倒圆角（Fillet / Round）
- 筋（Rib）
- 壳（Shell）
- 拔模（Draft）

目标是统一回答以下问题：

1. 这些特征在 `CADExchange` 中的统一结构是什么
2. `Creo <-> CADExchange <-> SolidWorks` 如何进行映射
3. 原生 CAD 参数如何被转换并落入 `CADExchange`
4. 历史上在提取与创建阶段出现过哪些问题，当前如何处理
5. 哪些特征已经比较稳定，哪些仍存在“设计口径”和“代码口径”差异

本文是总览文档。每类特征的细节以各自专题文档为准。

## 2. 统一设计总原则

这五类工程特征虽然几何行为不同，但在统一建模层遵循相同原则。

### 2.1 先统一语义，再做 CAD 桥接

统一链路始终应理解为：

- `SW -> CADExchange -> Creo`
- `Creo -> CADExchange -> SW`

桥接实现不应理解为：

- `SW` 直接理解 `Creo`
- `Creo` 直接理解 `SW`

而应理解为：

- 两端都先收敛到 `CADExchange`
- 再由目标 CAD 使用自己的稳定 API 重建

### 2.2 统一结构优先保存“参数语义”，不直接镜像原生字段名

例如：

- 倒圆角不再以 `DefaultRadius / Distance / Distance2` 为统一本体，而是收敛为：
  - `driveType`
  - `primaryValue`
  - `secondValue`

- 倒角统一为：
  - `ChamferMode`
  - `CChamferParams`

- 壳统一为：
  - `thickness`
  - `direction`
  - `facesToRemove`
  - `thicknessFaces`

### 2.3 拓扑引用尽量保存“可重建语义”，不依赖不稳定拓扑编号

统一层当前倾向保存：

- `CRefFace`
- `CRefEdge`
- `CRefEntityBase`
- 面/边几何指纹
- 派生语义点（如 `firstEndFaceMarker`）

而不是把目标放在跨 CAD 完全保留原生拓扑编号上。

### 2.4 跨系统允许受控降级，但不能隐式改义

例如：

- Creo chordal fillet 可降级为统一层的单距离驱动
- SW `D1=D2` 非对称圆角也可归一化为单距离驱动
- SW rib 高级选项可保存但不承诺被 Creo 消费
- Step Draft / Variable Draft 可以降级，但必须显式记录

### 2.5 要区分“设计文档口径”和“当前代码口径”

这一点在：

- `Rib`
- `Draft`

上尤其重要。

当前仓库中有些专题文档已经给出了更理想、更完整的统一设计，但 [UnifiedFeatures.h](D:\CADHHU\CADProj\CADExchange\core\UnifiedFeatures.h:1) 中的实际落地结构仍处于阶段性版本。

因此本文会对每类特征分别标出：

- 设计口径
- 当前代码口径

## 4. 五类特征总对比表

| 特征 | 统一建模核心问题 | SW 侧难点 | Creo 侧难点 | 当前主要风险 |
|---|---|---|---|---|
| 倒角 | 参数模式归一 | `EqualDistance` API 不稳定、`swType=0` | schema 分流较多 | 第一端面语义对齐 |
| 倒圆角 | 驱动方式归一 + shape 保真 + 控制点方向归一 | 非对称 `D1/D2`、变圆角点位 | chordal / D1XD2 / conic / C2 | `ConicValue`、控制点位置、端面语义 |
| 筋 | 厚度侧 / 材料侧 / 草图局部坐标系 | 方向投影、草图法向依赖 | `PRO_RIB_SIDE_*` 语义反直觉 | 代码口径与设计口径不一致 |
| 壳 | 默认厚度 + 开口面 + 多厚度面 | 选择集 mark 驱动 | `PRO_E_SHELL_FLIP` 语义反直觉 | 面引用解析稳定性 |
| 拔模 | 构造参考 + 方向 + 双侧/分割 | 类型覆盖不完整、选择集复杂 | Hinge/Split 组合多 | 当前代码仍是最小子集 |

## 5. 倒角（Chamfer）

专题文档：

- [倒角特征统一结构与SwCreo转换过程_20260511.md](D:\CADHHU\CADProj\CADExchange\doc\倒角特征统一结构与SwCreo转换过程_20260511.md:1)
- [三系统倒角跨系统参数调研_20260504.md](D:\CADHHU\CADProj\CADExchange\doc\三系统倒角跨系统参数调研_20260504.md:1)

### 5.1 CADExchange 统一结构

当前倒角统一结构核心为：

- `ChamferMode`
  - `EQUAL_DISTANCE`
  - `TWO_DISTANCES`
  - `TWO_OFFSETS`
  - `DISTANCE_ANGLE`
  - `VERTEX_3DISTANCES`

- `CChamferParams`
  - `distance1`
  - `distance2`
  - `distance3`
  - `offset1`
  - `offset2`
  - `angle`

- `CChamfer`
  - `mode`
  - `params`
  - `references`
  - `firstEndFaceMarker`

### 5.2 参数统一思想

倒角的核心不是保留原生 CAD 字段名，而是保留“当前倒角如何被驱动”的统一语义。

统一规则如下：

| 统一模式 | 有效字段 | 语义 |
|---|---|---|
| `EQUAL_DISTANCE` | `distance1` | 等距倒角 |
| `TWO_DISTANCES` | `distance1 + distance2` | 双距离倒角 |
| `TWO_OFFSETS` | `offset1 + offset2` | 双偏移倒角 |
| `DISTANCE_ANGLE` | `distance1 + angle` | 距离 + 角度 |
| `VERTEX_3DISTANCES` | `distance1 + distance2 + distance3` | 顶点三边倒角 |

关键约定：

- `EqualDistance` 在统一模型中只保留 `distance1`
- 不再冗余写 `distance2 = distance1`

### 5.3 Creo / CADExchange / SW 映射总表

| 原生语义 | CADExchange | 目标语义 |
|---|---|---|
| SW `swChamferEqualDistance` | `EQUAL_DISTANCE + distance1` | Creo `PRO_CHM_D_X_D` |
| SW `swChamferDistanceDistance` | `TWO_DISTANCES + distance1 + distance2` | Creo `PRO_CHM_D1_X_D2` |
| SW `swChamferAngleDistance` | `DISTANCE_ANGLE + distance1 + angle` | Creo `PRO_CHM_45_X_D / PRO_CHM_ANG_X_D` |
| SW `swChamferVertex` | `VERTEX_3DISTANCES` | Creo `PRO_FEAT_CORN_CHAMF` |
| Creo `PRO_CHM_O_X_O / O1_X_O2` | `TWO_OFFSETS + offset1 + offset2` | SW 无原生直等价，需按统一参数重建策略处理 |

### 5.4 提取阶段问题

#### SW 侧

1. `swType=0`
   - UI 上有倒角
   - 但 API `get_Type()` 返回 `0`
   - 当前需要结合：
     - `get_EqualDistance()`
     - `GetEdgeChamferDistance()`
     - `get_EdgeChamferAngle()`
     - `GetVertexChamferDistance()`
   - 做二次推断

2. `FirstEndFaceMarker`
   - 不能依赖 SW 原生某个稳定字段直接得到
   - 需要根据参考边邻接几何与端面关系推断

#### Creo 侧

1. schema 分裂较细
   - 同样是“倒角”，原生 schema 可能是：
     - `D_X_D`
     - `D1_X_D2`
     - `O_X_O`
     - `ANG_X_D`
     - `45_X_D`
     - `CORN_CHAMF`
   - 读侧必须先压平到 `ChamferMode`

### 5.5 创建阶段问题

#### CADExchange -> SW

最重要的坑是：

- 直接用 `swChamferEqualDistance` 创建，UI 可见但 API 回读不稳定

当前稳定策略：

1. 先按 `DistanceDistance` 创建
2. 令 `d1 = d2 = distance1`
3. 创建后读取 definition
4. 显式 `put_EqualDistance(true)`
5. `ModifyDefinition(...)`

#### CADExchange -> Creo

核心问题不是 API 不稳定，而是 schema 必须正确分流，否则会直接建错类型。

### 5.6 当前结论

倒角是五类工程特征中结构最清晰的一类之一。

当前已收敛的关键结论：

- 统一层按参数语义建模，不按原生字段名建模
- `EqualDistance` 只保留一个距离值
- `FirstEndFaceMarker` 是端面语义锚点，不是绝对坐标约束

## 6. 倒圆角（Fillet / Round）

专题文档：

- [倒圆角统一结构优化方案与SwCreo映射.md](D:\CADHHU\CADProj\CADExchange\doc\倒圆角统一结构优化方案与SwCreo映射.md:1)
- [三系统倒圆角跨系统参数调研_20260506.md](D:\CADHHU\CADProj\CADExchange\doc\三系统倒圆角跨系统参数调研_20260506.md:1)
- [SW与Creo倒圆角字段映射整理_20260518.md](D:\CADHHU\CADProj\CADExchange\doc\SW与Creo倒圆角字段映射整理_20260518.md:1)

### 6.1 CADExchange 统一结构

倒圆角当前统一结构核心为：

- `mode`
  - `Constant`
  - `Variable`
  - `FullRound`

- `driveType`
  - `RADIUS`
  - `SINGLE_DISTANCE`
  - `TWO_DISTANCES`

- `primaryValue`
- `secondValue`

- `radiusPoints`
  - `position`
  - `primaryValue`
  - `secondValue`
  - `edgeMidPoint`

- `crossSection`
  - `Circular`
  - `Conic`
  - `CurvatureContinuous`

- `conicValueMode`
- `conicValue`

- `referenceMode`
  - `EdgeChain`
  - `FaceFace`
  - `FullRoundThreeFaces`

- `firstEndFaceMarker`

### 6.2 统一设计核心

倒圆角的第一核心不是 `mode`，而是 `driveType + primaryValue + secondValue`。

统一解释规则：

| `driveType` | `primaryValue` | `secondValue` | 含义 |
|---|---|---|---|
| `RADIUS` | 半径 | 无 | 半径驱动 |
| `SINGLE_DISTANCE` | 单距离 | 无 | 单距离驱动 |
| `TWO_DISTANCES` | 第一距离 | 第二距离 | 双距离驱动 |

这套设计的价值是把：

- Creo chordal
- Creo D1XD2
- SW asymmetric D1/D2

统一到同一驱动框架下。

### 6.3 Creo / CADExchange / SW 映射总表

| 原生语义 | CADExchange | 目标语义 |
|---|---|---|
| Creo 普通半径圆角 | `Radius + PrimaryValue` | SW 对称半径圆角 |
| Creo chordal | `Distances + PrimaryValue` | SW 非对称路径，`D1=D2=PrimaryValue` |
| Creo `D1XD2` | `Distances + PrimaryValue + SecondValue` | SW 非对称 `D1/D2` |
| SW 对称半径圆角 | `Radius + PrimaryValue` | Creo 普通半径圆角 |
| SW 非对称 `D1=D2` | `Distances + PrimaryValue` | Creo chordal |
| SW 非对称 `D1!=D2` | `Distances + PrimaryValue + SecondValue` | Creo `D1XD2` |

### 6.4 参数与 shape 保真规则

倒圆角除了驱动值，还要保真：

- `crossSection`
  - `Circular`
  - `Conic`
  - `CurvatureContinuous`

- `conicValueMode`
- `conicValue`

这部分是历史问题高发区。

### 6.5 提取阶段问题

#### Creo 侧

1. chordal / `D1XD2` / 半径圆角并不共享同一原生语义
   - 必须先统一为 `driveType`

2. `CurvatureContinuous` / `Conic` 需要显式提取 `ConicValue`

3. `tangentPropagation` 历史上曾漏映射

#### SW 侧

1. SW 非对称 `D1/D2`
   - 若 `D1 == D2`
   - 当前统一层将其归一为“单距离”

2. 变半径控制点位置高度依赖边方向
   - 控制点不能只存 `position`
   - 还要能恢复挂靠边

3. `FirstEndFaceMarker`
   - 仍需通过几何规则推断，而不是依赖某个稳定原生字段

### 6.6 创建阶段问题

#### CADExchange -> Creo

要正确区分：

- `Radius`
- `Distances + one value`
- `Distances + two values`
- `Variable`

否则会把：

- chordal
- `D1XD2`
- variable fillet

建成错误 schema。

#### CADExchange -> SW

统一规则是：

- `Radius` -> 对称半径创建
- `Distances` -> 统一走 SW 非对称路径
  - 只有 `PrimaryValue` -> `D1 = D2`
  - 同时有两个值 -> `D1 / D2`

### 6.7 历史问题清单

#### 1. `ConicValue` 被错误按长度缩放

代表案例：

- [Creo圆角141与178重建参数异常排查记录_20260522.md](D:\CADHHU\CADProj\CreoLibs\doc\Creo圆角141与178重建参数异常排查记录_20260522.md:1)

现象：

- `ConicValue` 从无量纲 `0.8`
- 错误变成 `20.32`

根因：

- 把 `ConicValue` 当成长度参与单位缩放

#### 2. `CurvatureContinuous` 的 `ConicValue` 丢失

现象：

- `CrossSection = CURVATURE_CONTINUOUS`
- 但旧序列化只在 `Conic` 时写 `ConicValue`

修复方向：

- `SaveFillet` 对 `CurvatureContinuous` 也输出 `ConicValue / ConicValueMode`

#### 3. 变半径控制点位置方向错误

当前收敛规则：

- 若目标边方向与源边方向相反：
  - `mappedPosition = 1.0 - sourcePosition`
- 否则：
  - `mappedPosition = sourcePosition`

#### 4. `FirstEndFaceMarker` 语义对齐

当前收敛规则：

- 它是端面身份代表点
- 成功标准是“仍落在同一几何面”
- 不是坐标逐值相等

### 6.8 当前结论

倒圆角是五类工程特征中最复杂、问题最多的一类，但也是统一设计最成熟的一类。

已经收敛的关键结论：

- 驱动方式必须独立建模
- `ConicValue` 不是长度
- 变半径控制点位置必须考虑边方向归一化
- `FirstEndFaceMarker` 只能按面语义对齐

也就是说，Rib 的真正难点是“方向解释一致性”，不是简单参数抄写。

### 7.4 当前桥接实现中的实际关注点

虽然当前代码结构还未完全切换到上述设计，但桥接层实际已经集中在以下问题上：

- 草图局部坐标系真实方向
- 厚度偏置向量如何投影回目标 CAD 的 side 枚举
- 材料方向如何通过法向与参考点稳定重建

也就是说：

- 统一模型字段还在演进
- 但方向算法已经相对明确

### 7.5 历史问题清单

#### 1. Datum plane flip 污染 Rib 方向

见：

- [Creo筋特征PRO_RIB参数绝对方向映射记录_20260603.md](D:\CADHHU\CADProj\CreoLibs\doc\Creo筋特征PRO_RIB参数绝对方向映射记录_20260603.md:1)

现象：

- Datum plane 一旦整体法向翻转
- 其上 sketch 与 rib 的局部法向全部连带翻转
- 后续厚度侧、材料侧、拉伸方向全部失真

#### 2. `PRO_RIB_SIDE_ONE / SIDE_TWO` 语义与直觉相反

当前确认：

- `PRO_RIB_SIDE_ONE` 对应 `-N`
- `PRO_RIB_SIDE_TWO` 对应 `+N`

这要求读写两侧都按投影结果选 side，不能按字面猜。

#### 3. 不能只相信统一模型草图方向

跨 CAD 后，目标 CAD 侧实际创建的 sketch 局部坐标系可能与源侧名义方向有偏差。

因此当前稳定策略是：

- 创建时优先读取目标 CAD 实际 sketch csys
- 再把统一模型方向投影回去

#### 4. 双重引用解析逻辑冲突、材料方向鲁棒性不足

这在 git 记录中表现为一系列持续修复：

- 初版 rib 实装
- SW rib 读写补齐
- 由相对参数切换到绝对向量模式
- 修复 SW->Creo rib_test_2 的方向取反
- 加入首创 + 反转重试增强材料方向鲁棒性

### 7.6 当前结论

Rib 是当前最需要做“设计文档与代码结构对齐”的工程类特征。

当前应理解为：

- 文档已经定义了更规范的公共主干 + SW 扩展块方案
- 代码当前仍保留绝对方向表达
- 桥接实现的真正核心是方向投影与 local csys 对齐

### 8.3 提取阶段问题

     - 每个面的厚度值
   - 组合成 `CShellThicknessFace`

#### Creo 侧

1. `PRO_E_SHELL_FLIP` 语义反直觉

当前已确认：

- `PRO_B_TRUE` 表示内侧
- `PRO_B_FALSE` 表示外侧

这是壳特征最重要的坑。

2. 多厚度面是 compound list
   - 不是平铺字段
   - 需要遍历 `PRO_E_ST_SHELL_LOCL_LIST`

### 8.4 创建阶段问题

#### CADExchange -> SW

SW 创建非常依赖选择集 mark：

- 移除面
- 多厚度面
- `InsertFeatureShellAddThickness`
- `InsertFeatureShell`

这些 API 调用细节不应污染统一模型，所以都在桥接层消化。

#### CADExchange -> Creo

Creo 创建关键在于：

- 正确构造 `PRO_E_SHELL_SRF`
- 正确构造 `PRO_E_ST_SHELL_LOCL_LIST`
- 正确写 `PRO_E_SHELL_FLIP`

### 8.5 历史问题与当前状态

壳特征是这五类中状态最整齐的一类之一。

- 已实现

- 现阶段实际落地更偏“最小无损子集”

### 9.2 Draft 的统一建模核心

Draft 的核心不是单一角度值，而是：

1. 拔模类型
   - 中性面
   - 分型线
   - 台阶

2. 拉伸/开模方向
3. 被拔模面集合
4. 基准构造参考
   - neutral plane
   - parting lines
5. 双侧角 / 分割模式

所以 Draft 的困难本质上是“构造组织差异”，不是单纯数值字段差异。

### 9.3 Creo / CADExchange / SW 映射总表

| 原生语义 | CADExchange | 目标语义 |
|---|---|---|
| SW Neutral Plane Draft | `draftType = NeutralPlane` | Creo Hinge = Plane |
| SW Parting Line Draft | `draftType = PartingLine` | Creo Hinge = Curve |
| SW Step Draft | 设计文档中保留 `Step`，当前代码多以降级/补偿处理 | Creo 侧需降级实现 |
| Creo Hinge = Plane | `NeutralPlane` | SW Neutral Plane Draft |
| Creo Hinge = Curve | `PartingLine` | SW Parting Line Draft |
| Creo Split / Side2 Angle | `isTwoSided + draftAngleSide2 + split info` | SW 双向分型线拔模或降级路径 |

### 9.4 提取阶段问题

#### SW 侧

1. 历史上最开始没有具体 Draft 提取器
   - 读侧会直接丢失拔模角度、参考面、开模方向、目标面

2. 若 `DirectionPull` 为空
   - 当前要用 neutral plane 作为 fallback
   - 这是为了满足统一层校验与重建要求

3. 当前稳定实现以 Neutral Plane Draft 为主
   - 更复杂类型仍需要继续补齐

#### Creo 侧

1. Draft 不是单一模式
   - Hinge 类型、Split 模式、双侧角等组合很多

2. 变角、台阶、复杂 split 很难天然映射到 SW 同构语义

### 9.5 创建阶段问题

#### SW 侧

1. FeatureType 分发容量曾写死到 `DatumPlane + 1`
   - 新增 `Draft` 后数组越界
   - 报 `Unsupported feature type (out of range): 10`

2. 即使有统一模型，写侧没有 `DraftHandler::Create` 也无法重建

3. 目前已补齐至少：
   - `NeutralPlaneDraft`
   - 基础方向、角度、目标面重建

#### Creo 侧

当前主要困难不在“有没有 API”，而在：

- 如何把统一层的中性面/分型线/双侧角正确映回 element tree
- 如何处理 Step Draft 与 Variable Draft 的降级

### 9.6 历史问题清单

#### 1. SW 读侧缺失

现象：

- 遇到 `L"Draft"` 只能空壳注册
- 统一模型里丢失关键拔模语义

#### 2. SW 写侧分发器越界

现象：

- `Draft` 的枚举值超出旧容量上限
- 导致写侧直接跳过

#### 3. 复杂类型只能受控降级

专题设计文档已经明确：

- Step Draft
- Variable Draft

不应假装成完全无损直映射。

### 9.7 当前结论

Draft 当前应理解为：

- 设计层已经有较完整统一方案
- 代码层以最小通用子集为主
- SW 基础 Neutral Plane Draft 已可读写闭环
- 更复杂 Draft 类型仍是后续扩展重点

## 12. 结论

如果把这五类工程类特征放在一起看，可以得到以下整体判断：

1. `CADExchange` 对工程类特征的统一设计已经形成了稳定方法论：
   - 先统一语义
   - 再桥接到目标 CAD
   - 允许受控降级
   - 尽量避免把原生 API 细节直接塞进统一结构

2. 倒角与倒圆角已经形成比较成熟的“参数语义统一”体系：
   - 倒角重在 `ChamferMode`
   - 倒圆角重在 `DriveType + Primary/SecondValue + Shape`

3. Rib 与 Draft 当前仍处于“结构持续收敛”阶段：
   - 专题文档已经走在代码前面
   - 后续需要把文档方案与 `UnifiedFeatures.h` 做一次系统对齐

4. Shell 是当前五类工程特征中最整齐的一类：
   - 结构稳定
   - 四向验证通过
   - 适合作为后续新增面操作类特征的参考模板

5. 对这五类特征而言，真正最难的从来不是“把字段读出来”，而是：
   - 统一解释不同 CAD 的方向语义
   - 统一解释构造方式差异
   - 在目标 CAD 中用稳定 API 重建出同一建模意图

因此后续若继续扩展工程类特征，最应该优先复用的不是某个单一特征的字段名，而是本文抽象出的统一设计原则与问题拆解方法。
