## 4.2 Creo：历史态 / 抑制恢复 / 引用对象

### 4.2.1 历史态相关能力

从 `ProFeature.h` 可直接确认：

- `ProFeatureWithoptionsSuppress(...)`
- `ProFeatureWithoptionsResume(...)`
- `ProFeatureWithoptionsRedefine(...)`
- `ProFeatureWithoptionsCreate(...)`

本机头文件证据：

- `E:\Application\Creo\Creo7.0_x64\Crack\Creo 7.0.0.0\Common Files\protoolkit\includes\ProFeature.h`

关键语义：

- `ProFeatureWithoptionsSuppress`：抑制指定特征
- `ProFeatureWithoptionsResume`：恢复指定特征
- `ProFeatureWithoptionsRedefine`：在可接受修改的上下文中重定义特征

这说明 Creo Toolkit 原生明确支持：

1. **临时让某批特征失活**
2. **恢复特征**
3. **在特征树上下文中修改特征**

虽然它不叫 SW 那种 `EditRollback`，但从能力上已经足够支撑：

- “停在某个历史位置”
- “暂时不考虑后续特征”

### 4.2.2 OTK 文档中的 suppress / resume 语义

本机 OTK HTML 直接给出：

- `t-pfcFeature-ResumeOperation.html`  
  Description: `Specifies a resume operation for a feature.`
- `t-pfcFeature-SuppressOperation.html`  
  Description: `This class defines a suppress operation.`

并且 `ResumeOperation` 里明确有：

- `SetWithParents(...)`

这说明在 OTK 层，恢复/抑制不仅存在，而且可以显式讨论父特征依赖。

### 4.2.3 几何与特征遍历

`ProFeature.h` 中还直接给出：

- `ProFeatureGeomitemVisit(...)`
- `ProFeatureStatusGet(...)`
- `ProSolidFeatVisit(...)`

含义很直接：

- 可以遍历某个特征产生的几何项
- 可以读取特征状态
- 可以遍历实体中的特征

这说明“**按特征收集几何**”在 Toolkit 层是有直接支撑的。

### 4.2.4 引用 / 选择对象是否存在

`ProReference.h` 直接给出：

- `typedef void* ProReference;`
- `ProReferenceTypeGet(...)`
- `ProReferenceIdGet(...)`
- `ProReferenceOriginaltypeGet(...)`
- `ProReferenceOriginalidGet(...)`
- `ProReferenceOwnerGet(...)`

从注释可以直接看出一个非常关键的点：

- `original type / original id`
- `backed up / copied locally / replaced with an alternate`

这说明 `ProReference` 不是简单“几何点坐标”，而是带有：

- 当前引用类型 / id
- 原始引用类型 / id
- owner
- alternate / local copy 状态

这是明显强于“只存 start/end/mid point”的。

`ProSelection.h` 直接给出：

- `ProSelectionAlloc(...)`
- `ProSelectionModelitemGet(...)`

仓库代码里也已实际使用：

- [CreoLibs/read_write/src/internal/read_extract.cpp](../CreoLibs/read_write/src/internal/read_extract.cpp:1386)  
  `ProReferenceToSelection(reference, &selection)`

因此，可以确认：

- **Creo 原生存在稳定引用对象体系**
- 其核心链路是 `ProReference <-> ProSelection <-> ProModelitem`

### 4.2.5 对本问题的结论

仅从官方资料与头文件角度：

1. **Creo 具备 rollback-like 能力**  
   不是叫 `EditRollback`，但 `suppress/resume/redefine` 足以构造历史态。
2. **Creo 可以按特征采最终几何**  
   `ProFeatureGeomitemVisit` 给了按特征遍历几何项的直接能力。
3. **Creo 存在比几何打分更稳定的引用体系**  
   `ProReference` / `ProSelection` 就是直接证据。

## 4.3 UG / NX：当前特征历史位置 / suppress / JournalIdentifier

### 4.3.1 “当前特征 / rollback”相关能力

`Features_Feature.hxx` 直接给出：

- `MakeCurrentFeature()`
- `Suppress()`
- `Unsuppress()`
- `Suppressed()`
- `Timestamp()`
- `GetParents()`
- `GetChildren()`
- `GetEntities()`

`Features_FeatureCollection.hxx` 直接给出：

- `SuppressFeatures(...)`
- `UnsuppressFeatures(...)`
- `SetEditWithRollbackFeature(...)`

其中最关键的是：

- `SetEditWithRollbackFeature(...)`

该注释明确写的是：

- `Sets the feature being edited with rollback`

这不是推断，而是头文件里直接出现了 `rollback` 语义。

所以 NX 原生能力上，不只是“能 suppress”，而是**明确提供了带 rollback 语义的编辑入口**。

### 4.3.2 当前特征最终几何

`Features_Feature.hxx` 同时给出：

- `GetEntities()`

这意味着从 NXOpen 特征对象本身，就可以直接问：

- 这个 feature 创建了哪些实体

再配合：

- `MakeCurrentFeature()`
- `SetEditWithRollbackFeature(...)`

理论上就能构造：

1. 把模型历史停到某特征
2. 只读取该特征在该历史态下的几何结果

### 4.3.3 持久引用 / 稳定对象标识

`NXObject.hxx` 直接给出：

- `JournalIdentifier()`
- `FindObject(journalIdentifier)`

头文件注释明确说明：

- journal identifier 可能在不同版本中生成值不同
- 但新版本应该还能通过旧 journal identifier 找回同一对象
- 该机制用于 journal record/playback

这已经非常接近“持久引用”概念：

- 它不等于 session 内临时 `tag_t`
- 它面向跨记录/跨重放找回对象

因此对本问题来说，可以明确得出：

- **UG/NX 存在比 `tag_t` 更适合做稳定对象恢复的原生标识：`JournalIdentifier`**

### 4.3.4 对本问题的结论

仅从官方头文件与本机 NX SDK：

1. **UG/NX 具备明确 rollback-like 能力**  
   `SetEditWithRollbackFeature(...)` 是直接证据。
2. **UG/NX 具备当前特征几何读取能力**  
   `GetEntities()` + feature history APIs 是直接能力基础。
3. **UG/NX 具备比 tag 更稳定的对象标识**  
   `JournalIdentifier()` / `FindObject(...)` 是直接证据。

## 6. 最终结论

### 6.1 关于“rollback / 当前特征历史态”

- **SolidWorks**：仓库已落地使用。
- **Creo**：官方能力存在，但当前仓库未落地成特征级历史态几何快照。
- **UG/NX**：官方能力明确存在，尤其 `SetEditWithRollbackFeature(...)` 是非常强的直接证据；当前仓库未接入。

### 6.2 关于“当前特征的最终几何，不含后续特征”

- **SW**：已实际做到。
- **Creo**：官方 `ProFeatureGeomitemVisit(...)` + feature suppress/resume 能支撑。
- **UG/NX**：官方 `GetEntities()` + `MakeCurrentFeature()` / `SetEditWithRollbackFeature(...)` 能支撑。

### 6.3 关于“持久引用”

- **SW**：仓库已直接使用 `GetPersistReference3`。
- **Creo**：没有找到与 SW 同名同形态的 byte-array persist API 证据；但 `ProReference` 明显是原生引用对象体系，且带 original/current id/type/owner 等信息，稳定性明显强于纯几何点。
- **UG/NX**：`tag_t` 不是理想持久标识，但 `JournalIdentifier()` / `FindObject(...)` 明确提供了更适合跨记录/回放恢复对象的能力。
