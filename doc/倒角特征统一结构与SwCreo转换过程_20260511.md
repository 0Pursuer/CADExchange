# 倒角特征统一结构与 Sw-CADExchange-Creo 转换过程_20260511

## 1. 文档目的

本文档以 `CADExchange` 为中心，整理 `SolidWorks` 与 `Creo` 倒角特征在统一模型中的转换过程。

重点回答三类问题：

1. `CADExchange` 里倒角特征是如何定义的
2. `SolidWorks -> CADExchange -> Creo` 的倒角转换链路是怎样的
3. `Creo -> CADExchange -> SolidWorks` 的倒角转换链路是怎样的

本文强调的是：

- `Sw <-> CADExchange <-> Creo`

## 2. CADExchange 中的倒角统一结构

核心定义位于：

- [UnifiedFeatures.h](D:\CADHHU\CADProj\CADExchange\core\UnifiedFeatures.h:594)
- [ChamferBuilder.h](D:\CADHHU\CADProj\CADExchange\service\builders\ChamferBuilder.h:22)
- [ChamferAccessor.h](D:\CADHHU\CADProj\CADExchange\service\accessors\ChamferAccessor.h:43)
- [TinyXMLSerializer.cpp](D:\CADHHU\CADProj\CADExchange\service\serialization\TinyXMLSerializer.cpp:1203)

### 2.1 `ChamferMode`

`CADExchange` 用 `ChamferMode` 表示“当前倒角的参数语义”：

- `UNKNOWN`
- `EQUAL_DISTANCE`
- `TWO_DISTANCES`
- `TWO_OFFSETS`
- `DISTANCE_ANGLE`
- `VERTEX_3DISTANCES`

这些模式的含义是：

- `EQUAL_DISTANCE`
  - 等距倒角
  - 只需要 `distance1`

- `TWO_DISTANCES`
  - 双距离倒角
  - 需要 `distance1 + distance2`

- `TWO_OFFSETS`
  - 双偏移倒角
  - 需要 `offset1 + offset2`
  - 主要用于保留 Creo 的 offset 语义

- `DISTANCE_ANGLE`
  - 距离 + 角度倒角
  - 需要 `distance1 + angle`

- `VERTEX_3DISTANCES`
  - 顶点三边倒角
  - 需要 `distance1 + distance2 + distance3`

### 2.2 `CChamferParams`

参数结构：

```cpp
struct CChamferParams {
  std::optional<double> distance1;
  std::optional<double> distance2;
  std::optional<double> distance3;
  std::optional<double> offset1;
  std::optional<double> offset2;
  std::optional<double> angle;
};
```

设计原则：

- 线性尺寸统一放在 `distance* / offset*`
- 角度统一放在 `angle`
- 哪些字段有效，取决于 `ChamferMode`

### 2.3 `CChamfer`

统一倒角特征结构：

```cpp
struct CChamfer : public CFeatureBase {
  ChamferMode mode{ChamferMode::UNKNOWN};
  CChamferParams params;
  std::vector<std::shared_ptr<CRefEntityBase>> references;
  std::optional<CPoint3D> firstEndFaceMarker;
};
```

字段意义：

- `mode`
  - 统一倒角类型

- `params`
  - 当前倒角的统一参数槽位

- `references`
  - 倒角作用几何引用
  - 当前普通边倒角主要保留 `Edge`

- `firstEndFaceMarker`
  - 第一端面标识点
  - 这是跨 CAD 对齐端面语义的关键字段

### 2.4 XML 表示

`TinyXMLSerializer` 中的倒角 XML 规则位于：

- [TinyXMLSerializer.cpp](D:\CADHHU\CADProj\CADExchange\service\serialization\TinyXMLSerializer.cpp:1203)

当前写法：

- `EqualDistance`
  - 只写 `Distance1`

- `TwoDistances`
  - 写 `Distance1 + Distance2`

- `TwoOffsets`
  - 写 `Offset1 + Offset2`

- `DistanceAngle`
  - 写 `Distance1 + Angle`

- `Vertex3Distances`
  - 写 `Distance1 + Distance2 + Distance3`

例如：

```xml
<Feature ID="FB-3" Name="倒角1" Suppressed="false" Type="Chamfer" Mode="EqualDistance" FirstEndFaceMarker="(1.69333,1.27,0)">
  <Parameters Distance1="0.254"/>
  <References>
    <ReferenceEntity ParentFeatureID="FB-2" TopologyIndex="0" StartPoint="(0,1.905,0)" EndPoint="(2.54,1.905,0)" MidPoint="(1.27,1.905,0)" Type="Edge"/>
  </References>
</Feature>
```

也就是说，`EqualDistance` 在统一模型中已经被规范为：

- 只保留 `Distance1`

## 3. 统一转换思想：Sw <-> CADExchange <-> Creo

这条桥接链路的核心思想不是“分别实现两个导入器”，而是：

1. 先把原生 CAD 倒角语义收敛到 `CADExchange::CChamfer`
2. 再从 `CADExchange::CChamfer` 重建到目标 CAD

也就是：

- `SolidWorks -> CADExchange -> Creo`
- `Creo -> CADExchange -> SolidWorks`

统一收敛时有 4 个关键点：

### 3.1 模式收敛

无论原生 CAD 如何表示，最终都收敛到：

- `EQUAL_DISTANCE`
- `TWO_DISTANCES`
- `TWO_OFFSETS`
- `DISTANCE_ANGLE`
- `VERTEX_3DISTANCES`

### 3.2 参数收敛

原生 CAD 的尺寸、偏移、角度都要收敛到：

- `distance1 / distance2 / distance3`
- `offset1 / offset2`
- `angle`

### 3.3 引用收敛

普通边倒角当前统一采用：

- `Edge` 作为主 `reference`

而不是强绑定跨 CAD 不稳定的 face 拓扑引用。

### 3.4 第一端面语义收敛

两端 CAD 中“第一端面”的原生定义并不相同，因此统一模型里采用：

- `firstEndFaceMarker`

来保存第一端面的语义。

当前设计上不要求：

- marker 数值坐标在跨 CAD 后完全一致

而要求：

- 它们仍然落在同一几何端面上

## 4. SolidWorks -> CADExchange

SW 侧核心实现位于：

- [feature_chamfer.cpp](D:\CADHHU\CADProj\SwLibs\read_write\src\internal\feature_chamfer.cpp:844)

入口：

- `ChamferHandler::Extract(...)`

### 4.1 提取步骤

SW 读侧主要做 5 件事：

1. 获取 `IChamferFeatureData2`
2. 读取 SW 原生倒角类型 `swType`
3. 映射成 `CADExchange::ChamferMode`
4. 读取统一参数、引用和 `firstEndFaceMarker`
5. 构造 `CADExchange::CChamfer`

### 4.2 SW 原生类型到统一模式

映射在：

- [feature_chamfer.cpp](D:\CADHHU\CADProj\SwLibs\read_write\src\internal\feature_chamfer.cpp:211)

关系如下：

- `swChamferAngleDistance -> DISTANCE_ANGLE`
- `swChamferDistanceDistance -> TWO_DISTANCES`
- `swChamferVertex -> VERTEX_3DISTANCES`
- `swChamferEqualDistance -> EQUAL_DISTANCE`

### 4.3 `swType=0` 的容错处理

SW 中存在一个特殊情况：

- UI 里有倒角
- 但 `API get_Type()` 返回 `0`

当前处理方式是：

- 不立即丢弃
- 结合以下信息推断统一模式：
  - `get_EqualDistance()`
  - `GetEdgeChamferDistance(0/1)`
  - `get_EdgeChamferAngle()`
  - `GetVertexChamferDistance(...)`

这保证了部分“旧式/模糊型”倒角仍可回到 `CADExchange`

### 4.4 SW 参数到统一参数

SW 参数提取不是单纯按 `swType` 写，而是按**最终统一模式**写：

- `EQUAL_DISTANCE`
  - 只写 `distance1`

- `TWO_DISTANCES`
  - 写 `distance1 + distance2`

- `DISTANCE_ANGLE`
  - 写 `distance1 + angle`

- `VERTEX_3DISTANCES`
  - 写 `distance1 + distance2 + distance3`

### 4.5 SW 第一端面提取

SW 普通倒角当前也统一为：

- `Edge` 主引用
- `firstEndFaceMarker` 端面语义

当前第一端面规则是：

1. 优先从参考边相邻几何中推断第一端面
2. 必要时结合距离侧/方向侧规则选面
3. 失败后回退到可用 face 列表

## 5. CADExchange -> Creo

Creo 写侧核心实现位于：

- [feature_chamfer.cpp](D:\CADHHU\CADProj\CreoLibs\read_write\src\internal\feature_chamfer.cpp:1456)

入口：

- `CreateChamferFeature(...)`

### 5.1 重建步骤

`CADExchange -> Creo` 的主要过程是：

1. 从 `ChamferAccessor` 读取统一模式
2. 选择对应的 Creo schema
3. 将统一参数写入 Creo 特征树
4. 使用 `references` 与 `firstEndFaceMarker` 恢复参考语义
5. 创建 Creo 倒角特征

### 5.2 统一模式到 Creo schema

主要关系：

- `EQUAL_DISTANCE -> PRO_CHM_D_X_D`
- `TWO_DISTANCES -> PRO_CHM_D1_X_D2`
- `TWO_OFFSETS -> PRO_CHM_O_X_O / PRO_CHM_O1_X_O2`
- `DISTANCE_ANGLE -> PRO_CHM_45_X_D / PRO_CHM_ANG_X_D`
- `VERTEX_3DISTANCES -> PRO_FEAT_CORN_CHAMF`

### 5.3 角度单位

当前统一约定：

- `CADExchange.angle` 使用弧度
- 写回 Creo 时再转换回角度

这样可以保证：

- `Sw <-> CADExchange <-> Creo` 的角度口径一致

## 6. Creo -> CADExchange

Creo 侧核心实现位于：

### 6.1 提取步骤

Creo 读侧分两层：

1. 从 Creo 特征树读取原始倒角信息
2. 映射到 `CADExchange::CChamfer`

### 6.2 Creo schema 到统一模式

映射在：

- [feature_chamfer.cpp](D:\CADHHU\CADProj\CreoLibs\read_write\src\internal\feature_chamfer.cpp:71)

关系如下：

- `PRO_CHM_D_X_D -> EQUAL_DISTANCE`
- `PRO_CHM_D1_X_D2 -> TWO_DISTANCES`
- `PRO_CHM_O_X_O -> TWO_OFFSETS`
- `PRO_CHM_O1_X_O2 -> TWO_OFFSETS`
- `PRO_CHM_45_X_D -> DISTANCE_ANGLE`
- `PRO_CHM_ANG_X_D -> DISTANCE_ANGLE`
- `PRO_FEAT_CORN_CHAMF -> VERTEX_3DISTANCES`

### 6.3 Creo 参数到统一参数

提取后的统一参数语义为：

- `D_X_D`
  - `distance1`

- `D1_X_D2`
  - `distance1 + distance2`

- `O_X_O / O1_X_O2`
  - `offset1 + offset2`

- `45_X_D / ANG_X_D`
  - `distance1 + angle`

- 顶点倒角
  - `distance1 + distance2 + distance3`

### 6.4 Creo 引用与第一端面

Creo 当前普通边倒角也采用：

- `Edge` 作为主引用
- `firstEndFaceMarker` 保持第一端面语义

也就是说，读到 `CADExchange` 后，Creo 与 SW 的倒角都已经被压平到同一种统一结构。

## 7. CADExchange -> SolidWorks

SW 写侧核心实现位于：

- [feature_chamfer.cpp](D:\CADHHU\CADProj\SwLibs\read_write\src\internal\feature_chamfer.cpp:988)

入口：

- `ChamferHandler::Create(...)`

### 7.1 重建步骤

`CADExchange -> SW` 的过程：

1. 从 `ChamferAccessor` 读取统一模式
2. 映射成 SW 原生倒角类型
3. 准备尺寸/角度参数
4. 调用 `InsertFeatureChamfer(...)`
5. 必要时做 post-create 对齐

### 7.2 统一模式到 SW 原生类型

映射在：

- [feature_chamfer.cpp](D:\CADHHU\CADProj\SwLibs\read_write\src\internal\feature_chamfer.cpp:267)

关系如下：

- `DISTANCE_ANGLE -> swChamferAngleDistance`
- `TWO_DISTANCES -> swChamferDistanceDistance`
- `VERTEX_3DISTANCES -> swChamferVertex`
- `EQUAL_DISTANCE` 需要特殊处理

### 7.3 EqualDistance 的 SW 创建策略

当前已经验证：

- 直接用 `swChamferEqualDistance(16)` 创建，SW 有概率返回一个 UI 可见但 API 难稳定回读的特征

因此当前策略改为：

1. 先按 `DistanceDistance` 创建
2. 让 `d1 = d2 = distance1`
3. 创建后读取 `IChamferFeatureData2`
4. 显式 `put_EqualDistance(VARIANT_TRUE)`
5. `ModifyDefinition(...)`

这样可以保证：

- UI 上是等距离倒角
- API 回读时仍能恢复成 `EqualDistance`

### 7.4 第一端面 post-create 对齐

当前 SW 写侧不再依赖建前预测 flip。

采用的是：

1. 先创建倒角
2. 创建后重新检测第一端面
3. 如果与输入 `firstEndFaceMarker` 不一致，则尝试反转
4. 成功标准不是 marker 坐标完全相同，而是：
   - marker 足够接近
   - 或者 marker 仍然落在同一几何面上

这使得：

- `Creo -> CADExchange -> SW`
- `SW -> CADExchange -> Creo`

在端面 marker 数值不完全一致的情况下，仍能保证语义一致

## 8. 两条完整链路如何理解

### 8.1 `SolidWorks -> CADExchange -> Creo`

这条链路的本质是：

1. 从 SW 原生倒角提取出统一模式、统一参数、统一引用和第一端面语义
2. 在 `CADExchange` 中保持为 `CChamfer`
3. 再按统一模式重建为 Creo 倒角 schema

### 8.2 `Creo -> CADExchange -> SolidWorks`

这条链路的本质是：

1. 从 Creo 原生 schema 提取出统一模式、统一参数、统一引用和第一端面语义
2. 在 `CADExchange` 中保持为 `CChamfer`
3. 再按统一模式重建为 SW 原生倒角

因此真正的桥接核心不是：

- `Sw` 直接懂 `Creo`

而是：

- `Sw` 和 `Creo` 都先懂 `CADExchange`

## 9. 当前设计中的关键原则

### 9.1 offset 和 distance 不能混用

`TWO_OFFSETS` 必须与 `TWO_DISTANCES` 分开建模，否则会丢失原始语义。

### 9.2 EqualDistance 只保留 `Distance1`

在统一模型与 XML 中：

- `EqualDistance` 只保留一个有效参数 `Distance1`

不再冗余写 `Distance2=Distance1`

### 9.3 FirstEndFaceMarker 是端面语义，不是绝对点坐标

跨 CAD 后，marker 数值可能变化，但只要仍然落在同一几何面上，就应视为第一端面语义保持成功。

### 9.4 统一模型优先于 CAD 原生细节

桥接的目标不是把某个 CAD 的内部树原样拷贝给另一个 CAD，而是：

- 先转成统一倒角语义
- 再由目标 CAD 用它自己的稳定方式重建出来

---

## 10. 总结

如果从整体架构看，倒角桥接应理解为：

- `SolidWorks <-> CADExchange <-> Creo`

其中：

- `CADExchange` 提供统一倒角定义
- `Sw` 和 `Creo` 都分别实现“到统一模型”和“从统一模型”两条路径

统一模型中的倒角核心是：

1. `ChamferMode`
2. `CChamferParams`
3. `references`
4. `firstEndFaceMarker`

而当前实现中最关键的两个落地点是：

- `EqualDistance` 在统一模型中只保留 `Distance1`
- `FirstEndFaceMarker` 按“几何面语义一致”来判定跨 CAD 重建成功
