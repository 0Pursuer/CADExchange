# 半边结构冗余边过滤与几何对比规范化记录_20260525

## 1. 背景

在 `chamfer_test_2` 的 geometry compare 中，早期 `FB-2/FB-4` 存在一组典型差异：

- `SRC unmatched OPEN_EDGE`
- `DST extra OPEN_EDGE`
- 这些 edge 数量相同、位置都落在同一局部几何附近
- 视觉上实体外形一致，但 compare 仍报 feature mismatch

进一步对比 source / rebuilt geometry 后确认：

- 这些差异不是实体轮廓真的不同
- 而是“半边结构 / seam / edge partition”带来的额外 edge

常见表现：

- 同一圆柱面被分成两半
- 然后在 seam 上出现两条本不该影响比较结果的额外 line edge
- source 和 rebuilt 的 seam 位置不同，于是 compare 误判

---

## 2. 已确认的具体问题

在 `FB-2` 的对比中，source / rebuilt 的主体几何是同形的，但：

- source 圆孔对应的 seam 竖边落在一组点上
- rebuilt 圆孔对应的 seam 竖边落在另一组等价点上

这类差异的本质是：

- 不是实体生成语义变了
- 而是内核对同一周期面 / 半边结构的 edge 切分不同

因此，如果 compare 直接按 edge 的 `start/mid/end` 强匹配：

- 会把同形几何误判成 mismatch

---

## 3. 当前采用的规范化思路

经过针对 `FB-2` 的逐边分析，当前更合适的 compare 预处理规则是：

1. 先处理普通 edge
2. 再处理 arc/circle 关系
3. 如果若干 arc 可以合并为一个完整 circle：
   - 说明该局部存在“半边结构导致的 seam 变化”可能
4. 记录这些被合并 arc 的原始端点
5. 再扫描剩余 `OPEN_EDGE`
6. 只要某条 edge 的任一端点命中这些原始 arc 端点，就将其视为半边结构带来的冗余 edge，并从 compare 输入中剔除

这个规则不再限制：

- edge 必须是直线
- edge 必须属于圆柱面
- edge 必须沿某个固定方向

原因是实际冗余 edge 可能来自：

- 圆柱面
- 圆锥面
- 管道扫掠面
- 其他周期面 / 半边参数化结构

所以只靠“方向 / 面类型”来删边不稳，端点命中合并 arc 的原始端点更泛化，也更贴近该类问题本质。

---

## 4. 为什么要按“每个合并 circle 分组”

实现时不建议把所有端点放进一个全局黑名单数组，而应按每个合并得到的 circle 单独分组。

原因：

1. 不同局部区域可能恰好存在数值接近的点
2. 全局 point 池会扩大误删范围
3. 分组后可以把过滤影响范围限制在对应局部 circle/seam 上

因此当前推荐数据结构是：

- 每个“合并后的 circle”对应一组 `points`
- 后续 `FilterHalfStructureEdges(...)` 逐组判断

---

## 5. 当前代码落点

当前 compare 预处理逻辑位于：

- `CADExchange/service/geometry/GeometryCollectorBase.h`

关键逻辑链：

1. `ClassifyEdges(...)`
2. `MergeArcs(...)`
3. `FilterHalfStructureEdges(...)`
4. `MatchCircles(...)`
5. `MatchArcs(...)`
6. `MatchOpenEdges(...)`

规范化过程中的核心状态包括：

- `src_half_structure_groups`
- `dst_half_structure_groups`

每组包含：

- 合并 circle 的 `center`
- `radius`
- 被合并 arc 的原始端点集 `points`

---

## 6. 当前规则的收益

在本次 `chamfer_test_2` 排查中，应用该规则后：

- `FB-2` 的 mismatch 已经消失
- `FB-4` 也不再被这类 seam 额外边干扰
- 剩余问题集中到了真正更深层的 `FB-6/7/8/9`

这说明：

- 该规则成功去除了 compare 侧的拓扑噪声
- 后续剩余 mismatch 更接近真实建模语义问题

---

## 7. 这条规则解决的范围与边界

### 能解决的

- 同形几何因 seam / 半边结构不同导致的额外 edge
- 周期面或半边结构上的多余分边
- `arc -> circle` 合并后残留的等价冗余 `OPEN_EDGE`

### 不能解决的

- 真实几何轮廓变化
- 拉伸/倒角/圆角的参数差异
- 上游 sketch/profile 语义已经变了导致的实体差异

例如当前 `FB-6` 顶部轮廓问题就不属于本规则解决范围，它仍需要从草图提取 / useEdge 语义上继续排查。

---

## 8. 与后续 Creo 修复的关系

这条 compare 规范化规则是必要的，但不是全部：

1. 它解决的是 compare 误报
2. 它不能替代读写语义修复

在 `chamfer_test_2` 中：

- `FB-2/FB-4` 的 seam 噪声属于 compare 层问题
- `FB-6/FB-7/FB-8/FB-9` 则更接近 Creo 草图 projection/useEdge 语义丢失问题

因此后续修复应继续分层推进：

- compare 侧：保留半边结构冗余边过滤
- bridge 侧：修复 projection line -> useEdge 的提取与重建
