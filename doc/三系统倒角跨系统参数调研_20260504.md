# 三系统倒角跨系统参数调研

## 1. 目的

本文用于整理 SolidWorks、Creo、UG/NX 三个系统中“倒直角/倒角”特征的：

- 支持的生成方式
- 可提取参数
- 创建所需参数
- 三系统之间的重要差异

目标是为后续在 `CADExchange\core\UnifiedFeatures.h` 中设计统一倒角存储结构提供依据。

## 3. 三系统支持的倒角生成方式

### 3.1 SolidWorks

当前证据确认 SW 实体倒角至少支持：

- `AngleDistance`
  - 距离 + 角度
- `DistanceDistance`
  - 双距离
- `EqualDistance`
  - 等距倒角
- `VertexChamfer`
  - 顶点倒角，三边距离

SW 当前 demo 中选择对象可落在：

- 边集合
- 顶点
- loop
- face 接口可拿到，但 demo 将其视作非常规路径

### 3.2 Creo

Creo 原生支持的倒角类型比另外两家更细，普通倒角与角点倒角是两个 feature 家族。

普通倒角 `PRO_FEAT_CHAMFER` 支持：

- 边倒角
  - `PRO_ROUND_REF_EDGE`
- 面面倒角
  - `PRO_ROUND_REF_SURF_SURF`
- 边面倒角
  - `PRO_ROUND_REF_EDGE_SURF`

普通倒角参数 schema 支持：

- `PRO_CHM_45_X_D`
- `PRO_CHM_D_X_D`
- `PRO_CHM_D1_X_D2`
- `PRO_CHM_ANG_X_D`
- `PRO_CHM_O_X_O`
- `PRO_CHM_O1_X_O2`

普通倒角 shape 支持：

- `PRO_CHM_TANGENT_LEGS`
- `PRO_CHM_OFFSET_SURFACE`

角点倒角是独立 feature：

- `PRO_FEAT_CORN_CHAMF`
- 参数由 `corner + edge1 + edge2 + edge3` 组成

### 3.3 UG/NX

UG 明确拆成两套：

- 实体倒角 `Features::ChamferBuilder`
- 草图倒角 `SketchChamferBuilder`

实体倒角支持三种参数模式：

- `SymmetricOffsets`
- `TwoOffsets`
- `OffsetAndAngle`

实体倒角支持两种生成方法：

- `EdgesAlongFaces`
  - 常规沿相邻面偏置边
- `FacesAndTrim`
  - 偏置面并修剪，自由形意味更强

草图倒角支持三种模式：

- `Symmetric`
- `Asymmetric`
- `OffsetandAngle`

## 4. 三系统可提取参数

### 4.1 SolidWorks

当前已确认可直接从 `IChamferFeatureData2` 提取：

- 倒角类型
- 边倒角距离 1/2
- 顶点倒角距离 1/2/3
- 倒角角度
- 等距标志
- 保留特征
- 切向传播
- 传播到零件
- 方向翻转
- 选择对象集合
  - 边
  - 面
  - loop
  - 顶点

### 4.2 Creo 普通倒角

当前可确认可提取：

- `schema`
- `shape`
- `reference type`
- 边集合 / 面引用 / 参考面
- `leg1`
- `leg2`
- `leg type`
- `attach type`
- `transitions`
- patch / stop-at-reference 等 transition 子结构

### 4.3 Creo 角点倒角

当前可确认可提取：

- corner 引用
- `edge1`
- `edge2`
- `edge3`

### 4.5 UG 草图倒角

## 5. 三系统创建倒角所需参数

### 5.1 SolidWorks

最小创建参数集可归纳为：

- 通用：
  - 选择对象集合
  - 倒角类型
- `AngleDistance`
  - `distance1`
  - `angle`
- `DistanceDistance`
  - `distance1`
  - `distance2`
- `EqualDistance`
  - `distance1`
- `VertexChamfer`
  - 顶点引用
  - `distance1`
  - `distance2`
  - `distance3`
- 常见可选行为参数：
  - `isFlipped`
  - `tangentPropagation`
  - `keepFeatures`
  - `propagateToParts`

### 5.2 Creo 普通倒角

最小创建参数集可归纳为：

- 通用：
  - `feature type = PRO_FEAT_CHAMFER`
  - `attach type`
  - 至少一个 set
- 每个 set：
  - `schema`
  - `shape`
  - `reference type`
  - 对应参考几何
    - 边集合，或
    - 两个面，或
    - 边 + 面
  - `leg1`
  - 某些 schema 还需要 `leg2`
  - 某些 schema 还需要 `reference surface`

### 5.3 Creo 角点倒角

最小创建参数集可归纳为：

- `feature type = PRO_FEAT_CORN_CHAMF`
- `corner`
- `edge1`
- `edge2`
- `edge3`

### 5.4 UG 实体倒角

最小创建参数集可归纳为：

- 边选择
- `Option`
  - `SymmetricOffsets / TwoOffsets / OffsetAndAngle`
- `Method`
  - `EdgesAlongFaces / FacesAndTrim`
- 参数值
  - `firstOffset`
  - `secondOffset` 或 `angle`

常见可选控制：

- `reverseOffsets`
- `allInstances`
- `tolerance`

### 5.5 UG 草图倒角

最小创建参数集可归纳为：

- 两条草图曲线
- `ChamferOption`
- `distance1`
- `distance2` 或 `angle`
- `helpPoint`

常见可选控制：

- `trimInputCurves`
- 是否创建尺寸

## 6. 三系统之间的重要差异

### 6.1 普通倒角 vs 角点倒角

- SW
  - 有 `VertexChamfer`，但仍落在同一 feature 参数体系里
- Creo
  - 普通倒角和角点倒角是两个独立 feature 家族
- UG
  - 当前证据没有看到与 Creo `PRO_FEAT_CORN_CHAMF` 完全等价的单独“角点倒角 feature family”

这说明统一模型不能简单假设“所有 corner/vertex chamfer 都只是普通倒角的一种 mode”。

### 6.2 边/面参考拓扑的表达能力

- SW
  - 当前强证据主要围绕边、顶点、loop
- Creo
  - 明确支持 `edge / surf-surf / edge-surf`
- UG
  - 实体倒角主要围绕边选择，但通过 `Method` 区分是沿面偏置边还是偏置面修剪

这说明统一模型里最好显式保留参考拓扑或生成方法，而不要只保留“输入对象列表”。

### 6.3 参数模式差异

- SW
  - `equal_distance / distance_distance / angle_distance / vertex_3dist`
- Creo
  - `45_x_d / d_x_d / d1_x_d2 / ang_x_d / o_x_o / o1_x_o2`
- UG
  - `symmetric_offsets / two_offsets / offset_and_angle`

Creo 的 schema 数量最多，也最不适合被粗暴压缩成仅有 `distance1/distance2/angle`。

### 6.4 自由形/偏置面语义

- Creo
  - `shape = tangent_legs / offset_surface`
- UG
  - `method = edges_along_faces / faces_and_trim`
- SW
  - 当前本地强证据里没有找到与上述两类严格对等的显式公开维度

这说明统一模型可能需要一个跨系统的“生成策略/shape”层，而不是只存数值。

### 6.5 草图倒角是否纳入同一统一结构

- UG 明确有 `SketchChamferBuilder`
- SW/Creo 本次调查重点是实体倒角，没有拿到等强度的草图倒角对照证据

因此如果当前目标是实体特征统一，建议先把草图倒角排除在第一版 unified feature 之外，或者用单独 family 表达。

## 7. 对统一存储结构的直接建议

结合当前目标，倒角的 unified 结构第一版建议尽量收缩，只保留：

- `mode`
- `params`
- `references`

不建议第一版引入：

- `family`
- `reference_kind`
- `shape_or_method`
- `reversed/flipped`
- `tangentPropagation`
- `keepFeatures`
- `propagateToParts`
- `allInstances`
- 其它系统行为型字段

原因：

- 当前目标是先定义统一存储结构，不是把各 CAD 原生 feature 家族一比一镜像到 `UnifiedFeatures.h`
- `references` 本身已经通过 `CRefEntityBase` 体系承载了“边 / 面 / 顶点 / 草图段 / 基准”这些引用对象类型
- `reversed`、`tangentPropagation` 等字段更像写回或 CAD 原生行为控制，不属于第一版最核心的统一语义

### 7.1 建议的最小 unified 结构

建议第一版倒角统一结构等价于：

- `mode`
  - `EQUAL_DISTANCE`
  - `TWO_DISTANCES`
  - `DISTANCE_ANGLE`
  - `VERTEX_3DISTANCES`
- `params`
  - `distance1`
  - `distance2`
  - `distance3`
  - `angle`
- `references`
  - 使用现有 `std::vector<std::shared_ptr<CRefEntityBase>>`

### 7.2 Creo schema 到 unified mode 的映射规则

Creo 原生 schema 建议在 unified 层压缩映射为通用 mode，而不是保留 Creo 专属枚举：

- `PRO_CHM_45_X_D`
  - 映射为 `DISTANCE_ANGLE`
  - 其中 `angle = 45deg`
- `PRO_CHM_D_X_D`
  - 映射为 `TWO_DISTANCES`
- `PRO_CHM_D1_X_D2`
  - 映射为 `TWO_DISTANCES`
- `PRO_CHM_ANG_X_D`
  - 映射为 `DISTANCE_ANGLE`
- `PRO_CHM_O_X_O`
  - 统一按双距离语义处理，映射为 `TWO_DISTANCES`
- `PRO_CHM_O1_X_O2`
  - 统一按双距离语义处理，映射为 `TWO_DISTANCES`

这意味着 unified 层更关心“参数解释方式”，而不是 Creo 原生命名本身。

### 7.3 第一版 unified feature 的落地建议

建议第一版实现范围控制为：

1. 先把实体倒角统一成一个 `CChamfer`
2. 只保留四种 `mode`
   - 等距
   - 双距离
   - 距离角度
   - 顶点三距离
3. `params` 只保留数值语义
4. `references` 直接复用现有引用体系
5. 暂不把 UG 草图倒角纳入第一版 unified feature

这样既能覆盖三系统当前最主要的共同语义，也避免第一版把结构做重。
